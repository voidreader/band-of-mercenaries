## 로드맵 요구사항 요약

M8a "세력의 귀환"은 M7 이후 첫 후속 마일스톤이다. 기존 단일 M8 "세력 재도입"을 축소된 수직 절편으로 재정의하고, 시작 생활권과 직접 연결되는 대표 세력 2~3개만 먼저 깊게 구현한다. 동시에 세력·지명·엘리트·연계 의뢰 결과에 전투 보고서 MVP를 적용하여 의뢰 결과가 "성공" 한 줄이 아니라 읽을 수 있는 사건으로 남도록 한다.

핵심 목표는 세력 기능의 기본 골격을 닫고, 전투 보고서의 읽는 재미를 최소 범위에서 검증하는 것이다. 실제 턴 기반 전투 시뮬레이터는 M8b에서 구현하며, M8a의 전투 보고서는 기존 성공/실패 결과를 해석하여 저장하는 보고서 계층으로 제한한다.

선행 의존성: M1~M7 모두 완료.

## 포함 범위

M8a는 다음 범위를 포함한다.

- 대표 세력 2~3개 선정과 생활권 접촉점 정의.
- 세력 지명 의뢰, 세력 상점 축소판, 세력 전용 보상 1~2개.
- 세력 의뢰와 위업·신뢰도·칭호·지역 상태 조건 연결.
- `CombatReportService` 또는 동등한 전투 보고서 생성 서비스.
- 세력·지명·엘리트·연계 의뢰 결과의 요약/상세 전투 보고서 저장.
- 전투 보고서 템플릿, 전장 분위기, 적 유형, 결정적 장면 키워드 데이터.

## 완료 기준

- [ ] 대표 세력 2~3개가 생활권 사건과 연결되어 등장한다.
- [ ] 세력 지명 의뢰가 위업, 신뢰도, 칭호, 지역 상태 조건으로 노출된다.
- [ ] 세력 상점 상품이 제작 재료 또는 레시피와 연결된다.
- [ ] 세력 보상이 용병단 위상 또는 지역 상태에 영향을 준다.
- [ ] 세력·지명·엘리트·연계 의뢰 결과에 전투 보고서가 저장된다.
- [ ] 전투 보고서는 요약과 상세 로그를 구분하여 표시된다.
- [ ] 전투 보고서는 완료 후 재접속해도 동일하게 유지된다.
- [ ] M1~M7 기능 회귀 이상 없음.
