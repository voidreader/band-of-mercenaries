### 14개 세력 tier_range 구성 (Supabase 실측)

| 분포 | 세력 수 | 예시 |
|------|--------|------|
| [1,3] | 1 | 모험가 길드 |
| [1,4] | 2 | 상인 연합, 뿌리의 맹세단(실측은 [1,4]) |
| [2,4] | 4 | 전사 길드, 도둑 길드, 태양 교단, 송곳니 결사 |
| [2,5] | 1 | 균형 감시자 |
| [3,5] | 5 | 마탑 연합, 금지된 서고, 황혼 공학회, 심층 망치단, 화산 심장단 |
| [4,5] | 1 | 혈계 귀족회 |

> **수정 반영**: 기획서 5절은 뿌리의 맹세단을 [1,3]으로 표기했으나 Supabase 실측은 **[1,4]**. 페이즈 1 기획서 역반영 필요 (주석 처리).

### 입력 기획서 초안 수치

- **태그 확률**: 거점 1~3 = 30% / 4~7 = 20% / 8~15 = 10% / 16+ = 5%
- **태그 평판**: 일반 +1 / 거점 근접 +2
- **전용 난이도**: 기본=지역 평균 +1 / 고급=평균 +2
- **전용 보상**: 기본 ×1.3 / 고급 ×1.5
- **전용 평판**: 기본 +5~7 / 고급 +8~10
- **해금 평판**: 기본 11 / 고급 61

## 데이터 분석

### 분석 1: 1시간 실플레이 기준 퀘스트 완료 수 (기준 페이스)

3개 파티 × 평균 파견시간으로 **실플레이 시간당 완료 수**를 산정한다.

| 난이도 | base_duration 평균 | 시간배수 `(1+(d-1)×0.2)` | 파견 소요(초, AGI50) | 1파티 시간당 완료 |
|--------|--------------------|------------------------|-----------------------|--------------------|
| 2 | 71 | 1.2 | 85 | 42 |
| 3 | 71 | 1.4 | 99 | 36 |
| 4 | 71 | 1.6 | 114 | 32 |
| 5 | 71 | 1.8 | 128 | 28 |

> base_duration = (60+75+80+70)/4 ≈ 71초. 실제 게임은 AGI 보정·이동·결과 팝업 간 간격 발생.

**실 체감 페이스 (파티 3개 병행, 난이도 3 위주 가정):** 시간당 **50~60회** 가량 완료. 하한 30회 / 상한 90회.

**M1 검증용 기준치: 시간당 40회**를 가정하여 이하 분석 진행.

### 분석 2: 태그 확률 5~30%에서 세력 평판 누적 속도

가입 전(단서 보유) 상태에서 한 세력의 태그가 일반 퀘스트에 부여되는 확률 × 완료 평판.

| 확률 | 시간당 태그 건수 | 평판 +1 / 시간 | 평판 +2 / 시간 | 가입조건 평판 1 도달 |
|------|---------------|--------------|--------------|------------------|
| 5% | 2회 | 2 | 4 | 15분 (+1) / 8분 (+2) |
| 10% | 4회 | 4 | 8 | 8분 / 4분 |
| 20% | 8회 | 8 | 16 | 4분 / 2분 |
| 30% | 12회 | 12 | 24 | 2분 / 1분 |

**판정**: 가입 전 태그만으로 "평판 > 0" 조건은 **거점 근접 시 2~4분**이면 달성. 단서 발견 직후 자연스럽게 가입 트리거로 이어지는 속도로 적절.

**문제점 발견**: 단서 1수준만 보유한 상태에서도 거점 근접 10% + 평판 +1로 가입 조건을 쉽게 넘김. 단서 없이는 태그 대상에서 제외되므로 OK지만, **가입 조건이 "평판 > 0" 하나뿐**이면 동시 가입 3세력 상한이 의미 없어진다. 이 문제는 `FactionJoinService.canJoin`에서 `joinRankMin`/`joinNeedsClue`로 완화되므로 시스템적 문제는 아님.

### 분석 3: 가입 후 기본 트랙 해금 시간 (평판 11)

가입 후에는 태그 우선순위 상승 규칙이 적용된다 (분석 6 참조). **가입 1세력 + 태그 50% 고정** 시나리오로 분석.

- 시간당 태그 건수: 40 × 0.5 = 20건
- 평판 +1 기본 + 거점 근접 2/3 비중 가중치: 평균 `+1.5`/건
- 시간당 평판: 30

→ 평판 11 도달: **22분**. 기획서 "10분 내외" 목표와 2배 차이.

**조정안 검토**:
- (A) 가입 세력 태그 확률 **70% 고정**: 40×0.7×1.5 = 42/h → 16분 (여전히 미달)
- (B) 가입 세력 태그 확률 **100% 고정 (가입 시 강제)**: 40×1.0×1.5 = 60/h → **11분** ✅
- (C) 평판 +1/+2 → **+2/+3**로 상향 (50% 고정 시): 40×0.5×2.5 = 50/h → 13분 (근접)

**권고**: **(B)** 가입 세력 태그 확률 100% + 거점 근접도 규칙은 **비가입 세력에만 적용**. 이유:
- 가입 = 명시적 선택이므로 해당 세력 루프가 "명확히 진행"되어야 체감이 좋음
- 비가입 다중 세력 확률 가중 로직이 단순해짐 (가입 세력을 특별 케이스로 분리)
- 기획서 "가입 중은 해당 세력 태그 우선"과 일치

**가입 2~3세력 시 100% 분배**: 라운드 로빈(최근 태그 받은 세력 제외) 또는 균등 랜덤. 균등 랜덤 권장 (구현 단순).

### 분석 4: 전용 퀘스트 보상 배수 경제 영향

`최종 보상 = base_reward × difficulty.reward_multiplier × (1 + track_bonus)` 가산 구조.

**기본 트랙 ×1.3 (track_bonus = 0.30), raid 난이도 3, T3 ×2명**:

| 항목 | 값 |
|------|---|
| 기본 보상 | 100 × 2.2 = 220 |
| 트랙 보정 | 220 × 1.30 = 286 |
| 인건비 T3×2 | 100 |
| 파견비(평균) | 60 |
| 순수익(성공) | **126 G** |
| vs 일반 동급 | 220 − 160 = 60 G |
| **배수** | **2.1배** |

**고급 트랙 ×1.5 (track_bonus = 0.50), raid 난이도 5, T4 ×3명**:

| 항목 | 값 |
|------|---|
| 기본 보상 | 100 × 4.5 = 450 |
| 트랙 보정 | 450 × 1.50 = 675 |
| 인건비 T4×3 | 300 |
| 파견비(평균) | 100 |
| 순수익(성공) | **275 G** |
| vs 일반 동급 | 450 − 400 = 50 G |
| **배수** | **5.5배** |

**판정**: 고급 ×1.5는 **순수익 기준 5.5배**로 과다. 고정 비용(인건비+파견비) 비중이 커서 보상 배수가 순수익에서 크게 증폭된다. 이는 passive_values 리포트의 P2와 동일 패턴.

**권고**:
- 기본 트랙 `track_reward_bonus = 0.30` → **0.30 유지** (2.1배는 "가입 혜택"으로 허용)
- 고급 트랙 `track_reward_bonus = 0.50` → **0.40로 하향**
- 중첩 규칙 (가산): `final = base × mult × (1 + track + passive + rank)` — 세력 패시브(+0.12), 명성 A(+0.15)와 **모두 가산**. 상한 클램프 **+0.80** 설정 (과도 중첩 방지)

**조정 후 고급 트랙 시뮬레이션**:
- 450 × (1 + 0.40) = 630 → 순수익 230 G → **vs 일반 4.6배** (여전히 고수익이나 허용 범위)

### 분석 5: 난이도 "지역 평균 +1/+2" 정량화

세력 tier_range 기반으로 기본/고급 트랙 난이도를 clamp[1,5] 내에서 산정. 퀘스트 `difficulty`는 real이지만 `difficulties.level` 테이블이 1~5이므로 실사용 난이도는 **1~5 정수**로 배정.

| tier_range | 평균 | 기본 트랙 (평균+1, clamp) | 고급 트랙 (평균+2, clamp) |
|-----------|------|--------------------------|--------------------------|
| [1,3] | 2 | **2~3** (3개) | **3~4** (4개) |
| [1,4] | 2.5 | **3~4** | **4~5** |
| [2,4] | 3 | **3~4** | **4~5** |
| [2,5] | 3.5 | **4~5** | **5** |
| [3,5] | 4 | **4~5** | **5** |
| [4,5] | 4.5 | **5** | **5** |

**문제**: tier_range [4,5] 세력(혈계 귀족회)은 기본/고급 모두 난이도 5만 배정됨 → 단조로움.

**조정안**: 세력 기본 트랙은 tier_range **하위 3 난이도**(예: [4,5]이면 4), 고급 트랙은 **상위 3 난이도**(5)로 고정하는 단순화.

| 세력 | 기본 트랙 3개 | 고급 트랙 4개 |
|------|-------------|-------------|
| 모험가 길드 [1,3] | 2, 2, 3 | 3, 3, 4, 4 |
| 상인 연합 [1,4] | 2, 3, 3 | 4, 4, 5, 5 |
| 전사/도둑/태양/송곳니 [2,4] | 3, 3, 4 | 4, 4, 5, 5 |
| 균형 감시자 [2,5] | 3, 4, 4 | 5, 5, 5, 5 |
| 마탑/서고/공학회/망치단/화산 [3,5] | 4, 4, 5 | 5, 5, 5, 5 |
| 혈계 귀족회 [4,5] | 4, 5, 5 | 5, 5, 5, 5 |

→ 난이도 5 편중이 강함. 이는 **"세력 엔드게임 컨텐츠"**라는 포지션상 허용. 난이도 분포는 `difficulty` real 값 유지하되 실사용 clamp 1~5.

### 분석 6: 98개 풀 다양성 검증

가입 3세력 × 3(기본) = **최대 9개 기본 트랙** 풀 + × 4(고급, 평판 61 해금) = **최대 12개 고급** 풀 동시 유효.

**1시간 갱신 주기 × 5~13개 슬롯**으로 중복 발생 여부:

| 가입 세력 수 | 유효 전용 풀 (기본+고급) | 슬롯 5개 시 전용 점유율 가정 | 1시간 중복 위험 |
|-------------|------------------------|------------------------|---------------|
| 1 | 3 (기본만) ~ 7 (양 트랙) | 전용 2개/슬롯 | **높음** (풀 3일 때 5 슬롯에 2개 배치 → 1시간 주기 재등장) |
| 2 | 6~14 | 전용 3개/슬롯 | 중간 |
| 3 | 9~21 | 전용 4개/슬롯 | 낮음 |

**권고**:
- **전용 퀘스트 노출 상한**: 한 번에 `min(가입세력수 × 2, 슬롯 수 × 0.5)`개로 제한 → 슬롯 5/10/13에 전용 최대 2/5/6개
- **재노출 금지 쿨다운**: 전용 퀘스트가 만료·완료되면 해당 퀘스트 ID를 **6시간(실시간)** 동안 풀에서 제외
- **초반 보호**: 가입 1세력 기본 트랙 3개만 해금된 초반에는 전용 풀이 작으므로, 초반 플레이어용 `min(전용, 2)` 상한으로 중복 감소

98개 총량은 **충분**. 단, 가입 초반 경험만 위해 **기본 트랙을 3개 → 5개로 확장**하는 것도 선택지지만, 이는 페이즈 1 기획안 재조정을 요구하므로 **운영 규칙(쿨다운)** 으로 해결 권장.

### 분석 7: F → A 랭크 평판 vs 세력 평판 비례

**랭크 평판 진행**: 평균 난이도 3, 성공률 70%(성공) + 20%(대성공) + 10%(실패) 가정.
- 퀘스트당 기대 평판: 0.7×30 + 0.2×60 + 0.1×0 = **33**
- 시간당 완료 40회 → 시간당 **1,320** 랭크 평판
- F→E 500 = **23분**
- E→D 1500 = **68분 (1.1h)**
- D→C 6000 = **4.5h**
- C→B 17000 = **12.9h**
- B→A 55000 = **41.7h**
- F→A 총 **60시간**

**세력 평판 진행 (가입 후)**:
- 태그 분당 0.5~1 평판, 전용 기본 3회 × +6 = 18 평판
- 평판 11(기본 해금) 약 11분
- 평판 61(고급 해금) 약 50~70분
- 평판 100(만점) 약 100~140분 (약 2시간)

**비례 평가**:
- 랭크 "B 달성" = ~21시간, 한 세력 평판 만점 = ~2시간 → **세력이 10배 빠른 루프**
- 기획서 "30~60분 연속 플레이"는 **초반 한 세력 가입 → 기본 트랙 완료 사이클**에 해당 → 일치
- 랭크 A(60시간)와 세력 평판(2시간) 격차는 **장기 vs 단기 목표 계층**으로 의도된 설계

**판정**: 비례 적절. 단, 플레이어가 "세력 평판을 채운 뒤 할 일이 없다"는 감을 받지 않도록 **세력별 평판 유지/하락 규칙**(예: 적대 세력 태그 거절 시 소폭 감소)이 M3~M4에서 보완되어야 함. 현재 M1 범위 밖.

### 분석 8: 가입 세력 태그 우선순위 규칙

분석 3에서 "가입 세력 태그 100% 고정" 권고 완료. 상세 규칙:

```
태그 대상 선정 (일반 퀘스트 생성 시):
  1. 현재 리전에서 단서 보유 세력 목록 수집 (공개 세력 포함)
  2. 적대 상태(-100) 세력 제외
  3-A. 가입 세력이 목록에 1개 이상 있는 경우:
       → 태그 확률 100% (일반 퀘스트에는 무조건 가입 세력 중 하나 부여)
       → 복수 가입이면 균등 랜덤으로 1개 선택
  3-B. 가입 세력이 목록에 없는 경우:
       → 거점 근접도 기반 확률 5/10/20/30% 적용
       → 판정 성공 시 비가입 단서 보유 세력 중 가중 랜덤
  4. 태그 평판 획득량:
       - 일반 퀘스트 완료: 해당 리전 거점 근접도 1~3 → +2, 그 외 → +1
       - 전용 퀘스트 완료: 기본 트랙 +5~7 / 고급 트랙 +8~10 (별도 규칙)
```

**100% 적용 근거**:
- 가입자는 명시적 소속 선택 → 일상 루프에서 평판 진행이 체감되어야 "가입 의미" 전달
- 가입 3세력까지 상한이 있으므로 과도한 평판 인플레이션은 없음
- 기본 트랙 해금 시간 목표(10~15분)를 맞추려면 **가입 1세력 + 태그 50% 이상이 필수 전제**

**예외 (충돌 세력)**: 가입 세력의 `conflict_faction_ids`에 있는 세력은 타 세력 가입 시 자동 탈퇴 규칙으로 해결(기존 로직). 태그 선정에서는 별도 처리 불필요.

## 문제점

### 치명적

**P1. 기획서 "기본 트랙 해금 10분 내외" 목표와 태그 확률 5~30% 초안의 불일치**
- 근거: 분석 3. 태그 50%에서도 22분, 10% 기본 거점 확률에서는 55분.
- 해결: **가입 세력 태그 확률 100% 고정** 규칙 도입 (분석 8 상세).

**P2. 고급 트랙 ×1.5 보상 배수가 일반 대비 순수익 5.5배 증폭**
- 근거: 분석 4. 고정 비용 비중이 큰 고난도에서 배수가 과증폭.
- 해결: 고급 트랙 보상 보너스 `+0.50` → **`+0.40`로 하향**.

### 중요

**P3. 1시간 갱신 × 전용 풀 규모 불일치 (가입 1세력 초기)**
- 근거: 분석 6. 가입 1세력 기본 트랙 3개로 슬롯 5개에 2개 고정 배치 시 1시간마다 동일 퀘스트 재등장.
- 해결: **6시간 재노출 금지 쿨다운** + 전용 노출 상한 `min(가입수×2, 슬롯×0.5)` 도입.

**P4. 뿌리의 맹세단 tier_range 기획서 오기**
- 근거: 기획서 5절은 [1,3], Supabase 실측 [1,4].
- 해결: 페이즈 1 기획서에 **역반영 주석 추가**, 전용 퀘스트 난이도는 [1,4] 기준으로 생성.

### 경미

- tier_range [4,5] 세력(혈계)의 난이도 배분 단조 — 기획 설계상 의도된 것으로 판단, 허용.
- 세력 평판 만점 도달 후 장기 목표 부재 — M3~M4 범위로 이월.

## 플레이어 체감 분석

### 초반 (F~E, 가입 0~1세력)

- 공개 세력 도감에서 바로 가입 가능 (clue 없이도 `join_rank_min` 도달 시 가입).
- 가입 즉시 태그 100% 작동 → **"세력 가입한 순간부터 모든 일반 퀘스트가 해당 세력 소속감으로 엮임"**.
- 기본 트랙 해금(11) 약 11~13분 → 한 세션 안에 성취감 1회 도달. 플레이 세션 유지력 ✓.
- 전용 퀘스트 3개 해금 → 슬롯 5개 중 2개가 내 세력 전용으로 교체되는 변화 시각적 체감.

### 중반 (D~C, 가입 1~2세력)

- 고급 트랙 해금(61) 약 50~70분 → 첫 세션 말미 또는 2세션 째 도달.
- 전용 풀 확장 (3→7개), 쿨다운 규칙으로 중복 없이 **두 세션 이상 다양한 전용 노출** 가능.
- 두 세력 동시 진행 시 태그 균등 분배 → 한 세력에 몰입하고 싶은 플레이어는 충돌 세력 가입으로 슬림화 전략 선택.

### 엔드 (B~A, 가입 2~3세력)

- 세력 평판 100 도달 세력이 2~3개 생김 → 전용 풀 포화 (최대 21개).
- 랭크 A 도달 시간(60시간)은 세력 루프 복수 회 반복하며 자연히 달성되는 배경 목표.
- 전용 ×1.4 + 패시브 +12% + 명성 +15% 중첩 상한 `+0.80` 적용 → 순수익 일반 대비 최대 4~5배 수준. 과도 수익 방지.

### 체감 리스크

- **가입 세력 태그 100% 규칙이 경직되어 보일 수 있음** → 비가입 단서 세력의 평판을 올리고 싶은 플레이어는 불만족. 해결: 가입 세력이 자신의 현재 리전에 거점 없음(단서 미보유)일 경우 100% 규칙 적용 안 됨(가입 세력이 후보 목록에 애초에 없음) → 자연 해결.
- 쿨다운 6시간은 단일 세션을 길게 유지하는 유저에게 유의미하나 짧게 플레이하는 유저는 체감 불가 → 실시간 기준이므로 공정.

final_reward = base_reward
             × difficulty.reward_multiplier

### 3. 태그 선정 의사코드 (개발 명세용)

```dart
String? resolveFactionTag({
  required int regionId,
  required List<String> joinedFactionIds,
  required Map<String, int> clueLevelsInRegion,
  required int proximityTier, // 1~4 (1=1~3거리, 4=16+)
  required Random random,
}) {
  // 1. 단서 보유 세력 후보
  final candidates = clueLevelsInRegion.entries
      .where((e) => e.value >= 1)
      .map((e) => e.key)
      .where((id) => !_isHostile(id))
      .toList();
  if (candidates.isEmpty) return null;

  // 2. 가입 세력 우선 경로
  final joinedCandidates = candidates.where(joinedFactionIds.contains).toList();
  if (joinedCandidates.isNotEmpty) {
    return joinedCandidates[random.nextInt(joinedCandidates.length)];
  }

  // 3. 비가입 거점 확률
  final prob = switch (proximityTier) {
    1 => 0.30, 2 => 0.20, 3 => 0.10, _ => 0.05,
  };
  if (random.nextDouble() > prob) return null;
  return candidates[random.nextInt(candidates.length)];
}
```

### 4. 평판 획득량 (완료 처리 시)

```dart
int calculateFactionReputationGain({
  required bool isFactionExclusive,
  required bool isAdvancedTrack,
  required int proximityTier,
  required Random random,
}) {
  if (isFactionExclusive) {
    return isAdvancedTrack
      ? 8 + random.nextInt(3)   // +8~10
      : 5 + random.nextInt(3);  // +5~7
  }
  // 태그 퀘스트 (일반)
  return proximityTier <= 2 ? 2 : 1;
}
```

## 시뮬레이션 (확정안 적용)

### 시나리오 A — 초반 플레이어, 모험가 길드 E랭크 가입

**가정**: 세력 평판 0 가입 → 기본 해금(11)까지 시간 측정.

- 세션 1분 완료 6회(파티 3개 병행, 난이도 2~3)
- 가입 세력 태그 100% → 매 퀘스트 평판 +1 또는 +2
- 현재 리전 거점 근접 가정(proximity_tier=2) → 평판 +2 비중 50%
- 평균 분당 평판: 6 × 1.5 = 9
- **기본 해금 11 도달: 약 1.2분** → 너무 빠름

**재검토**: 파티 3개 병행이 안정적으로 유지되는 건 중반 이후. 초반 파티 1~2개 가정 시:
- 분당 1~2 퀘스트 완료 × 평판 +1.5 = 분당 1.5~3
- 기본 해금: **4~7분** → 기획 목표(10분 내외)보다 빠름

**판정**: 가입 세력 100% 규칙이 강력해 기본 해금은 빠르게 달성. **고급 해금까지의 시간**이 중간 목표로 기능 (50~70분).

조정 여지: 목표를 엄격하게 10분으로 맞출 필요 없음 — 빠른 기본 해금 = "세력에 합류하니 바로 세력 전용 퀘스트가 보인다"는 UX 일관성. 오히려 **유리한 방향**.

### 시나리오 B — 중반 플레이어, 2세력 가입, 고급 해금

**가정**: 모험가 길드 + 마탑 동시 가입, 균등 분배.

- 각 세력 평판 0에서 시작, 시간당 완료 40회 × 50% 균등 분배 = 각 세력 20회 태그/h
- 각 세력 시간당 평판: 20 × 1.5 = 30
- 기본 해금(11) 도달: **22분**
- 기본 트랙 3회 × 평균 +6 = 18 → 누적 평판 29 (기본 해금 후 빠르게)
- 고급 해금(61) 도달: (61-29) / 30 = 추가 **64분** 후, 기본 해금 포함 총 **86분**

**판정**: 2세력 병행은 "속도 저하 대신 넓이"라는 선택지로 작동. 어느 한 세력에 몰입하려면 충돌 세력 가입으로 슬림 전략 가능.

### 시나리오 C — 전용 풀 쿨다운 효과 검증

**가정**: 가입 1세력(모험가), 기본 트랙 3개 해금, 슬롯 5개.

- 슬롯 내 전용 노출 상한 `min(1×2, 5×0.5)` = **2개**
- 기본 트랙 풀 3개 중 2개가 활성 슬롯에 표시
- 1개 완료 시: 쿨다운 6h 시작 → 남은 2개에서 1개 추가
- 다음 1시간 갱신 시: 쿨다운 중인 1개 제외 후 풀 2개 중 1개 재선정 (1/2 확률로 나머지 1개 신규)

**판정**: 쿨다운 6h로 **단일 세력 초반도 중복 최소화**. 완료 페이스 1시간당 4~6 전용 소진이면 6h 후 재등장해도 자연스러움.

### 시나리오 D — 엔드게임 3세력 평판 100

- 3세력 모두 평판 100 달성: 각 2~3시간, 총 **6~9시간**
- 랭크 A 도달(60시간) 동안 세력 루프 반복 가능 횟수: 6회 정도
- "100 도달 후 할 일" 부재 → M3~M4 범위로 이월

> 본 리포트는 `quest_pools`의 **세력 전용 퀘스트 98개** 벌크 생성 대상. `/data-generator faction-quest --brief @Docs/content-design/[content]20260417_faction_quests.md`에서 참조.

- **대상 테이블**: `quest_pools`
- **생성 수량**: **98개** (14세력 × 7 = 기본 3 + 고급 4)
- **수치 범위**:
  - `difficulty` (real, 1~5 clamp):
    - 기본 트랙: 세력 tier_range 하위 3 난이도 (예: [2,4] → 3, 3, 4)
    - 고급 트랙: 세력 tier_range 상위 3 난이도 (예: [2,4] → 4, 4, 5, 5)
  - `min_region_diff` / `max_region_diff` (real): 세력 tier_range 기반
    - [1,3] → 0~3 / [1,4] → 0~4 / [2,4] → 1~5 / [2,5] → 1~6 / [3,5] → 2~8 / [4,5] → 3~8
  - `type` (real, quest_type.id 인덱스): 세력별 유형 배분은 페이즈 1 기획서 4절 표 적용
  - `faction_tag` (text, 신규 컬럼): 세력 ID 고정 (예: `faction_adventurers_guild`)
  - `is_faction_exclusive` (bool, 신규): 모두 `true`
  - `min_reputation` (int, 신규): 기본 3개 → 11, 고급 4개 → 61
  - `sector_type` (text, 신규): 모두 `null`
- **외래 키 제약**:
  - `faction_tag` → `factions.id` 존재 필수
  - `type` → `quest_types.id` 매핑 (raid/hunt/escort/explore 4개 중)
- **balance 근거**:
  - 난이도 clamp 1~5 = `difficulties` 테이블 범위
  - tier_range 하위 3 / 상위 3 분할 = 기본/고급 트랙 차별화 (분석 5)
  - `min_reputation` 11 / 61 = 세력 평판 만점 100 대비 11%/61% 지점 (분석 7)
