## 1. 개요

본 리포트는 페이즈 1 #3에서 정의된 4 감정 반응의 권장 수치(분노 atk ×1.30 / def ×0.80 / 절망 hit -0.20 / eva -0.15 / 슬픔 skip 0.50 / 투지 death_resist +0.20 / eva +0.15)를 정밀화한다. 검토 과정에서 페이즈 1 #3의 트레잇 키워드 가정과 실제 `traits` 테이블 데이터를 매칭하고, M8b CombatSimulator 산식 정합을 확인했다.

### 1.1 주요 보완 결과

다음 항목은 페이즈 3 #2 시드와 페이즈 4 #3 명세에 반드시 반영한다.

| 항목 | 초안 (페이즈 1 #3) | 보완 결과 |
|------|--------------------|----------|
| 트레잇 키워드 매칭 | `fierce`·`protective`·`caring`·`wrathful`·`compassionate`·`stoic` 등 가상 키워드 | 실제 `traits` 테이블 매칭으로 분노=`vengeful`·`berserker_talent`·`madman`·`slayer`, 슬픔=`guardian`·`empathic`·`team_player`·`mentor`, 절망 면제=`iron_will`·`unyielding`·`hardened`·`fearless`·`composed` 5종 |
| 분노 def 수치 | 0.20 감소 (multiplicative 0.80) | M8b 기존 `debuff_defense_down` 0.25 default 대비 균형 — 0.20 유지 |
| 절망 hit/eva 수치 | hit -0.20 / eva -0.15 | M8b 기존 `debuff_accuracy_down` 0.10 default의 **2배**. 위기 강조 — 그대로 유지 |
| 슬픔 skip 산식 | 50% 확률 행동 슬롯 스킵 | `apply_method='none'` + simulator effectId 분기. mez_stunned (100% 스킵 1턴)와 차별화 — 2턴 동안 50% 확률 |
| 투지 + 솔로 cap 가산 | 페이즈 1 #3 Q-1 미결 | 투지 +0.20은 사망 저항 합산식에 포함한 뒤 `capForQuest`로 재clamp. 솔로 cap은 **0.95 유지**하며 cap 통과 후 별도 가산은 금지 |
| stack_policy | ignore (감정끼리만 ignore) | 4 emotional 사이 ignore, 일반 buff/debuff와는 공존 (multiplicative 누적) |

### 1.2 최종 결론

페이즈 1 #3 권장 수치는 거의 그대로 유지하되, 트레잇 키워드를 실제 데이터로 교체하고 솔로 cap + 투지 가산 정책을 확정한다.

| 감정 | apply_method | intensity (대표) | duration | 발동 확률 | 트레잇 가중 |
|------|--------------|-----------------|---------|---------|-----------|
| 분노 | multiplicative | atk **0.30** / def -0.20 | 3턴 | 60% | +20% (4종 매칭) |
| 절망 | additive | hit **-0.20** / eva -0.15 | 3턴 | 80% | 면제 (5종 매칭) |
| 슬픔 | none (special) | skip_chance **0.50** | 2턴 | 50% | +30% (4종 매칭) |
| 투지 | additive | death_resist **+0.20** / eva +0.15 | 4턴 | 100% | (자동) |

## 2. 구현 기준 확인

본 절은 현재 M8b 상태 효과 모델과 전투 시뮬레이터 산식에서 감정 반응 4행을 안전하게 확장하기 위한 기준을 확인한다. 특히 기존 10개 상태 효과는 `effectId`별 helper로 일부 산식이 고정되어 있으므로, 페이즈 4 #3에서는 emotional 전용 helper를 추가해 동일 ID 재사용 없이 처리한다.

### 2.1 `combat_status_effects` 현재 10행 default 매트릭스

| ID | kind | hook_target | stack | apply | default_intensity | default_duration |
|----|------|-------------|-------|-------|-------------------|------------------|
| buff_attack_up | buff | [attack] | refresh | multiplicative | 0.20 | 2 |
| buff_defense_up | buff | [defense] | refresh | multiplicative | 0.20 | 3 |
| buff_accuracy_up | buff | [hit] | refresh | additive | 0.15 | 2 |
| buff_evasion_up | buff | [evasion] | refresh | additive | 0.10 | 2 |
| debuff_attack_down | debuff | [attack] | refresh | multiplicative | 0.20 | 2 |
| debuff_defense_down | debuff | [defense] | refresh | multiplicative | 0.25 | 3 |
| debuff_accuracy_down | debuff | [hit] | refresh | additive | 0.10 | 2 |
| mez_stunned | mez | [action_skip] | refresh | none | 1 (의미 없음) | 1 |
| dot_bleeding | dot | [round_end] | stack | proportional | 1 (stack) | 3 |
| dot_poisoned | dot | [round_start] | stack | absolute | 3 | 3 |

본 리포트 4 emotional 행은 기존 default 대비:
- 분노 atk 0.30 vs buff_attack_up 0.20: **+50% 강화** (위기 반응 강조)
- 분노 def -0.20 (multiplicative 0.80) vs debuff_defense_down 0.25: 약간 약화 (분노는 양날의 검이지만 buff 우선)
- 절망 hit -0.20 vs debuff_accuracy_down 0.10: **2배 강화** (전멸 임박의 무력감)
- 절망 eva -0.15: 신규 hook (debuff_evasion_down 미존재)
- 슬픔 skip 0.50: mez_stunned (100% 스킵 1턴)와 차별화 — 2턴 동안 50% 확률 스킵
- 투지 death_resist +0.20: 신규 hook (기존 상태 효과는 death_resist 미사용)
- 투지 eva +0.15 vs buff_evasion_up 0.10: **+50% 강화**

**부호 정책**: 기존 `debuff_accuracy_down`은 `default_intensity=0.10`을 양수로 저장하고 시뮬레이터가 `buffSum - debuffSum`으로 차감한다. 반면 `emotional_despair`는 `kind='emotional'`이고 `effectId` 전용 helper에서 `hit=-0.20`, `evasion=-0.15`를 직접 가산하는 signed additive 효과로 처리한다. 따라서 §9 SQL은 `default_intensity=-0.20`을 사용한다. 데이터 검증 계층에서 `default_intensity >= 0` 제약을 추가하지 않는다.

### 2.2 실제 `traits` 테이블 매칭 (109행 중 매칭 17행)

페이즈 1 #3 §3.3.2 가상 키워드와 실제 `traits.key` 매칭:

| 감정 가중 카테고리 | 페이즈 1 #3 가상 키워드 | 실제 `traits.key` 매칭 | 매칭 수 |
|------------------|----------------------|----------------------|--------|
| 분노 +20% | `fierce`·`vengeful`·`wrathful` | `vengeful` (Mental), `berserker_talent` (Talent innate), `madman` (Mental), `slayer` (CombatStyle) | **4종** |
| 슬픔 +30% | `protective`·`caring`·`compassionate` | `guardian` (CombatStyle), `empathic` (Mental), `team_player` (Behavior), `mentor` (Behavior) | **4종** |
| 절망 면제 | `iron_will`·`unyielding`·`stoic` | `iron_will` (Talent innate), `unyielding` (Survival evolved), `hardened` (Mental), `fearless` (Mental), `composed` (Mental) | **5종** |
| 투지 자동 (가중 없음) | — | — | 0종 |

→ **`TraitEmotionalKeywords` 정적 상수 13 키 매핑** (분노 4 + 슬픔 4 + 절망 면제 5). 페이즈 4 #3 spec에서 정확 ID 그대로 사용.

### 2.3 M8b CombatSimulator hook 위치 (페이즈 1 #3 §3.3.3 정합)

```
Phase 3 라운드 (M8b §FR-9) 흐름에 trailing 추가:
  1. DoT round_start
  2. 종료 조건 평가 (a)/(b)
  3. 행동 순서 정렬
  4. 행동 실행 (각 전투원 1회):
     - 행동 직전 sorrow=skip_chance 50% → recordSkippedTurn 평가
     - 액션 후 사망 마킹 → 감정 trigger: 분노 (광역 60%)
     - 액션 후 중상 마킹 → 감정 trigger: 슬픔 (인접 row 1명 50%)
     - 액션 종료 후 파티 HP 평가 → 최초 < 25% 도달 시 감정 trigger: 절망 (광역 80%)
     - 액션 종료 후 trigger mercId HP 평가 → 최초 < 30% 도달 시 감정 trigger: 투지 (본인 100%)
  5. 라운드 종료
     - DoT round_end
     - 모든 상태 효과 duration -1 (자연 해제)
```

전투 단위 중복 방지를 위해 `despairTriggered`와 `determinationTriggeredMercIds` 같은 latch를 둔다. `stack_policy='ignore'`는 현재 활성 emotional 중복만 막으므로, duration 종료 뒤 같은 전투에서 같은 임계값이 반복 발동되는 문제는 별도 latch로 차단한다.

## 3. 4 감정 default 수치 확정

| 필드 | 값 | 근거 |
|------|-----|------|

**special 처리**: mez_stunned는 보유 시 행동 슬롯 100% 스킵 1턴, sorrow는 보유 시 행동 슬롯 50% 확률 스킵 2턴. simulator에서 effectId 분기:
```dart
if (effect.id == 'mez_stunned') {
  recordSkippedTurn(combatant);  // 100% skip
} else if (effect.id == 'emotional_sorrow') {
  if (sorrowRng.nextDouble() < 0.50) {
    recordSkippedTurn(combatant);  // 50% skip
  }
}
```

### 3.5 솔로 cap 0.95 + 투지 +0.20 가산 정책 (페이즈 2 #2 정합)

페이즈 1 #3 Q-1 결정:
- M8b §FR-11.5 사망 저항 산식: `death_resistance_final = clamp(base + traitBonus + factionBonus + ..., 0.20, capForQuest)`
- 본 문서: emotional_determination +0.20을 **cap 통과 후 별도 가산하지 않고 합산식에 포함**
- 즉 `clamp(base + ... + emotional_determination, 0.20, capForQuest)` 형태
- 솔로 의뢰 capForQuest = 0.95 → base가 충분히 높은 용병만 cap 0.95에 도달

근거:
- 만약 cap 통과 후 +0.20 가산하면 솔로 의뢰에서 사망 저항 1.15 도달 가능 (사실상 100% 면역)
- M8.5 컨셉 "안전망 없음 + 영웅적 발휘" 정합 → cap 자체 0.95 유지, 투지가 cap 도달 가능성을 높임

→ **확정**: `clamp(base + traitBonus + factionBonus + emotional_determination_bonus, 0.20, capForQuest)`. cap은 페이즈 2 #2 매트릭스 그대로 (일반 0.80 / 체인 0.90 / 소수 0.90 / 솔로 0.95).

→ 회피 +0.15는 별도 hook이므로 evasion clamp [0, 0.75]에 자유 가산.

## 4. 발동 확률 정밀화

본 절은 사건 발생 시 감정 상태가 실제로 적용될 확률과 트레잇 보정 방식을 확정한다. 절망과 투지는 전투 단위 최초 임계값 도달 기준이며, 분노와 슬픔은 사망·중상 사건마다 평가한다. 이미 emotional 상태를 보유한 용병은 우선순위 평가 후에도 새 감정 적용을 받지 않는다.

### 4.1 사건별 발동 확률 매트릭스

페이즈 1 #3 §3.3.1 권장값 그대로 확정:

| 사건 | 트리거 시점 | 대상 | 감정 | 발동 확률 | 트레잇 가중 |
|------|----------|------|------|---------|-----------|
| 동료 사망 | M8b 사망 마킹 직후 | 생존 파티 전원 | 분노 | **60%** | `vengeful`/`berserker_talent`/`madman`/`slayer` 보유 시 **+20%** |
| 동료 중상 | M8b 중상 마킹 직후 | 중상자와 같은/인접 row 1명 | 슬픔 | **50%** | `guardian`/`empathic`/`team_player`/`mentor` 보유 시 **+30%** |
| 파티 HP 합계 <25% | HP 변화 처리 직후 최초 임계값 도달 | 생존 파티 전원 | 절망 | **80%** | `iron_will`/`unyielding`/`hardened`/`fearless`/`composed` 보유 시 **0%** (면제) |
| 트리거 mercId HP <30% | HP 변화 처리 직후 최초 임계값 도달 | 본인 (간판/솔로 참가/체인 주인공) | 투지 | **100%** | (가중 없음) |

중복 발동 정책:
- **분노**: 동료 사망 사건마다 평가한다. 이미 emotional 상태인 용병은 제외한다.
- **슬픔**: 동료 중상 사건마다 대상 후보 1명을 뽑아 평가한다. 이미 emotional 상태인 후보는 제외하고, 후보가 없으면 미발동 처리한다.
- **절망**: 파티 HP 합계가 처음 25% 미만으로 내려간 순간 전투당 1회만 평가한다.
- **투지**: eligible mercId별 전투당 1회만 평가한다. 발동 후 duration이 끝나도 같은 전투에서 재발동하지 않는다.

### 4.2 발동 확률 sensitivity 분석

각 감정 확률 ±10%p 변경 시 영향 (1주 평균 50 전투 가정):

#### 4.2.1 분노 발동 확률 sensitivity

| 발동 확률 | 동료 사망 시 분노 발생 (1주, 동료 사망 ~5건) | 1전투당 평균 분노 |
|---------|-------------------------------------------|------------------|
| 40% | ~2 | 0.04 |
| 50% | ~2.5 | 0.05 |
| **60%** (권장) | ~3 | **0.06** |
| 70% | ~3.5 | 0.07 |
| 80% | ~4 | 0.08 |

60% = 분노 보유자 1전투당 평균 0.06 ~ 1주 3건. 트레잇 보유 시 80% → 0.08 ~ 1주 4건. 적정.

#### 4.2.2 절망 발동 확률 sensitivity

| 발동 확률 | 전멸 임박 발생 빈도 (1주, ~3건) | 1전투당 평균 절망 |
|---------|------------------------------|------------------|
| 60% | ~1.8 | 0.04 |
| 70% | ~2.1 | 0.05 |
| **80%** (권장) | ~2.4 | **0.06** |
| 90% | ~2.7 | 0.07 |

전멸 임박은 의뢰 자체가 위험 → 80% 발동 = 위기에 위기 추가 자연. iron_will 보유자는 면제로 자연 차별.

#### 4.2.3 슬픔 발동 확률 sensitivity

| 발동 확률 | 동료 중상 시 슬픔 발생 (1주, 중상 ~8건) | 1전투당 평균 슬픔 |
|---------|------------------------------------|------------------|
| 30% | ~2.4 | 0.05 |
| 40% | ~3.2 | 0.07 |
| **50%** (권장) | ~4 | **0.09** |
| 60% | ~4.8 | 0.11 |

50% + 가중 80%(트레잇 보유) = 1주 4~6건. 적정.

#### 4.2.4 투지 발동 확률 sensitivity

투지는 100% 자동. 단 트리거 조건(`flagship/solo/chain 주인공 HP <30%`)이 자연 빈도 제한. 1주 평균 ~2~3건.

→ **권장 확률 그대로 확정** (페이즈 1 #3 §3.3.1).

### 4.3 트레잇 가중 + 확률 매트릭스

| 감정 | 기본 확률 | 트레잇 보유 시 |
|------|---------|--------------|
| 분노 | 60% | 80% (+20%p) |
| 슬픔 | 50% | 80% (+30%p) |
| 절망 | 80% | 0% (면제) |
| 투지 | 100% | (가중 없음) |

## 6. 전투 흐름 영향 시뮬레이션

본 절은 확정 수치가 평균 6라운드 전투 흐름에 주는 영향을 검산한다. 수치는 정밀 몬테카를로 결과가 아니라 구현 전 밸런스 sanity check이며, 페이즈 4 #3 구현 후 고정 seed 회귀 테스트로 재검증한다.

### 6.1 분노 발동 후 적 공격 사망률 증가

분노 발동 (atk +30% / def -20%) 후 1라운드 시뮬레이션:

| 케이스 | 적 공격 평균 피해 | 적 공격 사망률 (cap 0.80) |
|-------|---------------|------------------------|
| 일반 (분노 없음) | 30 (d3 적) | 2% |
| 분노 적용 (def -20%) | 30 × 1.20 = 36 | **3.2%** (+60%) |

분노 자체 피해 증가 (atk +30%):
| 케이스 | 파티 1라운드 처치 | 라운드당 처치 +% |
|-------|---------------|-----------------|
| 일반 | 1.5 적 | - |
| 분노 적용 | 1.5 × 1.30 = 1.95 | **+30%** |

→ 분노 발동은 1라운드당 추가 처치 +30%, 적 피해 +60% (방어 약화). 양날의 검 정합.

### 6.2 절망 발동 후 파티 평균 명중률 감소

절망 발동 (hit -20% / eva -15%) 후:

| 케이스 | 파티 평균 명중률 | 파티 평균 회피율 |
|-------|--------------|--------------|
| 일반 | 75% | 30% |
| 절망 적용 | 75% - 20% = **55%** | 30% - 15% = **15%** |

→ 명중률 27% 감소, 회피율 50% 감소. 전멸 임박이 가속화되는 효과. 의도된 위기 강화.

### 6.3 슬픔 발동 후 행동 슬롯 손실

슬픔 발동 (skip 50% / 2턴) 후 1전투(평균 6라운드) 영향:

| 케이스 | 행동 슬롯 손실 | 파티 라운드 손실 |
|-------|-------------|--------------|
| 슬픔 없음 | 0 | 0% |
| 슬픔 발동 (2턴 × 50%) | 평균 1턴 손실 | ~17% |

→ 1전투에서 평균 1턴 손실. 분노/절망보다 약하지만 라운드 행동 순서에 실질 영향.

### 6.4 투지 발동 후 사망 저항 변화

투지 발동 + 솔로 cap 0.95:

| 케이스 | 사망 저항 | 사망 확률 (적 critical 가정) |
|-------|---------|------------------------|
| 일반 의뢰 (cap 0.80, 투지 없음) | base 0.40 | 60% |
| 일반 의뢰 + 투지 가산 (cap 0.80) | clamp(0.40 + 0.20, 0.20, 0.80) = 0.60 | 40% |
| 솔로 의뢰 (cap 0.95, 투지 없음) | base 0.40 | 60% |
| **솔로 의뢰 + 투지 가산 (cap 0.95)** | **clamp(0.40 + 0.20, 0.20, 0.95) = 0.60** | **40%** (cap 미도달) |
| 솔로 의뢰 + 트레잇 + 투지 (cap 0.95) | clamp(0.70 + 0.20, 0.20, 0.95) = 0.90 | 10% |

→ 투지 +0.20 가산은 cap에 도달하기 전 구간에서만 실제 수치를 올린다. cap 도달 후에는 추가 효과 없음. 솔로 의뢰에서 base가 높은 용병(트레잇 보유 + 세력 보너스)이 투지 발동 시 cap 0.95 도달 가능. 정합.

### 6.5 4 감정 동시 발동 시나리오

페이즈 1 #3 §3.4.1 우선순위 (투지 > 분노 > 슬픔 > 절망) 정합 확인.

예: 동료 사망 + 본인 HP <30% + 파티 HP <25% 동시 발생
1. 투지 100% 자동 발동 (본인) → 본인은 이미 emotional 보유 → 후순위 차단
2. 분노 60% 평가 (생존 파티) → 본인 제외, 다른 mercenary에 적용 시도
3. 슬픔 평가 (해당 없음)
4. 절망 80% 평가 (다른 mercenary) → 적용

**결론**: 한 라운드 다중 트리거 시 우선순위 desc로 평가 → 같은 mercenary는 단일 emotional만 보유 (ignore). 다른 mercenary는 별도 평가.

## 8. 우선순위 정책 정합 (페이즈 1 #3 §3.4.1)

본 절은 같은 라운드에 여러 감정 트리거가 발생했을 때의 결정성을 보장한다. 우선순위는 감정 상태 간에만 적용하며, 일반 buff/debuff/dot/mez와의 적용 순서를 바꾸지 않는다.

### 8.1 우선순위 확정

페이즈 1 #3 §3.4.1 그대로:
1. **투지** (최우선) — 영웅적 반응, 간판/솔로/체인 주인공 보호
2. **분노** — 동료 사망 직후 즉각 반응
3. **슬픔** — 동료 중상 일시적 위축
4. **절망** (최후순위) — 전멸 임박 가장 약한 자에게

### 8.2 평가 알고리즘

```dart
EmotionalEffect? evaluateEmotionalForCombatant(
  Combatant combatant,
  List<EmotionalTrigger> roundTriggers,  // 이번 라운드 발생 사건
) {
  if (combatant.emotionalState != null) return null;  // 이미 보유 시 ignore

  // 우선순위 desc로 평가
  for (final emotion in [
    EmotionType.determination,
    EmotionType.rage,
    EmotionType.sorrow,
    EmotionType.despair,
  ]) {
    final trigger = roundTriggers.firstWhereOrNull(
      (t) => t.emotion == emotion && t.affects(combatant.id),
    );
    if (trigger == null) continue;
    final chance = computeActivationChance(emotion, combatant);
    if (rng.nextDouble() < chance) {
      return applyEmotion(emotion, combatant);
    }
  }
  return null;
}
```

같은 라운드에서 우선순위 desc 평가 → 첫 발동 emotional이 ignore로 후순위 차단. 명확한 결정성.

→ ±10%p는 1주 발동 빈도 +0.3~+0.8건. 영향 미미. **권장 확률 그대로 확정**.

## 11. data-generator 수치 가이드

본 절은 data-generator가 SQL 시드와 검증 규칙을 만들 때 사용할 입력을 요약한다. 구현 상수와 데이터 시드가 분리되는 값은 반드시 양쪽에 모두 반영한다.

본 리포트의 수치는 페이즈 3 #2 "감정 반응 상태 효과 시드 4행" SQL 시드 + 페이즈 4 #3 spec 정수 입력.

- **대상 타입**: 별도 타입 스펙 불요 (M8b `combat_status_effects` 패턴 재사용)
- **대상 테이블**: `combat_status_effects` 4행 INSERT + `kind` CHECK 확장 (`'emotional'` 추가)
- **수치 범위**:
  - `default_intensity`: -0.20~0.50 (분노 0.30·절망 -0.20·슬픔 0.50·투지 +0.20)
  - `default_duration_turns`: 2~4
  - `apply_method`: multiplicative / additive / none
  - `hook_target`: jsonb 리스트 (1~2 hook)
- **외래 키 제약**: kind ∈ {buff, debuff, mez, dot, **emotional**} (CHECK 확장)
- **balance 근거**: §3~§5 M8b 기존 default 대비 강화 비율 + §6 전투 흐름 시뮬레이션
- **특수 요구**:
  - `emotional_sorrow.apply_method='none'` — simulator effectId 분기 처리
  - `emotional_determination.hook_target=["death_resistance","evasion"]` — death_resistance hook은 M8b 신규
  - 4행 모두 `stack_policy='ignore'`
  - `emotional_despair.default_intensity=-0.20` — signed additive 값을 허용한다. 기존 debuff 상태 효과처럼 양수 저장 후 차감하는 방식이 아니다.
  - `emotional_*` 4행은 일반 상태 효과와 공존하지만, emotional끼리는 1명당 1개만 활성화한다.
- **검증**:
  - kind CHECK 위반 없음
  - jsonb hook_target 리스트 형식 정합
  - default_intensity가 음수도 허용 (numeric type)
  - `despairTriggered` / `determinationTriggeredMercIds` latch가 전투 단위 중복 발동을 막는지 확인
  - 투지 사망 저항이 `deathResistanceCaps[mercId] ?? 0.80`을 넘지 않는지 확인
