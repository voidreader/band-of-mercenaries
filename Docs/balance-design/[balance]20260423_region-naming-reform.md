# 리전 이름 개편 — 해안/늪 추가 및 분포 균등화

> 작성일: 2026-04-23
> 유형: 세계 구조 조정 (DB UPDATE 제안)
> 분석 대상: `regions.region_name` — coast/swamp 환경 태그 인프라 마련

---

## 배경

기존 분포 분석(`[balance]20260423_region-environment-tag-distribution.md`)에서 확인된 문제:
- `coast`/`swamp` 이름의 region이 **0개** — 환경 태그 보조 부여만으로는 게임 다양성 부족
- 사용자 방향: 신규 region 이름 추가 + 기존 region 일부 대체 → 세계 자체가 coast/swamp 지형을 내포

## 수학적 제약 분석

### 해안/늪 슬롯 확보 한계

신규 이름의 슬롯은 기존 이름에서 전환해야 총 199 유지.

| 전환 소스 | 가용 슬롯 | coast 확보 | swamp 확보 |
|---------|---------|----------|----------|
| 초원(40) 일부 → 해안 | 전환 수만큼 | +해안 수 | — |
| 숲(35) 일부 → 늪 | 전환 수만큼 | — | +늪 수 |
| 합산 가용 | 75 | — | — |

**목표 최소 요건:**
- coast(해안) ≥ 18 AND forest(숲) ≥ 25 → 해안 + 숲 ≥ 43. 초원(40)에서 해안 취득 시 초원 ≤ 22 → plains = 22+20 = 42 ✓
- swamp(늪) ≥ 11 AND forest(숲) ≥ 25 → 숲+늪 ≥ 36. **그런데 원래 숲 35개 = 36 미달 → 1 부족**

**해결**: 폐허(30) 1개를 숲으로 전환 → 숲 슬롯 +1 확보. 폐허=29로 ruins = 29개로 감소 (목표 내 유지).

## 권장 전환 계획

### 신규 region_name 2종 추가

| 신규 이름 | tier | 추가 수 | 주 태그 | 배경 설정 |
|---------|------|--------|-------|---------|
| **해안** | T1 | 18개 | `coast` | 대륙 가장자리 초원 지대가 해안으로 이어지는 지형 |
| **늪** | T2 | 11개 | `swamp` | 원시림 깊은 곳의 습한 밀림/습지 |

### 전환 대상 (Supabase UPDATE 대상 region_id)

#### 1. 초원 → 해안 (18개)

> 기준: 초원 region 중 ID가 높은 18개 — "세계 개척 후기에 발견된 초원 지역 = 대륙 끝자락 = 해안가"

```sql
UPDATE regions SET region_name = '해안' 
WHERE id IN (
  99, 110, 111, 118, 119, 120, 127, 129, 132, 138, 142, 161, 163, 165, 177, 185, 190, 193
);
```

초원 잔존: 40 - 18 = **22개** (IDs: 3, 6, 12, 14, 15, 27, 31, 33, 34, 39, 42, 43, 45, 64, 70, 71, 77, 85, 89, 95, 96, 97)

#### 2. 숲 → 늪 (11개)

> 기준: 숲 region 중 ID가 높은 11개 — "오래 방치된 밀림 심부 = 안개 습지로 변성"

```sql
UPDATE regions SET region_name = '늪' 
WHERE id IN (
  146, 147, 149, 151, 153, 164, 168, 175, 178, 194, 195
);
```

숲 잔존: 35 - 11 = **24개** (현재 단계. 아래 폐허 전환으로 +1)

#### 3. 폐허 → 숲 (1개)

> 기준: 폐허 region 중 가장 낮은 ID(id=5) — "가장 오래된 폐허 = 수백 년간 자연이 되찾아 밀림으로 회복"

```sql
UPDATE regions SET region_name = '숲' WHERE id = 5;
```

숲 최종: 24 + 1 = **25개** | 폐허 잔존: 30 - 1 = **29개**

## 구현 절차

### 1단계: DB region_name UPDATE (operation-bom 또는 Supabase SQL 에디터)

위 3개 SQL UPDATE 문을 순서대로 실행.

```sql
-- 실행 전 검증
SELECT region_name, COUNT(*) FROM regions GROUP BY region_name ORDER BY COUNT(*) DESC;
-- 예상 결과: 초원 40 / 숲 35 / 폐허 30 확인 후 실행

-- Step 1: 초원 18개 → 해안
UPDATE regions SET region_name = '해안' 
WHERE id IN (99,110,111,118,119,120,127,129,132,138,142,161,163,165,177,185,190,193);

-- Step 2: 숲 11개 → 늪
UPDATE regions SET region_name = '늪' 
WHERE id IN (146,147,149,151,153,164,168,175,178,194,195);

-- Step 3: 폐허 1개 → 숲
UPDATE regions SET region_name = '숲' WHERE id = 5;

-- 실행 후 검증
SELECT region_name, COUNT(*) FROM regions GROUP BY region_name ORDER BY COUNT(*) DESC;
-- 기대: 초원22 / 숲25 / 폐허29 / 산악25 / 전쟁터20 / 고대유적18 / 해안18 / 황무지15 / 늪11 / 마계경계10 / 심연6
```

### 2단계: data_versions 갱신

- `data_versions` 테이블에서 `regions` 버전 +1
- Flutter 앱 재동기화 시 region 데이터 전체 재다운로드

### 3단계: 환경 태그 data-generator 실행

`/data-generator region-environment-tag` 재호출 — 개편된 이름 기반으로 CSV 생성.

`types/region-environment-tag.md`의 분기 매핑 테이블에 `해안`, `늪` 추가 필요:

| region_name | 태그 |
|------------|------|
| 해안 | `["coast"]` |
