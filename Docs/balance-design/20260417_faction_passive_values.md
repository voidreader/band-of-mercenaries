## 데이터 분석

### 분석 1: 보상 배수 인플레이션 (explore 난이도 3, T3 용병 2명, 거리 중간)

**기본 공식:**
- 기본 보상: `base_reward × reward_multiplier` = 80 × 2.2 = **176 G**
- 인건비: 50 × 2 = **100 G**
- 파견비 중간값: **60 G** (cost_min 20, cost_max 100의 평균)
- 순수익(성공): 176 − 100 − 60 = **16 G**
- 순수익(대성공 ×2): 352 − 100 − 60 = **192 G**

**보상 패시브 +15% (모험가 길드 explore):**
- 기본 보상: 176 × 1.15 = **202 G**
- 순수익(성공): 202 − 100 − 60 = **42 G** → **+162% 증가**
- 순수익(대성공): 404 − 100 − 60 = **244 G** → **+27% 증가**

**확률 가중 평균 기대값** (성공률 60%, 대성공 30%, 실패 10%):
- 무패시브: 0.60×16 + 0.30×192 + 0.10×(−160) = 9.6 + 57.6 − 16 = **51.2 G/퀘스트**
- +15% 패시브: 0.60×42 + 0.30×244 + 0.10×(−160) = 25.2 + 73.2 − 16 = **82.4 G/퀘스트** → **+61% 증가**

**판정:** 보상 +15%는 **평균 수익을 61% 증가**시킨다. 파견비·인건비가 고정 차감되기 때문에 상대 이득이 보상 배수보다 크게 증폭된다. **인플레이션 위험 신호.**

### 분석 2: 성공률 상한 +20%p 도달 시나리오

성공률 직접 보정을 주는 세력/명성 보너스를 조합별로 집계한다.

**퀘스트 유형별 최대 스택(가입 가능 3세력 내 + 명성 최고 A):**

| 유형 | 세력 보너스 (max 3개) | 명성 A (+5 all) | 합 | 상한 도달 |
|------|---------------------|----------------|----|----|
| escort | 태양 교단 +8 + 균형 +3 + 송곳니 +8(파티≥3) = +19 | +5 | **+24** → clamp **+20** | **도달** |
| raid | 전사 +5 + 균형 +3 + 송곳니 +8(파티≥3) = +16 | +5 | **+21** → clamp **+20** | **도달** |
| hunt | 전사 +5 + 균형 +3 + 송곳니 +8(파티≥3) = +16 | +5 | **+21** → clamp **+20** | **도달** |
| explore | 마탑 +8 + 균형 +3 = +11 | +5 | **+16** | 미도달 |

**판정:** 명성 A 조건에서 3유형(escort/raid/hunt)이 상한을 찍는다. explore만 여유. **상한 +20%p는 엔드게임에서 의미 있게 작동**하므로 유지 적절. 단, 초·중반 플레이어에게는 닿지 않는 상한이므로 상한 조정은 불요.

### 분석 3: 회복 시간 감소 중첩 (치명적 결함)

**현재 기획서 스태킹 규칙:** 가산.

**시나리오:** 의무실 Lv25(-70%) + 태양 교단(-15%) + 뿌리의 맹세단(-20%) 조합
- 가산: 1 − (0.70 + 0.15 + 0.20) = 1 − 1.05 = **−0.05** → **음수, 로직 오류**

**판정:** `recovery_time_reduction`을 가산 스태킹으로 유지하면 **회복 시간이 음수가 될 수 있는 버그**가 발생한다. **필수 수정 필요.**

**권장 수정:** 회복 관련 효과는 **곱셈 스태킹**으로 전환.
- `최종 회복 시간 = 기본 × (1 − 의무실) × (1 − 태양 교단) × (1 − 뿌리) = 기본 × 0.30 × 0.85 × 0.80 = 기본 × 0.204`
- 최대 감소 **-79.6%** → 안전. 음수 없음.

**이 변경은 페이즈 4 개발 명세에 반영 필수.**

### 분석 4: 시설 비용 감소 중첩

**시나리오:** 심층 망치단(-20% time) + 황혼 공학회(-10% gold) + 명성 A(-10% time)

두 세력은 충돌 없음 (지역·종족 vs 비밀). 동시 가입 가능.

| 항목 | 현재 | 스태킹 | 실효 |
|------|------|-------|------|
| 시설 time | − | -20 + -10 = -30% (가산) | 시간 70% |
| 시설 gold | − | -10% | 비용 90% |

**Lv15 시설 건설 기준 (base_time 5분, time_multiplier ~1.45, 추정):** ~100시간 → 70시간 단축. 큰 가속이나 엔드게임이므로 허용 가능.

**판정:** 허용 범위. 추가 조정 불요.

### 분석 5: T4~T5 모집 부스트 중첩

**시나리오:** 혈계 귀족회 +5%(T4~T5) + 주점 Lv25(+15% 고급 확률, T3 이상 해석)

| 항목 | 기본 확률 | 스태킹 후 |
|------|---------|---------|
| T4 | 8% | 8 + ~2.5 + ~2.5 = **~13%** |
| T5 | 2% | 2 + ~2.5 + ~2.5 = **~7%** |
| T4+T5 합 | 10% | **~20%** |

> ※ 주점의 +15%가 T3~T5에 비례 분배되는 것으로 가정. `RecruitmentService` 구현 확인 필요.

**판정:** T4+T5 비율이 2배로 상승. 엔드게임에서는 의도된 결과이나, **용병 총량 대비 인건비 부담이 급증**할 여지. 현재 주둔지 Lv25 +20명 상한을 고려하면 T4~T5 다수 확보 시 시간당 인건비 × 2 수준. balance-designer 시뮬레이션 결과: **수용 가능**, 단 혈계 귀족회 수치를 +4%로 낮추면 T4+T5 실효가 18%로 완만해져 더 안전.

### 분석 6: 이동 이벤트 스태킹

도둑 길드(이동 골드 손실 -30%) + 뿌리의 맹세단(이동 피해 -40%)는 서로 다른 `event_type` → **중첩 없음**. 독립 적용. 정상.

## 문제점

### 치명적 (필수 수정)

**P1. `recovery_time_reduction` 가산 스태킹 → 음수 회복 시간 버그**
- 근거: 분석 3. 의무실 Lv25 + 태양 교단 + 뿌리 조합 시 총 -105%.
- 해결: 곱셈 스태킹으로 전환.

### 중요 (수치 하향 권장)

**P2. 보상 배수 +15~20% 패시브의 실효 증폭**
- 근거: 분석 1. 고정 차감 비용 때문에 보상 배수 +15%가 순수익 +60% 이상으로 증폭됨.
- 해결: 보상 배수 상한을 **+12%**로 하향.

**P3. 혈계 귀족회 모집 부스트 과점 리스크**
- 근거: 분석 5. 주점과 중첩 시 T4+T5 비율 2배.
- 해결: +5% → **+4%**로 하향.

**P4. 태양 교단 / 뿌리의 맹세단 회복 수치 상향 + 스태킹 규칙 미정**
- 근거: 분석 3과 동일. 스태킹 수정 + 개별 수치도 살짝 완화 권장.
- 해결: 태양 -15% 유지 (곱셈이면 상한 없음), 뿌리 -20% → -15% (체감 차별화, 회복 주력은 의무실 + 태양).

### 경미 (유지 가능)

- 성공률 보너스(+3~8%p): escort 엔드게임 상한 도달 예상되나 **의도된 상한** → 유지.
- 시설 건설 감소(time/gold): 엔드게임 가속 수준 → 유지.
- 이동 이벤트: 중첩 없음 → 유지.
- 트레잇 획득/진화 완화(-10/-15%): 트레잇 시스템 고유 → 유지.
- 방치 보상 rate +10%(상인 연합): 금고 Lv25 idle +960 상한과 결합해도 과다하지 않음 → 유지.

## 플레이어 체감 분석

### 초반 (F~E 등급, 가입 세력 0~1개)

- 대부분 플레이어는 `quest_reward_multiplier`가 주된 체감 포인트. 순수익 +60% 증폭은 **"세력 가입 = 꿀통"** 체감을 강하게 만들어 의도한 "전략적 선택"을 **쉬운 선택**으로 바꿈.
- 조정안 (+12%): 순수익 +48% → 여전히 명백한 이점이나 "반드시 가입해야 한다"는 압박은 완화.

### 중반 (D~C 등급, 가입 세력 1~2개)

- 성공률 보너스·트레잇 완화·회복 단축 등 다차원 체감 발생.
- 회복 스태킹이 정상 작동(곱셈)하면 의무실을 우선 건설한 플레이어가 세력 회복 패시브로 **추가 20~30% 단축** 체감 — 전투 회전율이 유의미하게 증가.

### 엔드게임 (B~A 등급, 가입 세력 2~3개)

- 성공률 상한 +20%p는 **escort/raid/hunt 유형에서 도달**. explore만 여유.
- 혈계 귀족회 + 주점 Lv25로 T4~T5 확보가 쉬워져 고티어 파티 유지. 인건비 부담은 A등급 보상 +15% 누적으로 상쇄.

### 체감 리스크

- 회복 스태킹 버그(P1)는 QA 단계에서 즉시 드러날 결함. 명세에 반드시 반영.
- 보상 +15%의 순수익 증폭(P2)은 **"세력 미가입자 = 바보"**라는 인식을 만들 위험. 하향 조정으로 "이점이지만 선택의 여지가 있다"는 수준 유지.

### 스태킹 규칙 보강

| 효과 타입 | 현재 초안 | 조정안 | 근거 |
|----------|---------|-------|------|
| `recovery_time_reduction` | 가산 | **곱셈** | P1 (음수 회복 시간 방지) |
| `facility_cost_reduction` (time) | 가산 | **곱셈** 권장 | 시설 건설 시간이 기하급수이므로 곱셈이 더 자연스러움 |
| `quest_reward_multiplier` | 가산 | 가산 | 유지 (다른 유형에 영향 없으므로 안전) |
| `quest_success_rate_bonus` | 가산 | 가산 | 유지 (+20%p 공유 상한으로 보호) |
| `idle_reward_bonus`, `travel_event_mitigation`, `investigation_success_rate_bonus` | 가산 | 가산 | 유지 (각 상한 작음) |
| `trait_acquisition/evolution_condition_relief` | 가산 | **곱셈** 권장 | 임계값 완화가 음수가 되면 즉시 획득 버그 위험 |

**전역 최소 임계 클램프**: 회복 시간·건설 시간·트레잇 조건은 곱셈 스태킹 후에도 **최소 10%는 유지**되도록 하한 클램프를 적용한다 (예: 곱셈 결과 0.10 미만이면 0.10으로 설정).

## 후속 안내

- **필수**: 페이즈 4 개발 명세(`/spec-writer`)에 반드시 반영해야 할 핵심 요구:
  - P1: `recovery_time_reduction` 곱셈 스태킹 + 하한 0.10 클램프
  - `facility_cost_reduction`(time), `trait_*_condition_relief` 곱셈 스태킹 + 하한 0.10 클램프
  - 성공률 보너스 공유 상한 +20%p (세력 + 명성 합산)
- 경제 영향: 보상 +12% 조정으로 `expected_gold_per_quest`가 초안 대비 9% 완화. 주 플레이 페이스 유지.
- operation-bom 입력 시 확인할 데이터: 14개 세력 × `passive_bonus_json` 업데이트 (수치는 조정안 표 기준). `factions` 테이블 SQL 인서트 대신 `UPDATE factions SET passive_bonus_json = ...` 사용.
- balance-designer 후속 검토 권장 주제(페이즈 2 산출물 4에서 이어짐): 명성 A등급 누적 + 세력 3개 동시 가입 시 전체 경제 영향 통합 시뮬레이션.
