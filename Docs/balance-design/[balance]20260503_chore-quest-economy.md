## 현재 상태

### 기존 시작 골드·모집비·갱신 주기 (코드)

`band_of_mercenaries/lib/core/constants/game_constants.dart`:

```
startingGold:           500
sectorCount:            10
baseQuestCount:         5      (활성 의뢰 슬롯 기본 5)
paidRecruitCost:        100    (유료 모집비)
freeRecruitCooldown:    2h     (무료 모집 쿨다운)
maxIdleRewardMinutes:   480
idleRewardPerMinute:    1
```

**주의**: 페이즈 1 #5 흐름은 **시작 골드 200G**를 가설로 사용. 본 문서가 200G 채택 vs 500G 유지 결정.

### 기존 난이도 1·2·3 일반 의뢰 데이터 (Supabase `difficulties` + `quest_types`)

```
difficulties Lv1: enemy_power 10, reward_multiplier 1.0, dispatch 5~30G,  injury 10%, death 5%, success_penalty 0
difficulties Lv2: enemy_power 20, reward_multiplier 1.5, dispatch 10~60G, injury 20%, death 10%, success_penalty 0.1
difficulties Lv3: enemy_power 35, reward_multiplier 2.2, dispatch 20~100G, injury 30%, death 15%, success_penalty 0.2

quest_types (base_reward / base_duration / risk_factor):
labor:    50  / 60s  / 0.05  ← 허드렛일 핵심 type
explore:  80  / 70s  / 0.20
escort:   90  / 75s  / 0.25
raid:    100  / 60s  / 0.30
hunt:    120  / 80s  / 0.50
survey:    0  / 180s / 0.10  (지역조사 — 의뢰 풀 미포함)

mercenary_wages: T1=10G / T2=25G / T3=50G / T4=100G / T5=200G
```

### 기존 quest_pools 난이도 1·2 분포

```
난이도 1: hunt 6 + explore 4 + escort 4 + raid 3 = 17건  (labor 0건 ⚠)
난이도 2: hunt 10 + escort 8 + raid 7 + explore 6 + labor 2 = 33건
```

**관찰**: 난이도 1 풀에 **labor type이 0건**. 더스트빌 시작 거점용 허드렛일 6건 신규 추가가 필요하다 (페이즈 4 #3 인라인 처리).

### 페이즈 2 #1 결정사항 (입력)

- 일반 의뢰 신뢰도 보상: 난이도 1 = **2점** / 난이도 2 = **3점** / 난이도 3 = **5점**
- 단계 진입 일회성 보상: 2단계 +100G+50XP / 3단계 +200G+100XP / 4단계 +500G+200XP+100명성
- 결과 보정: 대성공 ×1.0 / 성공 ×1.0 / 실패·대실패 ×0 (신뢰도 보상)

### 페이즈 2 #2 결정사항 (입력)

- 약초상 비용: 1단계 75G / 2단계 50G / 3단계 45G / 4단계 40G
- 약초상 쿨다운: 45m / 30m / 15m / 10m
- 채집 의뢰 보상 배수: 2단계 ×1.0 / 3단계 ×1.1 / 4단계 ×1.2 (골드만 적용, 신뢰도는 ×1.0 고정)

### XP 공식 (코드 — `core/domain/experience_service.dart`)

```
calculateXpGain = difficulty × 20 × resultMultiplier × (1 + facilityBonus + passiveXpBonus)
resultMultiplier: 대성공 2.0 / 성공 1.0 / 실패 0.5 / 대실패 0.0
levelThresholds: [0, 100, 350, 850, 1850]
maxLevel: 5
```

**난이도 1 평균 XP** (성공률 ~80% 가정, 대성공 30% + 성공 60% + 실패 10%):
- 0.3 × 40 + 0.6 × 20 + 0.1 × 10 = 12 + 12 + 1 = **25 XP/건** ✓ 페이즈 1 #5 권장값 정합

## 데이터 분석

### 1. 난이도 1 일반 의뢰 순수익 곡선

**공식**: `순수익 = base_reward × reward_multiplier(1.0) - dispatch_cost(5G, T1 1명) - wage(10G, T1 1명)`

| quest_type | base_reward | dispatch | wage | **순수익** | 소요시간 | 시간당 수익 |
|-----------|-------------|---------|------|----------|---------|------------|
| labor | 50 | 5 | 10 | **35G** | 60s | 35G/min |
| explore | 80 | 5 | 10 | **65G** | 70s | 56G/min |
| escort | 90 | 5 | 10 | **75G** | 75s | 60G/min |
| raid | 100 | 5 | 10 | **85G** | 60s | 85G/min |
| hunt | 120 | 5 | 10 | **105G** | 80s | 79G/min |

**관찰**:
- labor는 골드 효율 최저(35G), 단 risk_factor 0.05로 부상·사망 위험 가장 낮음
- raid는 시간당 효율 최고(85G/min), 단 risk_factor 0.30으로 부상 위험
- hunt는 골드 절대값 최고(105G), 단 80s로 가장 길고 risk_factor 0.50

**T1 2명 파견 시 순수익**:
- labor 2명: 50 - 5 - 20 = **25G/건** (인건비 부담 큼)
- explore 2명: 80 - 5 - 20 = **55G/건**
- escort 2명: 90 - 5 - 20 = **65G/건**
- raid 2명: 100 - 5 - 20 = **75G/건**
- hunt 2명: 120 - 5 - 20 = **95G/건**

**판정**: 난이도 1 일반 의뢰는 **T1 1명 단독 파견이 효율적**. 다중 파견은 raid·hunt 일부에서만 의미. 첫 2시간 학습 곡선 정합.

### 3. XP 보상 곡선 검증

**난이도 1 평균 XP**: 25 XP/건 (위 계산).

**첫 2시간 누적 XP** (이상 시나리오, 페이즈 1 #5):
- step 1·2 (난이도 1): 25 × 2 = 50 XP
- step 3·4 (난이도 2 평균): 50 × 2 = 100 XP
- step 5·6 (난이도 3 평균): 75 × 2 = 150 XP
- 일반 의뢰 ~10건 (난이도 1·2 평균 35 XP): 350 XP
- **합계 의뢰 XP ~650 XP** (전체 파티 분배)

**단계 진입 일회성 XP** (페이즈 2 #1):
- 2단계 +50 / 3단계 +100 / 4단계 +200 = **350 XP**

**총합** ~1,000 XP / 첫 2시간. 보유 용병 ~4명 분배 시 평균 250 XP/명.

**개별 용병 레벨업 곡선**:
- 첫 용병 (0:30 모집) — 첫 2시간 step 1·3·5 + 일반 의뢰 ~5건 처리 시 ~250 XP
- Lv1→Lv2 임계값 100 XP → 약 30분 시점 도달
- Lv2→Lv3 임계값 350 XP → 첫 2시간 외부

**검증**: 페이즈 1 #5 권장값 25 XP/건 + 단계 진입 50/100/200 XP는 **첫 용병이 첫 2시간에 Lv1→Lv2 도달 + 두 번째 용병 Lv1 도달 직전**의 합리적 곡선 형성 ✓

**조정 제안**: 없음. 현재 baseXp 20·levelThresholds [0,100,350,850,1850] 그대로 유지.

### 4. 갱신 주기 효과 분석 (60분 vs 30분)

**기존 시스템**: 1시간 게임 시간 = 60분마다 미수락 의뢰 자동 교체.

**시뮬레이션 — 60분 유지 (현재)**:

| 시점 | 활성 슬롯 상태 | 학습감 |
|------|--------------|--------|
| 0:00 | 시작 풀 6개 (step 1 + 5건) | "초기 풀 학습" |
| 30:00 | step 1·2 완료 → 슬롯 4개 + 신규 갱신 없음 → "채우기" 버튼으로 5개 채움 | 정적 |
| 60:00 | **첫 갱신** — 미수락 1~3건 자동 교체 + 슬롯 4~5개 유지 | 신규 풀 등장 → 학습감 +1 |
| 120:00 | **두 번째 갱신** + step 5·6 완료 | 외부 진입 직전 |

**시뮬레이션 — 30분으로 단축**:

| 시점 | 활성 슬롯 상태 | 학습감 |
|------|--------------|--------|
| 0:00 | 시작 풀 6개 | 정적 |
| 30:00 | **첫 갱신** — 미수락 자동 교체 + 신뢰도 2단계 진입과 동시 | 학습감 +2 (단계 진입 + 풀 갱신 동시) |
| 60:00 | **두 번째 갱신** | 익숙해진 시점 +1 |
| 90:00 | **세 번째 갱신** | 누적 +2 |
| 120:00 | **네 번째 갱신** | 외부 진입 직전 |

**평가**:

| 항목 | 60분 | 30분 |
|------|------|------|
| 학습 곡선 | ✓ 정적 풀로 학습 부담 적음 | ⚠ 너무 빠르면 풀 변화에 적응 부담 |
| 신뢰도 단계 진입과 동시성 | ❌ 60분 갱신은 3단계 진입 직후 (보너스 약함) | ✅ 30분 갱신은 2단계 진입과 동시 (보너스 강함) |
| 의뢰 누적 부담 | 활성 슬롯 5개 안정 | 활성 슬롯 5~7개 빠르게 채움 |
| 채집 의뢰 등장 시점 | 60분에 처음 노출 | 30분에 즉시 노출 (2단계 진입과 동시) |

**권장**: M4 시점 **60분 유지**. 페이즈 2 #2의 채집 의뢰 등장이 30분 시점 풀 갱신과 정합되지 않더라도, 단계 진입 시 즉시 채집 풀이 노출되는 것은 다음 60분 갱신 또는 "채우기" 버튼 사용으로 충분.

**대안**: 페이즈 4 #3에서 "단계 진입 시 신규 풀 강제 노출" 트리거를 추가하면 30분 단축 없이도 학습감 확보 가능. 본 문서는 60분 유지 + 단계 진입 트리거 권장.

## 첫 30분 골드 흐름 시뮬레이션

### 시작 골드 결정: 200G vs 500G

**현재 코드**: `GameConstants.startingGold = 500`.

**페이즈 1 #5 가설**: 200G로 의무실 Lv1 100G 부담 + step 1 보상 후 잔고 회복 가정.

**시뮬레이션 비교**:

| 항목 | 시작 200G | 시작 500G |
|------|----------|----------|
| 0:00 잔고 | 200G | 500G |
| 30:00 잔고 (이상) | ~340G | ~640G |
| 첫 시설 건설 부담감 | 의도된 부담 (학습 모멘트) | 부담 없음 |
| 두 번째 모집 (100G) 결정 | 신중 (잔고 부담) | 즉시 가능 |
| 의무실 + 약초상 동시 사용 | 빠듯 (잔고 ~40G 직후) | 여유 |

**판정**: 페이즈 1 #5 흐름의 "30분 시점 시설 건설 시작"이 의도된 학습 모멘트라면, 200G가 학습 곡선 안정. 500G는 첫 시설 건설이 "당연한 기본값"으로 인식되어 학습 모멘트 약화.

**조정 권고**: `GameConstants.startingGold` **500G → 200G 하향**. 페이즈 4 #1에서 마이그레이션 명세 직접 입력.

### 첫 30분 골드 흐름 — 이상 시나리오 (시작 200G)

**전제**:
- 첫 모집은 무료 (`freeRecruitCooldown 2h`, 게임 시작 시 0)
- 두 번째 모집은 paidRecruitCost 100G
- T1 용병 1명으로 step 1·2 + 일반 의뢰 1건 처리
- step 1 = explore 난이도 1 (보상 80G), step 2 = hunt 난이도 1 (보상 120G)
- 일반 의뢰 1 = labor 난이도 1 (보상 50G)

| 시간 | 액션 | 골드 변화 | 잔고 | 비고 |
|------|------|----------|------|------|
| 0:00 | 게임 시작 | +200 | 200 | startingGold 200G |
| 0:30 | 첫 무료 모집 → T1 1명 | 0 | 200 | freeRecruitCooldown 2h 시작 |
| 2:00 | step 1 파견 (explore Lv1, T1 1명) | -5 | 195 | dispatch 5G |

## 문제점

### 문제점 1: 현재 quest_pools에 난이도 1 labor 0건

**근거**: Supabase 쿼리 결과 — 난이도 1 풀은 hunt 6 + explore 4 + escort 4 + raid 3 = 17건. labor 0건. 페이즈 1 #5의 더스트빌 허드렛일 컨셉 (짐 나르기·야간 순찰·약초 채집·실종자 수색·도적 흔적 조사) 미존재.

**해결**: 본 문서의 허드렛일 풀 10건 (labor 6 + escort 1 + explore 2 + hunt 1) 신규 추가. 페이즈 4 #3 quest_pools 컬럼 확장 명세에 인라인 처리.

### 문제점 2: 시작 골드 500G는 학습 곡선 약화

**근거**: 페이즈 1 #5 가설 200G로 첫 시설 건설 + 약초상 사용 + 모집 결정이 "의식적 학습 모멘트"로 설계. 현재 500G는 자유도 너무 높음.

**해결**: `GameConstants.startingGold` **500 → 200 하향**. 페이즈 4 #1 마이그레이션 명세 직접 입력.

### 문제점 3: 시작 의뢰 풀 6개 권장 (난이도 1 ×4 + 난이도 2 ×2)는 다소 부담

**근거**: prompt 권장 "난이도 1 ×4 + 난이도 2 ×2"는 첫 30분 안에 난이도 2 의뢰 2건 노출. 첫 30분 학습 곡선에서 난이도 2는 부담 (T1 1명으로 도전 어려움).

**해결**: **난이도 1 ×5 + 난이도 2 ×1**로 조정. 난이도 2 의뢰 1건은 step 3 사전 안내 효과 (도전 가능 시그널). 페이즈 4 #1 명세 직접 입력.

### 문제점 4: 일반 의뢰 갱신 주기 60분 vs 30분 단축 검토

**근거**: 페이즈 1 #5에서 첫 갱신 60분 시점은 신뢰도 3단계 진입 직후 — 보너스 학습감 약함. 30분 단축 시 2단계 진입과 동시 발생 — 학습감 강화.

**해결**: M4 시점 **60분 유지**. 단, 페이즈 4 #3에서 "**단계 진입 시 신규 풀 강제 노출**" 트리거 추가 권장 (60분 갱신 + 단계 진입 트리거 결합).

**합산**: 80 + 120 + 200 + 185 + 270 + 500 = **1,355G**.

**페이즈 2 #1 4단계 진입 보상 합산** (100 + 200 + 500 = 800G) + **step 보상 1,355G** + **일반 의뢰 ~10건 평균 50G = 500G** = **2,655G** (총 첫 2시간 골드 유입).

**비용 합산** (시설 4개 × 평균 100~200G + 모집 5명 × 100G + 파견·인건비 ~600G + 약초상 ~140G):
- 시설 4개: 의무실 100 + 주둔지 100 + 정보망 150 + 훈련소 50 = **400G**
- 모집 4회 (1회 무료 + 3회 유료): 300G
- step 1~6 파견·인건비: ~150G + ~120G = 270G
- 일반 의뢰 ~10건 파견·인건비: ~50G + ~100G = 150G
- 약초상 2~3회: ~140G
- **합계**: ~1,260G

**100분 시점 잔고 추정**: 200 (시작) + 2,655 - 1,260 = **~1,595G**. 외부 진입 + 시설 5번째 큐 + 추가 모집 충분.

**검증**: 페이즈 2 #1의 100분 시점 잔고 추정 (~260G) 대비 본 문서 추정 (~1,595G)은 큰 차이 — 페이즈 2 #1 추정의 시설 4개 비용 가설 (300G/시설 = 1,200G)이 과대 추정. **실제 시설 4개 비용은 400G** (의무실·주둔지·정보망·훈련소 Lv1 합산). 본 문서의 추정이 더 정확.

**페이즈 2 #1 보정 권고**: 페이즈 2 #1 시뮬레이션 3 "100분 시점 잔고 ~260G" 추정은 시설 비용 과대 가정으로 도출됨. 실제 잔고는 ~1,595G로 단계 진입 보상 4단계 +500G가 외부 진입 동기 강화에 충분. 본 문서가 보정값 제공.

## data-generator 수치 가이드

본 문서로부터 data-generator를 호출하여 더스트빌 허드렛일 10건을 벌크 생성할 수 있다. 단, 10건은 인라인 처리도 가능한 소량이므로 **페이즈 4 #3 명세에 SQL INSERT 직접 작성 권장**. data-generator 호출은 선택적.

- **대상 타입**: 신규 타입 정의 필요 (`types/dustvile-chore.md` 또는 기존 `quest-pool` 변형)
- **대상 테이블**: `quest_pools`
- **수치 범위**:
  - difficulty: **1** (고정)
  - type_id: labor (6건) / escort (1건) / explore (2건) / hunt (1건)
  - min_region_diff: **0** (고정)
  - max_region_diff: **0** (고정)
  - min_trust_level: 1~2 (채집 의뢰 1건만 2)
- **외래 키 제약**: type_id ∈ {labor, escort, explore, hunt}, min_trust_level 1~4
- **balance 근거**: 본 문서 1·2절 시뮬레이션 — labor 35G·escort 75G·explore 65G·hunt 105G 순수익 곡선 + 단계별 풀 노출 정합
