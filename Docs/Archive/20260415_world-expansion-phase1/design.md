# 세계 확장 시스템 — 자율성 강화 컨텐츠 기획서

> 작성일: 2026-04-13
> 유형: 신규 컨텐츠
> 상태: 기획 확정, 구현 대기

---

## 개요

"파견 → 결과 보고" 단일 루프를 넘어서, 세계와 상호작용하는 다양한 경로를 제공한다. 플레이어가 지역을 깊이 파고들수록 숨겨진 컨텐츠가 열리고, 세계에 영향을 미치는 주체가 되는 경험을 목표로 한다.

**프로듀서 비전**: "이런 컨텐츠도 있어? 이런 것도 가능해?"라는 유저 반응

**레퍼런스 세계관**:
- 특성 쌓는 김전사 (산호초) — 트레잇 축적
- 메모라이즈 (로유진) — 숨겨진 층, 특수 직업
- 게임 속 바바리안으로 살아남기 (정윤강) — 희귀 정수, 특수 아이템, 엘리트 몬스터
- 용마검전 (김재한) — 세력, 진영

---

## 레퍼런스 분석

| 게임/작품 | 차용 메커니즘 | 적용 방식 |
|----------|-------------|----------|
| 바바리안 — 정수 흡수 | 엘리트 처치 → 정수 드랍 → 영구 스탯 강화 | 소모형 아이템(정수) 시스템 |
| 바바리안 — 넘버드 아이템 | 희귀 장비 드랍 | 엘리트/연계 퀘스트의 장비 보상 |
| 메모라이즈 — 히든 스테이지 | 조건 충족 시 숨겨진 영역 발견 | 지역 지식 → 숨겨진 섹터/연계 퀘스트 해금 |
| 용마검전 — 세력 관계 | 다수 세력의 공존과 경쟁 | 세력 발견 → 가입 → 기여 로드맵 |
| Melvor Idle — 마스터리 | 반복 행동이 전문성으로 축적 | 지역 지식 시스템 (조사 반복 → 지식 축적 → 해금) |
| Darkest Dungeon — 탐험 깊이 | 던전 탐색 깊이에 따른 발견 | 지식 임계값별 발견 이벤트 트리거 |

## 상세 설계

### 1. 지역 상태 시스템 (전체의 뿌리)

현재 199개 리전은 "이동 후 퀘스트 뽑기" 용도로만 쓰인다. 유저별 지역 상태를 추가하여, 파고들수록 보상이 나오는 구조로 전환한다.

#### 지역 상태 모델

```
RegionState {
  regionId: String
  knowledge: int                       // 지역 지식 (0~100)
  triggeredDiscoveries: List<String>   // 발동된 discovery ID 목록
  eliteDefeated: Map<String, int>      // 처치한 엘리트 {ID: 횟수}
  sectorChanges: Map<int, String>      // 변형된 섹터 {섹터번호: 타입}
}
```

#### 발견 이벤트 — 데이터 드리븐 구조

지역마다 기획자가 발견 이벤트를 조합 배치한다. 모든 지역에 모든 유형이 있을 필요는 없다.

**region_discoveries 테이블 (Supabase 신규)**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | String (PK) | 고유 ID |
| region_id | String (FK) | 리전 ID |
| knowledge_threshold | int | 이 지식 수치에 도달하면 발동 |
| discovery_type | String | `info` / `elite` / `hidden_quest` / `faction_clue` / `transform` |
| discovery_data | JSONB | 타입별 상세 데이터 |
| description | String | 발견 설명 텍스트 |

**배치 예시 — 리전별 조합이 다르다:**

철광 협곡 (Tier 2) — 엘리트 + 변형, 세력 없음:

| knowledge | type | description |
|-----------|------|-------------|
| 30 | info | "협곡 깊숙한 곳에 폐광이 있다는 소문을 확인했다" |
| 60 | elite | "폐광의 주인, 강철 골렘이 목격되었다" |
| 90 | transform | "폐광 입구가 정리되어 접근 가능해졌다" |

은빛 평원 (Tier 3) — 세력 단서 특화, 엘리트 없음:

| knowledge | type | description |
|-----------|------|-------------|
| 20 | info | "이 평원에 정기적으로 드나드는 집단이 있다" |
| 50 | faction_clue | "은빛 교단의 전초기지 흔적을 발견했다" |
| 80 | hidden_quest | "교단이 남긴 봉인된 문서를 해독할 수 있게 되었다" |

안개 숲 (Tier 4) — 풀 조합:

| knowledge | type | description |
|-----------|------|-------------|
| 20 | info | "숲의 지형 구조를 파악했다" |
| 40 | elite | "안개 속에서 울부짖는 존재가 있다" |
| 60 | faction_clue | "야만족 연합의 제단을 발견했다" |
| 80 | hidden_quest | "제단의 비밀 의식에 대한 단서를 얻었다" |
| 100 | transform | "안개가 걷힌 구역에 고대 마을 터가 드러났다" |

**기획자의 역할:**
- 199개 리전 중 모든 리전에 발견 이벤트가 있을 필요는 없음
- Tier 1 리전은 info만 1~2개, Tier 5 리전은 풀 조합 — 티어가 높을수록 풍성하게
- 발견이 없는 리전은 기존처럼 "퀘스트만 돌리는 곳"으로 유지
- operation-bom 웹앱에서 지역 단위로 발견 이벤트를 CRUD

#### 저장 방식

Hive에 `regionStates` 박스 신설. 유저가 방문/조사한 지역만 엔트리 생성.

### 2. 지역 조사 메커니즘

파견과 독립된 별도 행동 슬롯으로, 용병을 지역 조사에 배치하는 시스템.

#### 기본 흐름

```
현재 위치의 리전 선택 → "지역 조사" 시작 → 용병 1명 배치
→ 시간 경과 (방치형) → 지식 포인트 획득
→ 임계값 도달 시 → 해당 지역의 discovery 이벤트 트리거
```

#### 핵심 규칙

| 항목 | 설계 |
|------|------|
| 배치 인원 | 1명 (조사는 은밀한 단독 행동) |
| 배치 제한 | 조사 중인 용병은 퀘스트 파견 불가 (역도 마찬가지) |
| 동시 조사 | 기본 1슬롯. 향후 시설(정보망)로 슬롯 확장 가능 |
| 조사 위치 | 용병단이 현재 위치한 리전만 조사 가능 |
| 이동 시 | 용병단이 이동하면 진행 중인 조사는 중단 (지식은 보존) |
| 조사 시간 | 티어별 1회 소요 시간 차등 |
| 반복 | 지식 100 도달까지 반복 배치 가능 |

#### 지식 획득량

| 리전 티어 | 1회 조사 시간 | 획득 지식 | 100 도달까지 |
|----------|-------------|----------|-------------|
| Tier 1 | 5분 | +10 | 약 50분 |
| Tier 2 | 8분 | +8 | 약 100분 |
| Tier 3 | 10분 | +6 | 약 170분 |
| Tier 4 | 15분 | +5 | 약 300분 |
| Tier 5 | 20분 | +4 | 약 500분 |

#### 조사 결과

조사 완료 시:
1. 지식 포인트 추가
2. 임계값 도달 여부 체크 → `region_discoveries`에서 매칭되는 이벤트 트리거
3. 트리거된 이벤트는 팝업 알림으로 전달
4. 활동 로그에 기록

#### 조사 실패/리스크

- 기본 성공률 85% (실패 시 지식 획득 없음, 시간만 소모)
- 실패 시 일정 확률로 용병 부상 (위험한 지역일수록 높음)
- 용병의 스탯(Speed, DEF 등)이 성공률에 소폭 영향

#### UI 위치

홈(야영지) 화면에 "지역 조사" 섹션 추가:

```
홈 화면
├── 현재 위치 / 골드
├── 진행 중 퀘스트 요약
├── [지역 조사] 조사 중인 용병, 잔여 시간, 지식 진행도 바
├── 활동 로그
```

### 3. 아이템 시스템 인프라

엘리트 몬스터, 숨겨진 퀘스트의 보상 뼈대.

#### 아이템 분류 체계

```
아이템
├── 장비형 (Equipment)
│   ├── 개인 장비 (Personal) — 용병 1명에게 장착
│   │   ├── 무기 (weapon)
│   │   ├── 방어구 (armor)
│   │   └── 장신구 (accessory)
│   └── 용병단 장비 (Band) — 전체 용병단에 효과 적용
│       ├── 군기 (banner)
│       └── 유물 (relic)
│
└── 소모형 (Consumable)
    ├── 정수 (essence) — 용병에게 사용, 영구 스탯 강화
    └── 일반 소모품 (supply) — 즉시 효과 (회복, 버프 등)
```

#### items 테이블 (Supabase 신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | String (PK) | 아이템 ID |
| name | String | 이름 |
| category | String | `personal` / `band` / `consumable` |
| slot | String? | 장비 슬롯 (`weapon`/`armor`/`accessory`/`banner`/`relic`/null) |
| tier | int | 1~5 |
| effect_json | JSONB | 효과 데이터 (예: `{"atk": 5, "def": 2}`) |
| description | String | 설명 텍스트 |
| max_stack | int | 최대 중첩 수 (장비: 1, 소모품: 99) |

#### 장비 슬롯

개인 장비 — 용병 1명당:

| 슬롯 | 효과 대상 |
|------|----------|
| weapon | ATK 위주 |
| armor | DEF, HP 위주 |
| accessory | 다양한 보정 |

용병단 장비 — 전체 효과:

| 슬롯 | 예시 | 효과 |
|------|------|------|
| banner | "철혈의 깃발" | 전체 ATK +3% |
| relic | "고대의 나침반" | 이동 시간 -10% |

#### 인벤토리 (Hive)

```
InventoryItem {
  itemId: String
  quantity: int
  equippedTo: String?  // 장착 중인 용병 ID (null이면 미장착)
}
```

인벤토리 용량 제한은 초기에 넣지 않고, 아이템이 충분히 쌓이는 단계에서 주둔지 시설과 연계.

#### 정수 시스템

엘리트 몬스터의 핵심 보상. '게임 속 바바리안으로 살아남기'의 정수 흡수 참고.

| 정수 종류 | 효과 | 드랍 출처 |
|----------|------|----------|
| 힘의 정수 | ATK +1 (영구) | 공격형 엘리트 |
| 수호의 정수 | DEF +1 (영구) | 방어형 엘리트 |
| 생명의 정수 | HP +3 (영구) | 대형 엘리트 |
| 상급 정수 | 스탯 +3 (영구) | Tier 4~5 엘리트 |

정수는 용병에게 "사용"하여 소멸. 향후 스탯 세분화(근력/내구/민첩/마력/체력) 시 정수 종류도 확장.

#### 아이템 획득 경로

| 경로 | 아이템 유형 | Phase |
|------|-----------|-------|
| 엘리트 몬스터 토벌 | 정수, 장비 (확률 드랍) | 3 |
| 숨겨진 연계 퀘스트 완료 | 장비, 용병단 장비 (확정 보상) | 4 |
| 향후: 세력 상점 | 세력 전용 장비/소모품 | 로드맵 이후 |
| 향후: 일반 퀘스트 보상 확장 | 소모품 (낮은 확률) | 로드맵 이후 |

### 4. 엘리트 몬스터 시스템

지역 지식 임계값 도달 시 출현하는 특수 적. 일반 퀘스트와 차별화된 긴장감과 희귀 보상 제공.

#### 출현 규칙

```
지식 임계값 도달 (최초 1회)
  → "이 지역에 엘리트가 존재한다"는 사실이 영구 해금
  → 이후 해당 리전의 퀘스트 갱신 때마다 확률 판정
  → 출현/미출현은 매 갱신마다 독립적으로 반복
```

| 상태 | 동작 |
|------|------|
| 지식 < 임계값 | 엘리트 퀘스트 절대 출현 안 함 |
| 지식 ≥ 임계값 (해금 후) | 매 퀘스트 갱신 시 확률로 출현 (처치 여부와 무관) |
| 엘리트 퀘스트 진행 중 | 동일 엘리트의 중복 출현 없음 |

한 번 해금되면 **반복 파밍이 가능**하다. "이 지역에 머물며 골렘을 반복 사냥해서 정수를 모은다"는 플레이가 성립.

#### 출현 확률

| 리전 티어 | 엘리트 퀘스트 출현 확률 |
|----------|---------------------|
| Tier 2 | 15% |
| Tier 3 | 12% |
| Tier 4 | 10% |
| Tier 5 | 8% |

#### elite_monsters 테이블 (Supabase 신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | String (PK) | 엘리트 ID |
| region_id | String (FK) | 출현 지역 |
| name | String | 이름 |
| tier | int | 몬스터 등급 |
| combat_power | int | 전투력 (일반 최대 80보다 높음) |
| description | String | 설명 |
| lore_text | String | 처치 시 서사 텍스트 |

#### 일반 퀘스트와의 차이

| 항목 | 일반 퀘스트 | 엘리트 토벌 |
|------|-----------|------------|
| 전투력 | 10~80 | 100~200 (티어별) |
| 소요 시간 | 난이도 비례 | 1.5~2배 |
| 파견 인원 | 자유 | 최소 2명 권장 |
| 결과 | 골드/XP/명성 | 골드/XP/명성 + 아이템 드랍 판정 |
| 실패 리스크 | 부상/사망 | 부상 확률 상승, 대실패 시 사망률 높음 |
| 반복 | 무한 갱신 | 처치 후에도 확률로 재출현 |

#### 드랍 시스템

elite_loot_tables 테이블 (Supabase 신규):

| 컬럼 | 설명 |
|------|------|
| elite_id | FK → elite_monsters |
| item_id | FK → items |
| drop_rate | 드랍 확률 (0.0~1.0) |

처치 시 각 항목별 독립 확률 판정. 0개~복수개 아이템 획득 가능. 대성공 시 드랍률 +15% 보너스.

| 아이템 유형 | 드랍률 가이드라인 |
|-----------|----------------|
| 정수 (일반) | 40~60% |
| 정수 (상급) | 10~20% (Tier 4~5만) |
| 개인 장비 | 15~25% |
| 용병단 장비 | 3~8% |

#### 검증용 최소 데이터 (3개)

| 엘리트 | 리전 | 티어 | 전투력 | 핵심 드랍 |
|--------|------|------|--------|----------|
| 강철 골렘 | Tier 2 리전 | 2 | 100 | 힘의 정수, 철제 장검 |
| 안개 포식자 | Tier 3 리전 | 3 | 140 | 수호의 정수, 안개빛 갑옷 |
| 심연의 감시자 | Tier 5 리전 | 5 | 200 | 상급 정수, 용병단 유물 |

### 5. 숨겨진 연계 퀘스트

지역 지식 임계값 도달 시 발동되는 다단계 퀘스트 체인. 엘리트가 반복 파밍 컨텐츠라면, 연계 퀘스트는 일회성 서사 컨텐츠.

#### 구조

```
[발견] 지식 임계값 도달 → 1단계 퀘스트 자동 생성
  → 1단계 완료 → 2단계 퀘스트 생성
  → ... → 최종 단계 완료 → 확정 보상 (장비/용병단 장비)
```

#### chain_quests 테이블 (Supabase 신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | String (PK) | 단계별 고유 ID |
| chain_id | String | 체인 그룹 ID |
| region_id | String | 발견 지역 |
| step | int | 단계 (1, 2, 3...) |
| target_region_id | String? | 수행 지역 (null이면 발견 지역) |
| name | String | 퀘스트 이름 |
| description | String | 서사 텍스트 |
| combat_power | int | 적 전투력 (전투 없는 단계는 0) |
| duration_minutes | int | 소요 시간 |
| reward_gold | int | 골드 보상 |
| reward_items | JSONB | 단계별 아이템 보상 |
| final_reward | JSONB? | 최종 단계만, 확정 보상 |
| next_step_delay | int | 다음 단계까지 대기 시간(분) |

#### 일반 퀘스트와의 차이

| 항목 | 일반 퀘스트 | 연계 퀘스트 |
|------|-----------|------------|
| 생성 | 자동 갱신 | 지식 임계값 도달 시 1회 |
| 소멸 | 1시간 후 교체 | 완료할 때까지 유지 |
| 단계 | 단일 | 2~5단계 체인 |
| 실패 시 | 보상 없음 + 끝 | 해당 단계를 재도전 가능 |
| 보상 | 골드/XP/명성 | 단계별 골드 + 최종 확정 아이템 |
| 반복 | 무한 | 1회 완료 시 영구 종료 |
| 서사 | 없음 | 단계마다 이야기 진행 |

#### 지역 이동 포함

각 단계는 서로 다른 지역에서 수행될 수 있다. `target_region_id`로 지정.

서사 흐름 예시 — "교단의 봉인 문서" (3단계):

| 단계 | 수행 지역 | 내용 | 전투력 | 보상 |
|------|----------|------|--------|------|
| 1 | Region 016 (발견 지역) | "문서를 해독하니 남쪽 산맥의 제단 위치가 적혀 있다" | 0 | 100G |
| 2 | Region 042 (이동 필요) | "산맥의 제단에 도착했다. 경비병을 처리해야 한다" | 60 | 200G + 수호의 정수(30%) |
| 3 | Region 042 | "제단의 봉인을 풀자 고대의 유물이 모습을 드러냈다" | 80 | 확정: 고대의 나침반 (용병단 장비) |

현재 위치와 다른 지역이면 "이동 필요" 안내 표시. 해당 지역에 도착해야 파견 가능.

#### UI 표시

파견 화면에서 구분 표시:

```
파견 화면
├── [연계] 🔗 봉인 문서 해독 (2/3 단계)    ← 상단 고정, 강조 색상
│   📍 수행 지역: 남쪽 산맥 — 이동 필요
├── ──────────────────────────
├── [엘리트] ⚔ 강철 골렘 토벌
├── 약탈 — 산적 소탕 (난이도 3)
└── ...
```

#### 진행도 관리 (Hive)

```

  chainId: String
```

`chainQuestProgress` Hive 박스에 저장. 완료된 체인은 `status: 'completed'`로 재발동 방지.

|------|------|--------|----------|----------|

### 6. 지역 변형 시스템

지역 지식 최고 임계값 도달 시 섹터가 물리적으로 변화. "내가 이 지역을 파고들었더니 세계가 바뀌었다"는 경험.

#### 변형 유형

| 유형 | 설명 | 효과 |
|------|------|------|
| `village` | 기존 섹터가 마을로 변경 | 마을 전용 퀘스트, 향후 평판/수익 분배 |
| `ruins` | 기존 섹터가 유적지로 변경 | 유적 전용 퀘스트 풀 (고난도, 고보상, 아이템 드랍률 상승) |
| `hidden_sector` | 새로운 섹터 추가 등장 | 기존 10개 + 1개, 전용 컨텐츠 |

#### discovery_data 구조 (transform 타입)

```json
// 마을 변형
{
  "transform_type": "village",
  "target_sector": 7,
  "name": "광부의 마을",
  "description": "폐광 주변에 사람들이 모여들어 작은 마을이 형성되었다"
}

// 유적지 변형
{
  "transform_type": "ruins",
  "target_sector": 3,
  "name": "고대 제단 유적",
  "description": "안개가 걷히자 잊혀진 제단이 드러났다"
}

// 숨겨진 섹터
{
  "transform_type": "hidden_sector",
  "sector_number": 11,
  "name": "봉인된 지하 통로",
  "description": "지역 조사 중 발견한 지하 통로가 개방되었다"
}
```

#### 변형 섹터 효과

- **마을**: 마을 전용 퀘스트(호위, 교역) 추가 생성. 향후 세력 시스템과 연결하여 마을 평판 → 수익 분배
- **유적지**: 전용 퀘스트 풀 (높은 난이도, 높은 보상, 아이템 드랍 확률 상승)
- **숨겨진 섹터**: 리전 섹터 10개 → 11개 확장. 전용 엘리트 몬스터나 연계 퀘스트 배치 가능

#### 현재 시스템과의 호환

현재 이동 시스템(리전 → 섹터 1~10)의 구조 위에 올라감:
- 섹터 선택 UI에서 변형된 섹터를 특수 아이콘 + 이름으로 표시
- 숨겨진 섹터는 목록 하단에 추가 등장
- 섹터 타입에 따라 퀘스트 생성 로직 분기

#### 첫 단계 범위

변형 트리거와 표시만 구현:
- 지식 도달 → 섹터 변형 발생 → 팝업 알림 + 활동 로그
- 이동 화면에서 변형된 섹터 시각적 구분
- 마을/유적지 전용 퀘스트 풀은 다음 단계에서 확장

### 7. 세력 시스템 기초

첫 단계는 세력의 존재 확립 + 지역 조사를 통한 발견까지만 구현.

#### 세력 설정 (검증용 3개)

| 세력 | 성격 | 활동 지역 | 분위기 |
|------|------|----------|--------|
| 은빛 교단 | 종교/학술 집단 | Tier 2~3 평원/도시 | 고대 지식을 수호하는 비밀 결사 |
| 야만족 연합 | 전투 부족 연맹 | Tier 3~4 산맥/숲 | 강함을 숭배하는 다부족 동맹 |
| 심연의 상회 | 상인/암흑 조직 | Tier 2~5 전역 | 정보와 돈으로 대륙을 움직이는 상단 |

#### factions 테이블 (Supabase 신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | String (PK) | 세력 ID |
| name | String | 이름 |
| description | String | 설명 |
| philosophy | String | 세력 이념 |
| tier_range | JSONB | 활동 티어 범위 |
| color | String | 대표 색상 |

#### 발견 메커니즘

`region_discoveries`에서 `type = 'faction_clue'` 트리거 시:

```json
{
  "faction_id": "faction_silver_order",
  "clue_text": "은빛 문양이 새겨진 석판을 발견했다.",
  "clue_level": 1
}
```

단서 레벨로 점진 공개:

| clue_level | 의미 | 예시 |
|------------|------|------|
| 1 | 흔적 발견 | "이 지역에 누군가의 활동 흔적이 있다" |
| 2 | 정체 파악 | "은빛 교단이라는 세력의 것으로 보인다" |
| 3 | 거점 발견 | "교단의 전초기지 위치를 파악했다" |

level 3 도달 시 향후 가입 시스템의 전제 조건 충족.

#### 유저 상태 (Hive)

```
FactionState {
  factionId: String
  discovered: bool
  maxClueLevel: int              // 1~3
  discoveredInRegions: List<String>
  // 향후: joined, contribution, rank
}
```

#### UI — 세력 도감

```
세력 도감
├── 은빛 교단          ★★★ (거점 발견)
│   "고대의 지식을 수호하는 비밀 결사"
│   발견 지역: 은빛 평원, 고원 도시
│
├── 야만족 연합        ★☆☆ (흔적만 발견)
│   "???"
│
└── ??? (미발견)       ☆☆☆
    "아직 발견하지 못한 세력이 있는 것 같다"
```

- 미발견 세력은 `???`로 표시 → 탐색 동기 부여
- 단서 레벨에 따라 세력 정보 점진 공개
- 세력 총 수는 감춤 (전부 발견했는지 모르게)

#### 세력 시스템 향후 로드맵

| 단계 | 내용 | 선행 조건 |
|------|------|----------|
| 1단계 (이번) | 세력 존재 + 단서 발견 + 도감 | 지역 조사 시스템 |
| 2단계 | 세력 가입 + 세력 전용 퀘스트 | 1단계 + 연계 퀘스트 인프라 |
| 3단계 | 기여도 + 세력 상점 (정수/장비 교환) | 2단계 + 아이템 시스템 |
| 4단계 | 세력 간 영향력 경쟁 (서버 연동) | 3단계 + Supabase 인증/쓰기 |
| 5단계 | 지역 점령 + 마을 평판 + 수익 분배 | 4단계 + 지역 변형 시스템 |

## 현재 시스템과의 연관
