# 스탯 체계 재설계 밸런스 분석 리포트

> 작성일: 2026-04-14
> 유형: 수치 조정 제안
> 분석 대상: 용병 스탯 시스템 (ATK/DEF/HP/speed → STR/INT/VIT/AGI)

---

## 현재 상태

### jobs 테이블 컬럼

| 컬럼 | 타입 | 범위 |
|------|------|------|
| base_atk | int | 4 ~ 50 |
| base_def | int | 3 ~ 32 |
| base_hp | int | 24 ~ 142 |
| speed | float | 0.76 ~ 1.39 |

### Mercenary 모델 스탯 필드 (HiveField)

| 필드 | HiveField | 사용처 |
|------|-----------|--------|
| atk | 4 | partyPower 계산에 사용됨 |
| def | 5 | 현재 미사용 (저장만 됨) |
| hp | 6 | 현재 미사용 (저장만 됨) |
| speed | 7 | 이동 시간 계산에 사용됨 |

### 현재 partyPower 공식

```dart
// quest_completion_service.dart:72
final partyPower = mercs.fold<int>(0, (sum, m) => sum + m.effectiveAtk);
```

**partyPower = ATK 합산만** 사용. DEF와 HP는 성공률·부상률·사망률 어디에도 기여하지 않는다.

## 데이터 분석

### 현재 아키타입별 직업 분포 (85개)

| 아키타입 | 직업 수 | 대표 직업 | ATK 범위 |
|---------|---------|---------|---------|
| 물리 딜러 (STR형) | 33개 | 검투사, 산적, 검성 | 4~50 |
| 마법/지원 (INT형) | 23개 | 마법사, 네크로맨서, 드루이드 | 25~50 |
| 탱커 (VIT형) | 16개 | 기사, 방패병, 팔라딘 | 15~32 |
| 속도/기동 (AGI형) | 10개 → 궁수 포함 시 14개 | 암살자, 정찰병, 레인저 | 13~39 |

### 핵심 불균형 지점

**탱커(VIT형)의 실질 전투력이 딜러보다 낮다:**

| 직업 (T3) | ATK | DEF | HP | speed | 실전 partyPower |
|----------|-----|-----|-----|-------|----------------|
| 전투 마법사 | 33 | 11 | 58 | 1.06 | **33** |
| 암살자 | 26 | 10 | 52 | 1.31 | **26** |
| 방패병 | 20 | 21 | 97 | 0.81 | **20** |
| 기사 | 20 | 21 | 97 | 0.81 | **20** |

→ 방패병(DEF 21, HP 97)이 마법사보다 전투력이 낮다. DEF, HP가 성공률에 기여하지 않기 때문.

**결론: DEF와 HP는 현재 유령 스탯이다.**

---

## 문제점

1. **탱커 직업의 존재 의의 없음**: 높은 DEF/HP가 게임 로직에 반영되지 않아 모집 우선순위에서 딜러 직업에 밀림
2. **마법사와 물리 딜러의 구분 없음**: 둘 다 ATK 합산으로만 평가 → 직업 아이덴티티 미분화
3. **speed 미활용**: 이동 시간에만 영향, 퀘스트 성공률과 무관
4. **향후 아이템 시스템 충돌**: Phase 2 아이템의 effect_json이 구 스탯(atk/def/hp) 기준으로 설계되면 이후 스탯 전환 비용 증가

---

## 플레이어 체감 분석

- **탱커 모집 동기 없음**: 방패병/기사는 파견 성공률이 낮아 플레이어가 기피하게 됨
- **직업 선택 무의미**: 마법사든 전사든 ATK 수치만 보고 선발 → 직업 다양성이 의미 없어짐
- **빠른 용병 활용 기회 없음**: speed가 높은 암살자/정찰병이 퀘스트에서 특별한 이점 없음

## 조정 제안

### 신규 4스탯 체계: STR / INT / VIT / AGI

| 스탯 | 역할 | 현재 대응 | 주력 직업군 |
|------|------|---------|-----------|
| STR | 기본 공격력 (전 직업의 근간 스탯) | base_atk | 전사, 검사, 기사 |
| INT | 스킬 공격력 + 스킬 사용 횟수 | *(신규)* | 마법사, 네크로맨서, 드루이드, 바드 |
| VIT | 체력(HP) + 방어력(DEF) 통합 | base_def + base_hp | 기사, 방패병, 팔라딘 |
| AGI | 이동속도 + 회피 + 스킬 쿨타임 감소 | speed | 암살자, 정찰병, 궁수 |

**핵심 원칙:**
- 기본 공격은 직업 무관하게 STR에만 의존
- 스킬(추후 도입)은 각 직업의 주력 스탯(INT/AGI 등)으로 시너지
- 단일 스탯이 여러 능력에 복합적으로 영향주지 않음 (단순성 유지)
- 퀘스트 유형별로는 복합 가중치 적용 (전략 깊이)

### 궁수류 분류: AGI형

궁수(T2), 탈영병(궁수), 숙련 궁수, 레인저는 현재 base_atk 기준으로 STR형이나, 전투 시스템에서 기동력·원거리가 주특기이므로 **AGI 주력**으로 재분류.
→ AGI형 14개로 균등화.

### 퀘스트 유형별 스탯 가중치

| 퀘스트 | STR | INT | VIT | AGI | 설계 의도 |
|--------|-----|-----|-----|-----|---------|
| raid (급습) | **0.70** | 0.10 | 0.10 | 0.10 | 정면 물리 충돌 |
| hunt (사냥) | **0.50** | 0.10 | 0.10 | **0.30** | 공격력 + 기동력 |
| escort (호위) | 0.20 | 0.10 | **0.60** | 0.10 | 생존력·방어 중심 |
| explore (탐험) | 0.10 | **0.45** | 0.15 | **0.30** | 지식 + 기동력 |

**신규 partyPower 공식:**
```
partyPower = Σ( merc.str × w_str + merc.int × w_int + merc.vit × w_vit + merc.agi × w_agi )
```

### VIT 통합 공식 (파생 예시)

```
실질 HP  = VIT × 3      (VIT 40 → HP 120, 기존 high-VIT 직업과 유사)
실질 DEF = VIT × 0.8    (VIT 40 → DEF 32, 기존 팔라딘 DEF와 유사)
```

### AGI 정수화 (speed → AGI)

```
base_agi = round(speed × 50)
→ 현재 범위: speed 0.76~1.39 → AGI 38~70 (정수 범위)
```

### 데이터 변환 방향 (85개 직업)

| 기존 | 신규 | 변환 방식 |
|------|------|---------|
| base_atk | base_str | ≈ base_atk (조정 필요) |
| base_def + base_hp | base_vit | 공식 역산 (VIT → def/hp 파생) |
| speed | base_agi | round(speed × 50) |
| *(없음)* | base_int | 전 직업 신규 설계 필요 (마법사 高, 전사 低) |

| *(신규)* | — | INT +1 (마력의 정수) |
| *(신규)* | — | AGI +1 (민첩의 정수) |
