## 2. 현재 상태

### 2.1 성공률 공식 (QuestCalculator.calculateSuccessRate)

```
rate = 50
     + (partyPower / enemyPower - 1) × 50       // 파티력 기여
     + questTypeMod                              // raid 0 / hunt -5 / escort 3 / explore 5
     + roleSynergy.clamp(-10, +10)               // 파견 상성 (파티 평균)
     + traitBonus.clamp(-10, +10)                // 트레잇 시너지
     + factionPassiveBonus                       // 세력/랭크 패시브 (+20%p 공유 상한)
     - distancePenalty                           // 리전 이동 패널티
     + randomVariance                            // -5 ~ +5%p
result = rate.clamp(5, 95)
```

계산 대상: `partyPower` = Σ mercenary raid 가중치 파티력. `enemyPower` = 현재 `difficulties.enemy_power` (레벨 1~5: 10/20/35/55/80). **M2b는 별도 엘리트 전용 전투력 컬럼(`elite_monsters.power`)을 참조**하여 공식의 `enemyPower`에 대입.

### 2.2 파티력 공식 (raid 기준 가중치)

raid: `str 0.70 + int 0.10 + vit 0.10 + agi 0.10`

현재 `jobs` 테이블 티어별 평균 스탯 기반 1인 raid 파티력 (레벨 1, 장비/정수 제외):

| 티어 | avg STR | avg INT | avg VIT | avg AGI | 1인 raid 파티력 |
|------|--------|--------|--------|--------|--------------|
| T1 | 7.6 | 5.8 | 26.3 | 53.3 | **13.9** |
| T2 | 20.2 | 12.4 | 52.1 | 53.5 | **25.9** |
| T3 | 25.0 | 21.5 | 68.6 | 51.2 | **31.6** |
| T4 | 31.3 | 35.2 | 83.1 | 49.3 | **38.7** |
| T5 | 37.7 | 37.5 | 102.0 | 46.6 | **45.0** |

레벨 보너스: `+10%p / level` (`GameConstants.levelBonusPerLevel = 0.1`). 레벨 5 최대 +40%.

### 2.3 role 상성 매트릭스 (RoleSynergyMatrix)

raid 유형 상위 role: warrior +8, rogue +5, ranger +3, specialist +2, support 0, mage -2. 파티 평균 ±10%p 독립 상한 클램프.

## 3. 데이터 분석 — 파티력 시나리오 4종

M2b 진입~엔드 기준 파티 구성 4시나리오. raid 유형, role 보너스는 warrior 비중 50% 가정(평균 +5.0%p). **장비·정수 실효 보정**은 M2a 완료 후 플레이어가 장비 1셋 장착 + T1~T2 정수 2~3개 투입한 상태를 `평균 +15~25 per 용병` 기준(`M2b 진입 기준선 가정`)으로 합산.

## 4. 문제점 — 미조정 전투력의 성공률 분포

로드맵 개략값(보통 T2=100 / T3=150 / T4=200 / 유니크 T2=150 / T3=180 / T4=210 / T5=250)을 단순 적용 시:

| 시나리오 × 엘리트 | partyPower | enemyPower | 기본 공식 | +roleBonus(+5) | 비고 |
|-----------------|-----------|-----------|----------|---------------|------|
| A × T2 보통 100 | 116 | 100 | **58%** | 63% | 진입 ✓ |
| A × T2 유니크 150 | 116 | 150 | 38.7% | 43.7% | **진입 너무 어려움** ✗ |
| B × T3 보통 150 | 183 | 150 | 61% | 66% | ✓ |
| B × T3 유니크 180 | 183 | 180 | 50.8% | 55.8% | 중반 ✓ |
| C × T4 보통 200 | 238 | 200 | 59.5% | 64.5% | ✓ |
| C × T4 유니크 210 | 238 | 210 | 56.7% | 61.7% | ✓ |
| D × T5 유니크 250 | 295 | 250 | 59% | 64% | **엔드 너무 쉬움** ✗ |

**문제점**:
1. **유니크 T2(단일 종, 늑대왕)가 150이면 진입 구간에 사실상 공략 불가** (43.7%). 유니크 T2는 "이 마일스톤의 첫 번째 맹주 처치 경험"을 체감시켜야 하므로 처음 발견한 플레이어가 3~5회 시도에 성공할 수 있어야 한다.
2. **T5 유니크(태고의 리치)가 250이면 엔드 파티(295)로 64%**는 "최종 보스급 도전"이라는 서사와 맞지 않게 너무 쉽다. 엔드 파티도 시작은 50% 내외, 반복 강화로 체감 상승하는 곡선이 필요.
3. 보통 엘리트 3개 티어(T2/T3/T4)의 간격이 50씩 동일하면 중후반(C, D 시나리오)에서 보통 엘리트가 70%+ 고정 — 리듬감은 좋지만 유니크와의 차별화(기대 보상 2~3배)를 감안하면 유니크 전투력을 더 올려야 한다.

## 5. 플레이어 체감 분석

### 5.1 엘리트의 "도전 vs 수익" 체감

- **보통 엘리트**: 일반 퀘스트의 1.5~2배 보상 + 1.5~1.8배 소요시간. 플레이어가 "일반보다 약간 더 힘들지만 수익이 명백히 높다"를 반복 체감해야 파밍 리듬 형성. 성공률 **60~75%** 구간이 이상적.
- **유니크 엘리트**: "도전 — 실패해도 배운다 — 다시 온다"의 서사. 성공률 **45~65%** 구간. 실패 시 인건비·파견비 손실 + 부상·사망 리스크로 "이기면 크고 지면 아프다"의 감정.

### 5.2 진행 벽 감지

- **A 시나리오(진입기)**: 유니크 T2(늑대왕)가 플레이어의 첫 유니크 경험이 될 가능성 큼. 전투력 120~125로 낮춰 **성공률 55~60%** 만들어야 "도전해볼 만하다" 체감.
- **D 시나리오(엔드)**: 태고의 리치가 단순 수치 도전이 아니라 "길드 장비 풀 장착 + 전설 정수 투입 후 비로소 안정 공략"이 되어야 함. 전투력 **245**로 엔드 파티(295) 기준 **60%+role → 68%**는 "Lv5 풀 강화 파티로 안정 공략 가능"이라는 의미.

### 5.3 의미 없는 선택지 배제

- 보통 T2와 유니크 T2(늑대왕)가 전투력 차이 35(95→120)로 구분되면 플레이어는 늑대왕을 "보통보다 한 단계 위 도전"으로 명확히 인지.
- 같은 티어 내 보통 엘리트들은 타입 가족에 따라 ±10~15 range (예: 골렘=VIT·STR 강함 → +10 / 정령=INT 편중 → -10)로 **스탯 편중이 전투력 차이로 드러나야** role 선택의 의미가 생긴다.

## 7. 시뮬레이션 — 시간당 엘리트 조우 기대값

퀘스트 갱신 주기 1시간, 정보망 시설 레벨로 퀘스트 슬롯 3~6개. **슬롯 4개 가정** (M2b 평균):

| 리전 티어 | 보통 엘리트 확률 | 유니크 (해금 후) | 슬롯 1개당 엘리트 총확률 | 슬롯 4개당 기대 엘리트 수 | 시간당 엘리트 조우 |
|----------|----------------|---------------|----------------------|----------------------|-----------------|
| T2 | 15% | 8% | 21.8% | 0.872 | **~0.87회/h** |
| T3 | 12% | 7% | 18.2% | 0.728 | **~0.73회/h** |
| T4 | 8% | 6% | 13.5% | 0.540 | **~0.54회/h** |

→ **M2b 진입기 T2 리전에서 시간당 ≈1회 엘리트 조우**, 후반 T4 리전에서는 2시간에 1회. 파밍 리듬·희소성 모두 충족.

**유니크 단독 조우 기대값** (T3 유니크 3종 모두 해금 시):
- T3 리전 1시간당 유니크 퀘스트 1개 이상 출현 확률 = 1 - (1-0.07)^4 ≈ 25.2%

→ 해금된 유니크 리전에 "4시간 머물면 1번은 유니크 만난다" 체감. 서사적 "맹주의 영역에 드물게 조우" 느낌과 합치.

## 9. data-generator 수치 가이드 (`elite-monster` 타입)

> 본 리포트는 **엘리트 몬스터 39종 신규 벌크 생성용 balance 기준**이다. 페이즈 3 `data-generator elite-monster` 실행 시 아래 범위·분포를 준수한다.

- **대상 타입**: `elite-monster` (신규, 페이즈 3-1 타입 스펙 선행)
- **대상 테이블**: `elite_monsters` (신규)
- **생성 수량**: 39행 (보통 31 + 유니크 8)
- **수치 범위**:
  - `power` (int, 전투력): §6.1·§6.2 고정값 매트릭스 준수 — 보통 85~200 / 유니크 120~245
  - `duration_multiplier` (real): §6.4 고정값 — 보통 1.5/1.6/1.8 / 유니크 1.7/1.8/2.0/2.0
  - `spawn_rate` (real 0.0~1.0): §6.5 고정값 — 보통 0.08/0.12/0.15 / 유니크 0.05/0.06/0.07/0.08
  - `tier` (int): 2~5
  - `is_unique` (bool): 유니크 8종 true
  - `type_family` (text): 10개 고정값 + `unique_transcendent` (기획서 §3.3·§3.4)
  - `environment_tags` (JSONB 배열): 1~3개 (§3.3·§3.4 매핑 준수)
  - `stat_weight` (JSONB, 합계 1.0): 타입별 편중 반영 — 골렘/트롤 {str:0.4, vit:0.4, int:0.1, agi:0.1} / 오크 {str:0.5, int:0.3, vit:0.1, agi:0.1} / 고블린 {agi:0.4, int:0.3, str:0.2, vit:0.1} / 언데드 {int:0.4, agi:0.3, str:0.2, vit:0.1} / 정령 {int:0.6, agi:0.2, vit:0.1, str:0.1} / 야수 {str:0.4, agi:0.4, vit:0.1, int:0.1} / 곤충 {agi:0.4, int:0.3, vit:0.2, str:0.1} / 리자드 {agi:0.4, str:0.3, vit:0.2, int:0.1} / 이매 {int:0.5, str:0.3, vit:0.1, agi:0.1} / 유니크 초월형 {자유, 서사 반영}
- **외래 키 제약**: `environment_tags` 값은 §3.2의 8개 태그 enum 준수. 유니크의 `fixed_region_environments`는 §3.4 "고정 리전 환경" 열 준수
- **분포 제약**:
  - 보통 엘리트 티어 분포: T2×8 / T3×13 / T4×10 (기획서 §3.3)
  - 유니크 엘리트 티어 분포: T2×1 / T3×3 / T4×3 / T5×1 (기획서 §3.4)
- **balance 근거**: §6.3 성공률 시뮬레이션이 파티 시나리오 A~D × 엘리트 전 조합에서 9~90% 범위 내 합리적 분포를 확인. M2a 파티(실효 116~295)에서 주 공략 엘리트 성공률 50~85% 분포 달성

## 10. 향후 확정 사항

- **[Q-1] 패시브 보너스 공유 상한 +20%p와 엘리트 성공률**: 세력·랭크 패시브가 엘리트 퀘스트에 모두 적용 시 시나리오 C × 보통 T2/T3가 95% clamp 구간 진입. 기대한 현상(엔드 파티가 저티어 엘리트를 "쓸어담는 구간")이므로 조정 없음. 페이즈 4 spec-writer 확인만.
- **[Q-2] T5 리전에 보통 엘리트 출현 여부**: 기획서 §3.3 "T5=0 (유니크 전용)". 본 리포트는 이 결정을 유지. T5 리전의 일반 퀘스트 슬롯은 100% 일반 퀘스트 or 유니크 T5(태고의 리치)만 채워짐. 후속 기획에서 T5 보통 엘리트 추가 검토 여지 둠.
- **[Q-3] 엘리트 퀘스트의 부상·사망률**: 현재 `difficulties.injury_rate/death_rate` 테이블은 level 1~5 기준. 엘리트는 기본적으로 `calculateDamage`의 level 5 값(injury 0.6 / death 0.3)을 상속할지, 별도 `elite_monsters.injury_rate/death_rate` 컬럼을 둘지 페이즈 4-2 모델 설계에서 확정. 본 리포트 추천: **별도 컬럼**으로 유니크 위험도를 차등(예: T2 유니크 0.5/0.2 → T5 유니크 0.8/0.4).
- **[Q-4] AGI 기반 시간 단축과 엘리트 소요시간**: 현재 `calculateDispatchDuration`은 `partyAverageAgi/50` 비례 시간 단축. 엘리트 배수 2.0×와 AGI 보정(파티 평균 AGI 75 → -33% 단축)이 상쇄되어 "유니크 T5 실제 시간 = 일반 1.33×" 정도. 의도 충족 여부 페이즈 4 체감 테스트 후 재검토.
