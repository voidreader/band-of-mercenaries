# 엘리트 몬스터 카탈로그 컨셉 기획서

> 작성일: 2026-04-20
> 유형: 신규 컨텐츠 (M2b 마일스톤 페이즈 1-1)
> 후속 산출물: 드랍 테이블 구조 기획(페이즈 1-2), 전투력·성공률 곡선(페이즈 2-1), 드랍률·파밍 시뮬레이션(페이즈 2-2)

---

## 1. 개요

M2b "엘리트의 시대" 마일스톤의 핵심 컨텐츠인 엘리트 몬스터를 **2계층 수직 구조**로 설계한다. 보통 엘리트(지속 파밍용, 31종)와 유니크 엘리트(지역 맹주, 8종)가 공존하여 "이 리전에 머물러 힘의 정수를 모을까, 다음 리전으로 넘어갈까"라는 전략적 판단을 만든다. 보통 엘리트는 환경 태그 기반으로 여러 리전에 퍼져 파밍 리듬을 공급하고, 유니크 엘리트는 고정 리전의 서사적 목표로 정수·장비·유물의 주 공급원이 된다.

**기대 효과**:
- 골드 경제 보강 (보통 엘리트의 골드 수급)
- M2a 정수·장비 소비 경로 활성화
- 환경 태그 기반 "전략 지도" (플레이어가 어느 환경=어느 스탯 정수를 배울 수 있음)
- M3 지역 변형·연계 퀘스트의 기반 인프라(환경 태그) 선행 구축

## 2. 레퍼런스 분석

| 레퍼런스 | 차용 포인트 |
|---------|------------|
| **Diablo 2/3** | "Champion / Rare pack"(보통 엘리트) vs "Super Unique Monster"(지역 고정 네임드 보스)의 2계층 구조. 엘리트마다 특성·내성이 다르고, 네임드는 확정 드랍 풀을 보유 |
| **Melvor Idle** | 몬스터 타입 가족 변주(고블린 → 언데드 고블린 → 네크로멘서 고블린)로 플레이어가 "같은 계통의 상위 개체"를 학습하는 구조 |
| **Kingdom of Loathing** | 네임드 유니크 보스(The Naughty Sorceress 등)에 집중된 서사·보상 밀도 |
| **"게임 속 바바리안으로 살아남기"** (idea_note.md 레퍼런스) | "희귀한 정수"와 "이 지역의 맹주" 감성. 유니크 엘리트의 네이밍·서사 톤에 반영 |

**차용하지 않은 요소**:
- Diablo의 엘리트 접두사 랜덤 조합(화염/냉기/광기 등) — 현재 게임은 텍스트 기반이라 조합 효과 시각화가 어렵고 밸런스 복잡도만 증가. M2b 범위 외로 둔다.
- Melvor의 슬레이어 티어 수집 메타 — 용병 간 전투 테마가 다르므로 미채택.

## 3. 상세 설계

### 3.1 2계층 엘리트 구조

| 계층 | 종수 | 서사 성격 | 보상 성격 | 출현 방식 |
|------|------|----------|----------|----------|
| **보통 엘리트** | 31종 | 일반 몬스터보다 강한 개체. 타입 가족 변주 | 골드 高, 정수·장비 낮은 확률 | 환경 태그 매칭 리전에서 퀘스트 갱신 시 확률 출현 (단서 불필요) |
| **유니크 엘리트 (맹주)** | 8종 | 지역을 상징하는 네임드 보스. 고유명·서사 | 골드 매우 高, 정수·장비 높은 확률, 용병단 유물 가끔 | 지역 조사로 단서 발견 후 해금. 처치 후에도 반복 출현 (쿨다운 없음) |

**핵심 철학**:
- 보통 엘리트 = **지속 수익 파밍의 리듬**
- 유니크 엘리트 = **목표 도전의 랜드마크**
- 두 계층이 공존해야 플레이어가 "지루하지 않게 보상 획득"과 "감정적 목표"를 동시 체감

### 3.2 환경 태그 세트 (8종, JSONB 다중 태그)

`regions.environment_tags` JSONB 배열 필드 신설. 한 리전은 1~3개 태그를 동시 보유 가능.

| 태그 | 한국어 | 예시 리전 성격 |
|------|--------|---------------|
| `ruins` | 폐허 | 고대 유적, 폐광, 버려진 요새 |
| `forest` | 숲 | 깊은 숲, 정글, 원시림 |
| `swamp` | 습지 | 늪, 안개 골짜기 |
| `mountain` | 산악 | 산맥, 고산, 협곡 |
| `desert` | 사막 | 황야, 건조지대 |
| `coast` | 해안 | 해변, 항구, 섬 |
| `underground` | 지하 | 동굴, 지하 미궁, 광산 내부 |
| `plains` | 평원 | 초원, 농지, 마을 근교 |

**다중 태그 예시**:
- 폐광 (Tier 2~3) = `[ruins, underground, mountain]`
- 늪의 유적 (Tier 3) = `[ruins, swamp]`
- 항구 마을 주변 (Tier 1) = `[coast, plains]`

**향후 확장 여지**: `arctic`(설원), `volcanic`(화산) 등은 M3+ 지역 변형 시점에 필요 시 추가.

### 3.4 유니크 엘리트 — 지역 맹주 8종

각 유니크는 10 타입 가족 중 하나의 "왕" 포지션을 맡거나, 타입 초월 유니크로 존재.

| 이름 | 티어 | 서사 요약 | 고정 리전 환경 | 포지션 |
|------|------|---------|--------------|-------|
| **늑대왕 울부르** | T2 | 북부 숲을 지배하는 거대 회색 늑대. 달밤이면 울음이 산맥을 울린다 | forest, mountain | 야수류 王 |
| **강철 골렘** | T3 | 폐광 최심부, 잊혀진 연금술사가 남긴 불사의 수호자. 로드맵 예시 반영 | ruins, underground | 골렘류 王 |
| **습지의 히드라** | T3 | 검은 늪 깊은 곳에서 깨어난 세 머리 독룡. 독안개를 뿜는다 | swamp | 리자드·용족 王 |
| **백골의 장군** | T3 | 잊혀진 왕국의 죽은 사령관. 부하 스켈레톤 군단을 거느린다 | ruins, plains | 언데드류 王 |
| **사막의 파수꾼** | T4 | 태양 아래 수천 년을 깨어있는 거대 석상. 고대 유적을 지킨다 | desert, ruins | 타입 초월 (이형의 수호자) |
| **검은 마녀 모르간** | T4 | 숲의 심부에서 저주를 엮는 고대 마녀. 주술로 용병을 정신 피해 입힌다 | forest, ruins | 이매·악마류 王 |

**티어 분포**: T2×1 / T3×3 / T4×3 / T5×1

**반복 출현 규칙**:
- 유니크 엘리트는 지역 조사 → `region_discoveries.discovery_type = 'elite'` 단서 발견으로 해금
- 해금 후 해당 리전의 퀘스트 갱신 시 5~8% 확률로 엘리트 퀘스트 생성
- **처치 후에도 반복 출현** (쿨다운 없음, 로드맵 준수). 보상 풀 전체가 확정 드랍이 아니므로 무한 파밍 우려 낮음

### 3.5 보통 엘리트 출현 규칙

- 지역 조사·단서 불필요 — 해당 리전 환경 태그 매칭 엘리트가 퀘스트 갱신(1시간 주기) 시 확률 출현
- 티어별 기본 출현 확률: T2 = 15% / T3 = 12% / T4 = 8% (정확 수치는 페이즈 2 확정)
- 한 리전에 환경 태그가 여러 개면 해당하는 모든 타입 가족이 후보 풀에 포함됨
- 같은 퀘스트 슬롯에 유니크와 보통이 모두 후보일 때: **유니크 판정 먼저, 실패 시 보통 판정**

## 4. 현재 시스템과의 연관

### 4.1 영향받는 기존 시스템

- **`regions` 테이블** — `environment_tags` JSONB 컬럼 신규 추가 (199행 데이터 채우기). 로드맵 M2b 기본 스코프에는 없었던 확장. M3 "지역 변형"에도 재활용 가능해 중장기 이득
- **`region_discoveries`** — `discovery_type = 'elite'` 타입 추가. 유니크 엘리트 8종에 대한 discovery 행 8~16개 생성 (리전 고정·다중 환경 매칭 시 여러 행)
- **`quest_pools`** — 엘리트 퀘스트 전용 풀 필요 여부는 페이즈 4 개발 명세에서 결정. 기본안은 `ActiveQuest.isElite` 플래그로 일반 풀 재활용
- **M2a `items` 테이블 (정수·장비)** — 드랍 테이블의 item_id 외래키로 참조. M2a에서 생성된 정수 20종·장비 세트가 드랍 타겟이 됨
- **M2a `InventoryRepository`** — 드랍 판정 결과를 `addItem()`으로 인벤토리에 적재 (기존 인프라 재사용, 변경 불필요)
- **`UserData` 용병단 유물 슬롯 (`artifactItemIds`)** — 유니크 엘리트 드랍의 용병단 유물이 인벤토리에 추가됨. 장착은 M2a 장착 UI 경로 재사용
- **`PassiveBonusService`** — 엘리트 토벌은 패시브 보너스 자동 적용 (세력/랭크 보상 배수 등). 기존 경로 재사용, 변경 불필요

### 4.2 신규 생성 필요 요소

- `elite_monsters` 테이블 (신규)
- `elite_loot_tables` 테이블 (신규)
- `regions.environment_tags` 컬럼 (기존 테이블 확장)
- data-generator 타입 스펙 — `elite-monster` (통합 생성), `region-environment-tag` (리전 태그 부여)
- Flutter `EliteMonsterData` / `EliteLootEntry` Freezed 모델
- `EliteLootService` 도메인 서비스 (드랍 판정)
- 파견 화면 엘리트 구분 시각 요소 (아이콘·강조 색상)
- 퀘스트 완료 팝업 아이템 드랍 리스트

### 4.3 호환성 검토

- **기존 일반 퀘스트와의 공존**: 엘리트 퀘스트는 `ActiveQuest.isElite` 플래그로 구분. 일반 퀘스트 풀 수 상한(정보망 시설 보너스 포함)은 기존 로직 유지, 엘리트는 해당 풀의 "특수 슬롯"으로 치환 출현
- **세력 전용 퀘스트(M1)와의 충돌**: 엘리트 퀘스트와 세력 전용 퀘스트는 독립된 슬롯 판정. 파견 화면 정렬 순서는 M3의 "공존 정책 정의" 시점에 확정 (현 시점 엘리트는 세력 전용 다음 / 일반 위 우선순위)
- **시간 가속 호환**: 엘리트 퀘스트 소요시간(일반의 1.5~2배)도 기존 `QuestCalculator`의 AGI 기반 시간 보정 + 시간 가속 재계산 경로를 그대로 탄다
- **성공률 공식**: 엘리트 전투력(100~200)이 기존 `partyPower vs enemyPower` 공식에 자연스럽게 반영. 파견 상성·트레잇·패시브 보정 전부 기존 경로 재사용. UI의 `SuccessRateBreakdownSheet`도 엘리트 퀘스트에서 변경 없이 작동

## 6. data-generator 지시사항

### 6.1 대상 1 — 엘리트 몬스터 생성

- **대상 타입**: `elite-monster` (신규 타입 스펙 작성 필요 — 페이즈 3 선행)
- **대상 테이블**: `elite_monsters`
- **생성 수량**: 39종 (보통 31 + 유니크 8)
- **톤/세계관 가이드**:
  - 몬스터 이름은 한국어. 판타지 표준 + 기존 게임 톤(세력명·트레잇명 참조)과 조화
  - 보통 엘리트: "수식어 + 종족명" 형식 권장 (예: "녹슨 골렘", "오크 대전사")
  - 유니크 엘리트: 고유명 + 칭호 권장 (예: "늑대왕 울부르", "검은 마녀 모르간")
  - 설명 텍스트: 2~3문장. 출신·특징·공격 방식 암시
  - 금기: 브랜드명·실존 인물명·과도한 성적/잔혹 표현
- **구조적 제약**:
  - `is_unique`(bool): 유니크 8종만 true
  - `type_family`(text): 10개 고정값 중 하나 (golem / orc / goblin / troll / lizardman / undead / elemental / beast / insect / demon) + 유니크 초월형은 `unique_transcendent`
  - `tier`(int): 2~5. 분포는 §3.3·3.4 표 준수
  - `environment_tags`(JSONB 배열): 1~3개. §3.3의 "출현 환경" 준수
  - `power`(int): 로드맵 100~200 범위 (정확 수치는 balance-design 페이즈 2 산출물 참조)
  - `duration_multiplier`(float): 일반 퀘스트 대비 1.5~2.0
  - `stat_weight`(JSONB): {"str": 0.4, "vit": 0.4, ...} 합계 1.0, 타입별 편중 반영 (§3.3 "스탯 편중" 열)
  - 유니크 전용 필드: `fixed_region_environments`(JSONB, 복합 환경 태그), `lore`(text, 서사 4~5문장), `title`(text, 예: "북부 숲의 왕")
- **수치 출처**: `Docs/balance-design/[balance]{페이즈2-1 산출물}` (페이즈 2 선행 필요)
- **특수 요구**:
  - 유니크 엘리트 8종은 본 기획서 §3.4 표의 이름·서사·포지션 고정. data-generator는 설명 텍스트(3~5문장)만 생성
  - 보통 엘리트 31종은 본 기획서 §3.3 표의 이름 고정. 설명(2~3문장)만 생성

### 6.2 대상 2 — 엘리트 드랍 테이블

- **대상 타입**: `elite-loot-table` (신규, 또는 `elite-monster` 통합)
- **대상 테이블**: `elite_loot_tables`
- **생성 수량**: 약 200~270행 (엘리트당 평균 5~7 드랍 항목)
- **구조적 제약**:
  - 각 행: `elite_id` (FK), `item_id` (M2a items FK), `drop_rate` (0.0~1.0)
  - 드랍 테마 매핑은 §3.6 표 준수 (타입 → 주 드랍 정수·장비 슬롯)
  - 보통 엘리트: 골드(기본)·정수(낮음)·장비(드물게) 구성
  - 유니크 엘리트: 골드(높음)·정수(높음)·장비(자주)·용병단 유물(가끔) 구성
  - 외래 키 무결성: item_id는 M2a `items` 테이블에 실재해야 함
- **수치 출처**: `Docs/balance-design/[balance]{페이즈2-2 산출물}` (드랍률 시뮬레이션 결과)

### 6.3 대상 3 — 리전 환경 태그 부여

- **대상 타입**: `region-environment-tag` (신규 타입 스펙 작성 필요)
- **대상 테이블**: `regions` (기존 199행 UPDATE)
- **생성 수량**: 199행 (기존 리전 전체)
- **톤/세계관 가이드**:
  - 8개 태그(§3.2) 중 1~3개 선정
  - 리전 `description` 키워드 기반 반자동 분류 (예: "폐광·유적" → ruins, "숲·정글" → forest)
  - 모호한 경우 `region_name`과 `region_tier`의 의미적 분위기 참조
  - 태그 없이 빈 배열로 남는 리전은 최소화 (특수한 서사 리전 외에는 최소 1개 태그)
- **구조적 제약**:
  - 다중 태그 시 최대 3개
  - 환경 태그 간 충돌 검증: `desert`와 `swamp` 동시 태깅 불가 (상식 수준)
  - `plains`는 다른 태그와 병행 가능 (평원+해안 = coast 우선)
- **특수 요구**: 리전별 태그 분포가 §3.7 추정 분포에서 ±30% 벗어나지 않도록 조정

### 6.4 대상 4 — 유니크 엘리트 region_discovery 행

- **대상 테이블**: `region_discoveries` (기존)
- **생성 수량**: 8~16행 (유니크 8종 × 평균 1~2 리전)
- **구조적 제약**:
  - `discovery_type = 'elite'`
  - `discovery_data` JSONB: `{ "elite_id": "...", "reveal_text": "..." }`
  - `knowledge_threshold`: 유니크 티어별 (T2: 30 / T3: 50 / T4: 70 / T5: 85)
  - 유니크가 "고정 리전 환경"에 다중 태그인 경우(예: 강철 골렘 = ruins+underground), 해당 환경을 모두 만족하는 리전 중 티어가 일치하는 것을 선정 (구체 리전 ID는 리전 태그 데이터 생성 후 매칭)
- **수치 출처**: 본 기획서 §3.4 표 + §3.7 환경 태그 분포

## 7. 미결·향후 확정 사항

- **드랍 확정 방식**: 각 드랍 항목 독립 확률 판정(현 계획) vs 가중치 기반 1개 드랍 — 페이즈 2 balance-designer가 결정. 기본안은 "독립 확률 판정, 평균 드랍 수 1.5~2.5개"
- **엘리트 퀘스트 유형 매핑**: 기존 4개 퀘스트 유형(raid/hunt/escort/explore) 중 엘리트는 주로 `raid`(약탈이 아닌 토벌 의미)로 고정할지, 타입별로 분기할지 — 페이즈 4 개발 명세 결정. 추천: 전 엘리트 `raid` 고정으로 시작
- **파견 상성 반영**: 엘리트 스탯 편중 + `role` 매트릭스를 고려한 추천 role 표시는 일반 퀘스트 로직(`RoleSynergyMatrix.topRolesForQuest`) 자동 적용
- **레벨 보정**: 엘리트가 Tier 5 리전에 출현할 때 최대 출현 티어를 T4로 제한할지, T5까지 해금할지 — 페이즈 2 "누적 플레이 기준" 체크 필요
- **스펙트럼 외 몬스터**: "드래곤(대형 용)"은 의도적으로 제외. M4 이후 "레이드급 유니크" 컨셉으로 별도 기획 권장

## 8. 밸런스·구현 플래그

- **[Q-1] 환경 태그 충돌 검증**: `desert`+`swamp`, `coast`+`desert` 등 물리적으로 모순되는 조합을 DB CHECK 제약으로 강제할지, data-generator 검증만 할지 — 구현 명세에서 결정
- **[Q-2] 유니크 엘리트 단서 중복**: 같은 유니크가 여러 리전에서 발견 가능할 때(강철 골렘 = ruins+underground 복합 리전이 3개 존재) 한 리전에서 해금해도 다른 리전에서 함께 해금될지, 리전마다 독립 해금일지 — 추천: 유니크는 "존재 자체"가 해금되면 해당 환경 매칭 모든 리전에서 출현 (리전마다 독립 해금은 플레이어 피로도 증가)
- **[Q-3] 페이즈 3 타입 스펙 범위**: `elite-monster`와 `elite-loot-table`을 단일 타입 스펙에서 통합 처리할지 분리할지. 추천: 분리 (드랍 테이블은 item_id 외래키 검증 로직이 별개). `region-environment-tag`는 완전 별개 타입 스펙
