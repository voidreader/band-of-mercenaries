## 개요

M8a는 14세력 전체를 확장하지 않고, M7 생활권에서 자연스럽게 등장할 수 있는 대표 세력 3개만 먼저 수직 절편으로 구현한다. 본 문서는 M8a에서 우선 구현할 세력을 **모험가 길드**, **상인 연합**, **전사 길드**로 확정하고, 각 세력이 더스트빌 생활권에 접근하는 계기와 접촉점을 정의한다.

핵심 원칙은 세력을 "메뉴의 보너스 제공자"가 아니라 "더스트빌에서 이름을 얻은 용병단을 발견한 외부 후원자"로 보이게 하는 것이다. 세력 본부는 아직 구현하지 않고, 지부·대리인·임시 사무소 형태로만 노출한다.

## 레퍼런스 분석

| 레퍼런스 | 차용 메커니즘 | 적용 방식 |

## 대표 세력 선정

### 선정 기준

대표 세력은 다음 조건을 만족해야 한다.

- M7 생활권 안에서 자연스러운 등장 계기를 가진다.
- 기존 M1 세력 패시브와 충돌하지 않는다.
- 세력별로 다른 퀘스트 유형과 보상 방향을 가진다.
- M8b 전투화 대상으로 확장하기 좋다.
- 14세력 전체 확장 없이도 MVP가 닫힌다.

### 확정 세력

| 우선순위 | 세력 | 기존 정체성 | M8a 생활권 역할 | 핵심 리전 | 채택 이유 |
|---------|------|------------|----------------|----------|----------|
| 1 | 모험가 길드 | 탐험·기록·의뢰 중개 | 더스트빌 성과를 공식 기록으로 편입하는 첫 외부 지부 | region 3, 10, 38 | M7 체인과 위업을 가장 자연스럽게 흡수한다 |
| 2 | 상인 연합 | 계약·교역·호송 | 외래 좌판과 변방 해안 교역로를 관리하는 대리인 | region 3, 127, 31 | M7 인프라 Tier 3과 직접 연결된다 |
| 3 | 전사 길드 | 명예·토벌·강자 증명 | 부서진 요새와 외곽 숲의 강적을 시험장으로 삼는 무장 연락관 | region 9, 38 | 엘리트·전투 보고서 MVP와 가장 잘 맞는다 |

### 보류 세력

| 세력 | 보류 사유 | 후속 후보 |
|------|----------|----------|
| 도둑 길드 | region 31 도적길과 연결성은 높지만, M8a에서 상인 연합과 전사 길드 갈등까지 동시에 넣으면 초반 정치 구도가 과밀해진다 | M8b 또는 M9에서 상인 연합의 반대축으로 추가 |
| 태양 교단 | 저주·정화 테마는 회색 늪지와 연결 가능하지만, M8a의 생활권 경제·보고서 검증 목표와 우선순위가 낮다 | M8b 전투 상태 이상 또는 M9 고위험 지역 |
| 심층 망치단 | 부서진 요새·폐광·제작과 연결 가능하지만, M7 인프라 성장에서 이미 대장간 축을 많이 사용했다 | M9 제작 데이터 확장 |
| 지역 유목민 세력 | region 127에 강한 접촉점이 있으나 기존 14세력에 독립 세력이 없다 | 상인 연합의 현지 협력자 또는 반응 텍스트로 처리 |

## 생활권 접촉점 설계

### 모험가 길드 — 더스트빌 임시 기록소

모험가 길드는 더스트빌의 위업과 지역 안정화 기록을 수집하기 위해 임시 기록소를 세운다. 플레이어에게는 "이제 용병단의 사건이 지역 소문을 넘어 공식 기록에 남는다"는 신호로 작동한다.

| 항목 | 설계 |
|------|------|
| 접촉점 이름 | 더스트빌 임시 기록소 |
| 접촉 NPC | 기록원 라우덴 |
| 최초 노출 조건 | `infrastructureTier >= 2` 또는 위업 `settlement_event_completed:settlement_3_pyegwang_reopen` 보유 |
| 핵심 리전 | region 3 더스트플레인, region 10 풍신 숲, region 38 부서진 요새 |
| 연결 시스템 | 위업, 칭호, 연계 퀘스트, 지명 의뢰 |
| 첫 반응 | "폐광을 다시 연 용병단이라면, 길드 장부 한 칸쯤은 내줄 수 있겠군." |

모험가 길드의 접촉점은 마을 방문 화면의 더스트빌 광장 또는 정보 탭의 세력 상세에서 접근한다. M8a에서는 독립 건물 UI를 만들지 않고, 세력 상세에 "생활권 접촉점" 섹션을 추가하는 방향을 권장한다.

### 상인 연합 — 케일의 거래 장부

상인 연합은 M7 외래 좌판의 NPC 케일을 통해 더스트빌에 접근한다. 유목민 친교와 변방 해안의 교역로가 열린 뒤, 케일은 단순 행상이 아니라 상인 연합의 임시 대리인으로 격상된다.

| 항목 | 설계 |
|------|------|
| 접촉점 이름 | 케일의 거래 장부 |
| 접촉 NPC | 외래 상인 케일 |
| 최초 노출 조건 | `infrastructureTier >= 3` 또는 `region_127_nomad_friendly` 보유 |
| 핵심 리전 | region 127 변방 해안, region 31 도적길, region 3 더스트빌 |
| 연결 시스템 | 외래 좌판, 재료 거래, 호위 의뢰, 지역 상태 |
| 첫 반응 | "더스트빌 길이 안전해졌다면, 이제 장부에 더 큰 이름을 올릴 때입니다." |

상인 연합은 유목민을 독립 세력으로 만들지 않는다. 유목민은 변방 해안의 현지 협력자로 남기고, 상인 연합이 이 교역망을 공식화하는 구조를 취한다.

### 전사 길드 — 부서진 요새 결투 표식

전사 길드는 region 9의 야수 처치와 region 38의 요새 안정화를 보고 연락한다. 접촉점은 더스트빌이 아니라 부서진 요새와 외곽 숲의 전투 흔적에 배치한다.

| 항목 | 설계 |
|------|------|
| 접촉점 이름 | 부서진 요새 결투 표식 |
| 접촉 NPC | 붉은 창의 벨란 |
| 최초 노출 조건 | 위업 `elite_unique_first_kill:*` 또는 `region_38_ironbound_pact_completed` 보유 |
| 핵심 리전 | region 9 외곽 숲, region 38 부서진 요새 |
| 연결 시스템 | 엘리트 의뢰, 전투 보고서, 칭호, 위험도 |
| 첫 반응 | "짐승을 쓰러뜨린 이름은 들었다. 요새의 돌벽 앞에서도 그 이름이 통하는지 보자." |

전사 길드는 M8a의 전투 보고서 MVP와 가장 강하게 연결된다. 세력 의뢰 완료 후 누가 전열을 버텼고, 누가 결정타를 냈는지를 보고서에 남기는 첫 실험 대상이다.

## 접촉점 해금 흐름

### 5~8시간 이후 기본 흐름

M7 종료 시점의 플레이어는 보통 인프라 Tier 3~4, 지역 상태 플래그 5개 이상, 위업 2개 이상을 보유한다. M8a는 이 상태를 기준으로 다음 흐름을 제공한다.

1. 더스트빌 귀환 시 모험가 길드 기록원이 도착한다.
2. 외래 좌판 방문 시 케일이 상인 연합 대리인임을 밝힌다.
3. region 9 또는 region 38의 의미 있는 전투를 마치면 전사 길드 표식이 나타난다.
4. 정보 탭의 세력 상세에 "생활권 접촉점" 섹션이 열린다.
5. 각 접촉점은 세력 지명 의뢰 2~5개와 축소 상점 5~8개를 제공한다.

### 게이트 정책

| 게이트 | 적용 세력 | 이유 |
|--------|----------|------|
| 인프라 Tier 2 | 모험가 길드 | 더스트빌이 외부 기록을 받을 정도의 안정성을 갖췄다는 신호 |
| 인프라 Tier 3 | 상인 연합 | 외래 좌판이 생긴 뒤 상업 접촉이 자연스럽다 |
| 엘리트 처치 또는 region 38 사건 | 전사 길드 | 전투력을 증명한 뒤에만 접촉한다 |
| 평판 > 0 | 모든 세력 | 기존 `FactionJoinService` 규칙 유지 |
| 가입 전 평판 상한 10 | 모든 세력 | M1 구조 유지 |

## 현재 시스템과의 연관

| 시스템 | 영향 | 설계 방침 |
|--------|------|----------|
| `FactionData` | 생활권 접촉점 필드가 필요할 수 있다 | `contact_json` 또는 신규 테이블을 페이즈 4에서 검토한다 |
| `FactionState` | 접촉점 방문 이력, 접촉 해금 여부 저장 후보 | HiveField 6 이후 사용 가능 여부를 페이즈 4에서 검토한다 |
| `RegionState` | 인프라 Tier와 unlockedFlags를 게이트로 사용 | 기존 M7 필드를 재사용한다 |
| 지명 의뢰 | 세력 조건이 지명 의뢰 hook에 추가된다 | 기존 `is_named` 구조를 확장한다 |
| 정보 탭 | 세력 상세 화면에 접촉점과 미니 상점 진입점을 추가한다 | 독립 화면보다 기존 세력 상세 확장을 우선한다 |

## 구현 우선순위 제안

| 우선순위 | 항목 | 이유 |
|----------|------|------|
| 높음 | 세력 3개 접촉점 정의와 해금 조건 | M8a 전체 데이터와 명세의 기준점이다 |
| 높음 | 접촉점 반응 텍스트 30~50개 | 세력이 "도착했다"는 체감을 만든다 |
| 중간 | 접촉점 방문 이력 영속 | MVP에서는 단순 해금 상태로도 가능하지만 재방문 반응을 만들려면 필요하다 |
| 낮음 | 독립 세력 지부 화면 | M8a 범위에서는 세력 상세 확장으로 충분하다 |

## data-generator 지시사항

- **대상 타입**: 신규 타입 스펙 필요. 기존 `faction-quest` 타입은 의뢰용이므로 접촉점·반응 텍스트 타입 스펙을 별도로 정의한다.
- **대상 테이블**: 신규 후보 `faction_contacts`, `faction_reaction_texts`
- **생성 수량**: 접촉점 3개, 반응 텍스트 30~50개
- **톤/세계관 가이드**:
  - 모험가 길드: 기록, 장부, 지도, 공식 인정
  - 상인 연합: 계약, 장부, 호송, 교역로, 외래 좌판
  - 전사 길드: 표식, 결투, 강자 증명, 요새, 야수
- **구조적 제약**:
  - 세력 본부는 등장시키지 않는다.
  - 더스트빌 생활권 밖의 대도시 이름은 사용하지 않는다.
  - 유목민은 상인 연합의 현지 협력자로만 다룬다.
- **수치 출처**: 페이즈 2 밸런스 검토 필요
