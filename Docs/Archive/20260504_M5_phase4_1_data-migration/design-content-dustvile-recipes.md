## 개요

M5 시작 거점에서 플레이어가 만들 수 있는 **레시피 10개**를 정의한다. roadmap M5 라인 962~968의 대표 제작 목표 3개(낡은 용병단 깃발 복원·광부의 단검·폐광의 유물 조각)를 정식 레시피로 명문화하고, 학습 곡선·재료 활용 보장·결과물 다양화를 위한 부가 레시피 7개를 추가한다.

본 문서가 확정하는 것:
- 레시피 10개 명단 (결과물·입력·해금 조건·효과 방향성)
- **제작 위치 단일화** 정책 — 모든 레시피는 낡은 대장간(region 3 sector 1)에서 제작
- **즉시 제작** 정책 — 재료 보유 시 클릭 1회로 결과물 획득 (제작 큐 없음)
- 첫 제작 30~45분 / 첫 희귀 장비 90~150분 누적 플레이 검증
- 미사용 재료 #3 마른 약초·#4 산기슭 버섯의 부가 레시피 활용
- T2 weapon 2종 학습 분기 (광부의 단검 STR / 녹슨 곡괭이 VIT)
- 페이즈 2 #2 입력으로 effect_json 보편 거시 지표 키 + 거점 특화 효과 옵션 분리

본 문서가 확정하지 않는 것 (후속 위임):
- effect_json 구체 수치 — 페이즈 2 #2
- 입력 재료 수량 최종 밸런스 (드랍률과 동시 검증) — 페이즈 2 #1·#2
- `crafting_recipes` 테이블 vs JSONB MVP 결정 — 페이즈 4 #1
- 거점 특화 effect_json 키 신설 여부 — 페이즈 4 #1

## 레퍼런스 분석

| 레퍼런스 | 차용 메커니즘 | 적용 방식 |
|---------|-------------|----------|
| Melvor Idle — Smithing 중간재 정제 (Bronze Bar → Iron Bar) | 원자재 → 정련 중간재 → 최종 결과물의 3단계 가공 | 부가 레시피 #4 거친 가죽끈 묶음(hide 정제) + #5 연마된 쇳조각(ore 정제) — 첫 제작 목표 3개에 입력 옵션 추가 |
| Stardew Valley — Tool Upgrade 양자택일 | 동일 슬롯 도구를 다른 효과로 강화하는 분기 (Steel vs Iridium 곡괭이 등) | 광부의 단검(STR+AGI) vs 녹슨 곡괭이(VIT+STR) — T2 weapon 2종 선택. 동일 등급 다른 효과 |
| Path of Exile — Crafting Bench 단일 제작소 | 모든 제작이 한 NPC 인터페이스에서 일어남. 분산 제작소 없음 | M5 MVP는 낡은 대장간 단일 제작소. 거점별 제작 분기는 M5+ 다중 거점 도입 시 진화 |
| Disco Elysium — Sub-equipment Trio | accessory 1슬롯 + 메인 슬롯의 효과 조합으로 빌드 다양화 | "약초 향낭"(accessory T2) — VIT 보정 + 부상률 감소. 깃발/단검과 함께 장착하여 빌드 완성 |
| Kingdom of Loathing — Quest 진행에 맞춘 레시피 해금 | "이 사건을 해결해야 이 레시피가 열린다"의 단계적 해금 | 폐광의 유물 조각 = step 6 완료 후 해금 (`#10 고대 인장 조각`이 step 6 보상이므로 자연 해금) |

**기존 게임과의 차별 포인트**: 본 게임은 "제작이 곧 학습"의 구조. 레시피 10개를 통해 재료 출처(마을 의뢰·폐광 조사·엘리트·고정사건) 5종을 모두 거치게 한다. 한 빌드를 완성하려면 **여러 활동을 순환**하며, 이 순환 자체가 M5 핵심 게임 루프의 확장이다.

## 상세 설계

### 1. 제작 정책 (전역 규칙)

#### 1-1. 제작 위치 — 낡은 대장간 단일

| 항목 | 정책 |
|---|---|
| 제작 가능 위치 | **낡은 대장간** (region 3 sector 1 더스트빌 내 거점) |
| 제작 NPC | 하겐 (M4 NPC `npc_smith_hagen` 그대로 활용) |
| 제작 거리 | 마을 방문 시 3개 거점 중 낡은 대장간 진입 → 기존 M4 거점 화면 확장 |
| 제작 외 거점 | 약초상·촌장 집은 **제작 미지원**. 채집 의뢰·신뢰도 보상 등 재료 공급 역할만 |
| M5+ 확장 | 다중 거점 도입 시 거점별 제작 분기 가능. M5 범위 외 |

**근거**:
- roadmap M5 라인 985 "거점 대장간 또는 제작 장소에서 레시피를 확인하고 제작" 충족
- M4 페이즈 1 #3 stub에서 하겐의 "제작 목표 보기" 버튼이 이미 예고됨 → 자연스러운 정식 활성화
- 분산 제작소 도입 시 UX 학습 부담 + 거점별 잠금 정책 복잡도 증가 → M5 MVP 단순성 우선

#### 1-2. 제작 시간 — 즉시 제작

| 항목 | 정책 |
|---|---|
| 제작 소요 시간 | **즉시** (게임 시간 0초) |
| 제작 큐 | **없음** (시설 건설 큐와 별도) |
| 제작 비용 | **재료만** (M5 MVP는 골드 비용 없음 — 페이즈 2 #2에서 검토 가능) |
| 제작 횟수 제한 | **없음** (재료 충족 시 무제한 반복 가능) |
| 결과물 stack 처리 | personal_equipment·guild_equipment = 인스턴스 1개 추가 / material 결과물(중간재) = 수량 누적 |

**근거**:
- M4 약초상 즉시 회복 패턴과 정합 (게임 시간 미소모)
- 제작 큐 도입 시 시설 건설 큐(1슬롯)와의 정책 충돌 검토 필요 → M5 범위 외
- 페이즈 2 #2 "제작 시간 곡선"은 **재료 수집 시간**으로 해석. 첫 제작 30~45분은 재료 누적 시간

#### 1-3. 제작 결과물 카테고리 분포

본 문서의 레시피 10개의 결과물 카테고리:

| category | 결과물 수 | 비고 |
|---|---|---|
| `personal_equipment` | 4종 | 광부의 단검·녹슨 곡괭이·단단한 갑옷 조각·약초 향낭 |
| `guild_equipment` | 4종 | 깃발 복원·폐광의 유물 조각·약초사 인장·광부의 부적 |
| `material` (중간재) | 2종 | 거친 가죽끈 묶음·연마된 쇳조각 |
| `consumable` | 0종 | 페이즈 1 #1 §8 "일반 소모품 M5 미도입" 정책 준수 |

→ 첫 제작 목표 3개 + 학습 다양화 7개 = 균형 분포

#### 1-4. 해금 정책

| 해금 단계 | 해금되는 레시피 | 해금 트리거 |
|---|---|---|
| 게임 시작 직후 | (없음) | 낡은 대장간 자체가 신뢰도 1단계에서 비활성 (M4 정책 그대로) |
| 신뢰도 2단계 진입 | #1·#2·#4·#5·#7·#8 (6종) | 대장간 [제작 목표 보기] 활성화 시점. 첫 제작 진입점 |
| step 1 완료 (폐광 입구 정찰) | #2 광부의 단검 (`#7 녹슨 곡괭이 머리` 입수) — 사실상 신뢰도 2단계 + step 1 더블 게이트 | step 1 완료가 #7 입수의 유일 경로(주 출처) |
| 신뢰도 3단계 진입 | #3 약초사 인장 + #6 약초 향낭 (약초상 대상 활성화) | 약초 시리즈 4·5번 재료 수급이 안정된 시점 |
| step 6 완료 (폐광 재개방식) | #10 폐광의 유물 조각 | `#10 고대 인장 조각` 입수 = step 6 클라이맥스 보상 |
| 부가 | #11 광부의 부적 (artifact T2) | 신뢰도 2단계 + `#8 폐광의 유물 파편` 첫 입수 시 |

**해금 검증**:
- 신뢰도 1단계에서 제작 0개 → 학습 부담 분산
- 신뢰도 2단계에서 6~7종 레시피 노출 → 학습 폭발 위험 있으나 입력 재료 차이로 자연 분기
- step 1·step 6 단일 게이트 결과물 2종 → 사건 라인 진행과 직결

- M5 MVP는 보편 효과만 적용 권장. 거점 특화는 후속 마일스톤에서 추가 가능

### 6. crafting_recipes 데이터 구조 권고 (페이즈 4 #1 입력)

본 문서는 페이즈 4 #1의 `crafting_recipes` 테이블 (또는 JSONB MVP) 설계 입력으로 다음 컬럼을 권고:

| 컬럼 | 타입 | 의미 |
|---|---|---|
| id | TEXT PK | recipe_id |
| name | TEXT | 결과물 이름 또는 레시피 표시명 |
| description | TEXT | 서사 텍스트 (`flavor_text` 동등) |
| result_item_id | TEXT FK items(id) | 결과물 아이템 |
| result_quantity | INT NOT NULL DEFAULT 1 | 결과물 수량 (M5 MVP는 모두 1) |
| inputs_json | JSONB NOT NULL | `[{"item_id":"...","quantity":N},...]` 입력 재료 배열 |
| unlock_condition_json | JSONB NULLABLE | `{"trust_level":2}` / `{"chain_step":{"chain_id":"settlement_3_pyegwang_reopen","step":1}}` 등 |
| craft_location_id | TEXT NOT NULL DEFAULT 'old_smithy' | M5는 단일 값 'old_smithy' (대장간) |
| created_at | TIMESTAMPTZ DEFAULT now() | |

**JSONB MVP 대안**:
- 신규 테이블 없이 `items.craft_recipe_json` 컬럼에 입력 배열 임베드
- 장점: 마이그레이션 단순. 단점: 동일 결과물의 다중 레시피 표현 불가
- M5 MVP는 1결과물:1레시피 가정 → JSONB MVP 가능. 페이즈 4 #1에서 결정

**중간재 정제(#4·#5) 처리**:
- 결과물도 `material` 카테고리이므로 `result_item_id`는 신규 items 행 가리킴
- 페이즈 4 #1에서 `mat_hide_rough_bundle`·`mat_ore_polished_scrap` 신규 items 행 INSERT 필요 → 페이즈 1 #2의 재료 10종에 + 2종 추가 = **총 12종 items 행**

→ 페이즈 1 #2 §1-1 "재료 10종"에서 12종으로 확장 권고 (페이즈 4 #1 명세 기준)

### 7. 미사용 재료 활용 검증

페이즈 1 #2가 식별한 미사용 재료(첫 제작 목표 3개에서 안 쓰이는) #3 마른 약초·#4 산기슭 버섯의 활용 검증:

→ 페이즈 1 #1 §8 "일반 소모품 M5 미도입" 정책 준수 (consumable 결과물 0종)
