## 개요

M5 시작 거점에서 플레이어가 "재료를 모으고 → 인벤토리에서 확인하고 → 대장간에서 제작하는" 핵심 학습 루프의 **두 화면 UI 컨셉**을 확정한다. 첫째는 인벤토리의 신규 4번째 탭 "재료" 탭이며, 둘째는 마을 방문 시 진입하는 낡은 대장간의 정식 제작 화면이다. 두 화면은 페이즈 1 #1~#3에서 결정된 분류·재료·레시피 데이터를 표시하는 가시면(presentation surface)이며, M4 하겐 stub의 3버튼 임시 인터페이스를 자연스럽게 흡수·진화시킨다.

본 문서가 확정하는 것:
- 인벤토리 재료 탭 sub-filter 정책 (slot 5종 + 정렬 기본값 + 빈 슬롯 전체 보기)
- 재료 카드의 4가지 정보 축 (이름·tier 색·수량·출처 힌트)
- region_exclusive 4종의 시각 차별화 (테두리 + 라벨)
- 재료 → 레시피 점프 정책 ("이 재료가 쓰이는 레시피 N개" 배지 + 클릭 시 대장간 점프)
- 낡은 대장간 제작 화면의 **해금/잠김/충족/부족 4상태** 카드 디자인 (Path of Exile 빨강/초록 차용)
- M4 하겐 3버튼의 진화 매핑 (제작 목표 보기 → 레시피 목록 / 수리 의뢰 → 폐기 또는 흡수 / 재료 힌트 → 인벤토리 점프)
- banner 양자택일 그룹 시각화 (깃발 복원 vs 약초사 인장)
- artifact 시너지 그룹 시각화 (광부의 부적 + 폐광의 유물 조각)
- 제작 결과 피드백 (즉시 토스트 + 활동 로그 + 인벤토리 갱신)
- 인벤토리 ↔ 대장간 양방향 점프 (재료에서 레시피로 / 부족 재료에서 출처 힌트로)
- M4 마을 방문 시간 미소모 정책의 대장간 적용 (제작도 게임 시간 0)

본 문서가 확정하지 않는 것 (후속 위임):
- 화면별 위젯 구성·여백·폰트 크기 — 페이즈 4 명세
- region_exclusive 테두리의 정확한 색·두께 — 페이즈 4 (`AppTheme.settlementAccent` 재사용 권고만 명시)
- 정확한 토스트 텍스트 카피 — 페이즈 4 명세
- 화면 전환 애니메이션 — 페이즈 4 명세

## 상세 설계

### 1. 인벤토리 재료 탭 — 4탭 구조 안의 4번째 탭

#### 1-1. 탭 진입 위치

```
[ 개인 장비 ]  [ 용병단 장비 ]  [ 정수 ]  [ 재료 ]
                                         ▲
                                  M5 신규 4번째 탭
```

페이즈 1 #1 §7-1 4탭 구조를 그대로 적용. 탭은 좌→우 학습 순서(개인 → 용병단 → 영구 강화 → 입력재)를 따른다.

**탭 진입 시 기본 sub-filter**: "전체" 선택. 첫 진입 시 보유 중인 재료 모두 노출.

#### 1-2. sub-filter 정책 — slot 5종 + "전체"

탭 상단에 **6개 칩(chip) 가로 스크롤**:

```
[전체]  [광석]  [가죽]  [약초]  [유물 파편]  [몬스터 부산물]
   ▲      ▲       ▲       ▲          ▲              ▲
  기본  ore    hide    herb     relic_fragment   monster_part
```

| chip | 표시 대상 | M5 더스트빌 기준 보유 가능 재료 (페이즈 1 #2) |
|---|---|---|
| 전체 | 모든 slot | 10종 + 중간재 2종 = 최대 12종 |
| 광석 | `material_ore` | #1 녹슨 쇳조각 / #7 녹슨 곡괭이 머리 / 중간재 #5 연마된 쇳조각 |
| 가죽 | `material_hide` | #2 마른 가죽끈 / #6 빛바랜 천 조각 / 중간재 #4 거친 가죽끈 묶음 |
| 약초 | `material_herb` | #3 마른 약초 / #4 산기슭 버섯 / #5 접착 수액 |
| 유물 파편 | `material_relic_fragment` | #8 폐광의 유물 파편 / #10 고대 인장 조각 |
| 몬스터 부산물 | `material_monster_part` | #9 거대 박쥐 송곳니 |

**빈 slot 처리**: 보유량 0인 slot의 chip은 표시는 되지만 회색 톤. 클릭 시 "아직 입수한 재료가 없습니다" 빈 상태 메시지 + "출처 가이드 보기" 버튼(2-3절 출처 힌트 화면 이동).

**정렬 기본값**:
1. tier desc (T3 → T2 → T1) — 희귀 재료가 먼저
2. 동 tier 내 보유량 desc — 많이 가진 게 먼저
3. 동 보유량 내 id asc — 안정 정렬 (페이즈 4 #2에서 결정)

**정렬 변경 옵션**: M5 MVP는 정렬 변경 UI 없이 위 기본값 고정. M6+에서 정렬 셀렉터 추가 검토.

#### 1-3. 재료 카드 디자인 — 4가지 정보 축

각 재료는 그리드 또는 리스트 형태의 **카드 1개**로 표시된다.

```
┌────────────────────────────────┐
│ ▎ [tier 색 좌측 바]              │
│   재료 이름            🔨 ×3    │
│   slot 한국어 / T2              │
│                                 │
│   "더스트빌 한정"  [라벨]        │
│   [출처 1줄: 폐광 labor 의뢰]    │
│                                 │
│                          ×42    │ ← 보유 수량 (3자리 고정)
└────────────────────────────────┘
```

**4가지 정보 축**:

| 위치 | 정보 | 시각 |
|---|---|---|
| 좌측 바 | tier 색 | T1 회색 / T2 초록 / T3 파랑 (CLAUDE.md 색상 체계 준수) |
| 상단 좌 | 재료 이름 + slot/tier 라벨 | 일반 텍스트 + tier 색 작은 라벨 |
| 상단 우 | 제작 사용 배지 🔨 ×N | M5 MVP 가시 배지: "이 재료가 들어가는 레시피 수" |
| 하단 우 | 보유 수량 ×N | 큰 글자, 3자리 고정 (Melvor 패턴) |
| 카드 외곽 | region_exclusive 표식 | settlementAccent 테두리 1px + "더스트빌" 라벨 (1-4절) |

**카드 펼침** (선택적 — 페이즈 4 #2에서 인라인 펼침 vs 별도 시트 결정):
- 출처 1~3줄 (페이즈 1 #2의 출처 매핑표 §2-2 텍스트 활용)
- "이 재료로 만들 수 있는 레시피" 버튼 → 대장간 레시피 화면 점프 (1-5절)
- description / flavor_text 1~2줄

#### 1-4. region_exclusive 4종 시각 차별화

페이즈 1 #2 §3-2가 식별한 **region_exclusive=3 한정 4종**을 인벤토리에서 일반 재료와 시각 차별화한다.

| # | 재료 | 차별화 시각 |
|---|---|---|
| #5 | 접착 수액 | 카드 테두리 1px settlementAccent(0xFFFFA000) + 좌상단 작은 "더스트빌" 라벨 |
| #7 | 녹슨 곡괭이 머리 | 위와 동일 |
| #8 | 폐광의 유물 파편 | 위와 동일 |
| #10 | 고대 인장 조각 | 위와 동일 + tier T3 파랑 좌측 바와 자연스럽게 어울림 |

**시각 일관성 원칙**:
- `AppTheme.settlementAccent`(0xFFFFA000)는 M4에서 마을 방문·신뢰도·거점 사건 강조에 이미 사용 중. 본 문서는 그 색을 "더스트빌과 직결된 재료"로 확장하여 시각 언어를 통일
- 테두리만으로는 시각 인지 폭이 좁으므로 작은 라벨 텍스트 "더스트빌"을 병기. 후속 마일스톤에서 다른 거점 한정 재료가 늘어나면 라벨 텍스트만 변화 (구조는 동일)
- 영구 활용을 위해 라벨은 다국어 키로 관리 (페이즈 4 #2)

**역효과 회피**:
- region_exclusive는 "더스트빌에서만 얻을 수 있다"라는 출처 정보일 뿐 희귀성을 의미하지는 않음 (페이즈 1 #2 §3-3)
- 따라서 테두리 색은 tier 색과 충돌하지 않게 좌측 바(tier)와 외곽선(region) 두 위치를 분리
- 라벨 "더스트빌"은 출처 힌트와 중복되지 않고 추가 정보 (출처 힌트는 1-3절 펼침 영역)

#### 1-5. "이 재료가 쓰이는 레시피" 점프 정책

각 재료 카드 우상단의 🔨 ×N 배지는 **"이 재료가 들어가는 레시피 수"**를 의미한다. 클릭 시 동작:

| 마을 방문 상태 | 클릭 동작 |
|---|---|
| 마을 방문 미진입 (region 3 sector 1 외부) | 토스트 "낡은 대장간에서 확인할 수 있습니다" + 이동 화면 강조 |
| 마을 방문 진입 (sector 1 village) | 낡은 대장간 [레시피 목록] 화면으로 점프 + **이 재료를 사용하는 레시피만 자동 필터** |

**자동 필터 동작**:
- 대장간 레시피 화면 진입 후 상단에 "필터: 녹슨 쇳조각 사용" 칩 1개 표시 + [필터 해제] 버튼
- 페이즈 1 #3 §4 요약표 기준 #1 녹슨 쇳조각이 들어가는 레시피 = #2 광부의 단검 / #5 연마된 쇳조각 / #6 단단한 갑옷 조각 / #7 녹슨 곡괭이 / #10 광부의 부적 = 5종 노출

**"제작 사용 배지 ×N" 계산 시점**:
- 정적 데이터 변환 (페이즈 4 #2 결정): 앱 시작 시 또는 staticDataProvider 갱신 시 재료별 사용 레시피 수 캐싱
- 동적 갱신 불필요 — recipes는 staticDataProvider이므로 변경 없음

#### 1-6. 출처 힌트 표시 정책 (M4 하겐 stub의 진화)

**M4 하겐 [재료 힌트 보기]** (starting-settlement.md §2.2):
> 더스트플레인 4섹터에서 얻을 수 있는 재료 힌트 텍스트 (폐광=녹슨 쇳조각·낡은 곡괭이 / 마른 초원=마른 가죽끈 / 먼지로 덮인 길=여행자 짐 더미 등)

→ M5에서는 이 텍스트가 **각 재료 카드 펼침 영역의 1~3줄**로 흡수된다. 거점 화면의 임시 텍스트 → 인벤토리의 실제 재료 위치로 정보 이동.

**힌트 텍스트 작성 정책** (페이즈 1 #2 출처 매핑표 §2-2 차용):

| 재료 | 힌트 텍스트 (예시) |
|---|---|
| #1 녹슨 쇳조각 | "주로 폐광 labor 의뢰에서 얻습니다. 가끔 폐광 일반 조사에서도 발견됩니다" |

### 2. 낡은 대장간 제작 UI — M4 stub의 정식화

#### 2-1. M4 하겐 3버튼의 진화 매핑

starting-settlement.md §2.2 낡은 대장간의 M4 stub은 "제작 목표 보기 / 수리 의뢰 확인 / 재료 힌트 보기" 3버튼이었다. M5 정식 활성화 시 다음과 같이 진화한다.

| M4 stub 버튼 | M5 진화 형태 | 진화 사유 |
|---|---|---|
| **제작 목표 보기** | **[레시피 목록]** — 정식 제작 화면의 메인 진입 | M4에서 "재료를 보여주기만 하던" 정보 표시가 M5에서는 "재료 충족 시 클릭 1회 제작"의 액션 화면으로 진화 |
| **수리 의뢰 확인** | **폐기 또는 [추가 의뢰] 보존** | M4 stub은 "1건 골드 보상"이었으나 M5는 정식 제작이 그 자리를 대체. 수리 의뢰는 (a) 완전 폐기 / (b) 재료 보상의 출처로 흡수(대장간 의뢰 풀로 통합) — 본 문서 권고: **(a) 폐기**, 카드 자리만 비움 |
| **재료 힌트 보기** | **[인벤토리 — 재료 탭] 점프 버튼** | 텍스트 4줄을 인벤토리 재료 탭 출처 힌트(§1-6)로 흡수. 대장간에서는 "재료 더 모으러 가기"의 점프 버튼 1개로 단순화 |

**M5 대장간 화면 최종 구성** (3개 버튼 → 2개 버튼 + 메인 영역):

```
┌────────────────────────────────────────────┐
│  낡은 대장간 — 하겐                           │
│                                            │
│  ▎ "(고개를 한 번 끄덕이고 다시 모루를 두드린다)"│
│    또는 신뢰도별 인사말 (페이즈 1 #4 §4.1)   │
│                                            │
│  ─────────────────────────────────────     │
│                                            │
│  [레시피 목록]                    (메인)    │
│  ├ 카드 1 — 낡은 용병단 깃발 (충족)           │
│  ├ 카드 2 — 광부의 단검 (부족)              │
│  ├ ...                                      │
│                                            │
│  ─────────────────────────────────────     │
│                                            │
│  [인벤토리에서 재료 보기]  [돌아가기]         │
└────────────────────────────────────────────┘
```

**버튼 폐기·단순화 의도**:
- M4 stub 3버튼은 "활동 안내"였고 M5는 "활동 실행"이므로 자연 단순화
- "수리 의뢰" 폐기는 페이즈 1 #1 §8 "일반 소모품 M5 미도입" + 페이즈 1 #3 §1-1 "제작 위치 단일" 정책의 자연스러운 귀결 (대장간 = 제작 단일)
- 페이즈 4 #3 명세에서 M4 stub 코드 제거 + M5 제작 화면 신설

#### 2-2. 레시피 카드 4상태

각 레시피는 카드 1개로 노출되며, **4가지 상태** 중 하나로 표시된다.

| 상태 | 시각 | 인터랙션 |
|---|---|---|
| **(a) 잠김 (LOCKED)** | 카드 전체 회색 + 자물쇠 🔒 아이콘 + 해금 조건 텍스트 | 클릭 시 해금 조건만 표시 (제작 불가) |
| **(b) 해금 + 재료 부족 (UNLOCKED-INSUFFICIENT)** | 카드 반투명 60% + 부족 재료 빨강 표기 | 클릭 시 카드 펼침 (부족 재료 출처 힌트 표시) — 제작 버튼 비활성 |
| **(c) 해금 + 재료 충족 (READY)** | 카드 또렷 + 충족 재료 초록 표기 + "제작 가능" 라벨 | [제작] 버튼 활성 → 클릭 1회로 제작 |
| **(d) 제작 완료 + 영구 1회 (CRAFTED)** | M5 MVP는 무제한 반복이므로 미적용 | (참고만 — M6+ 일회성 위업 보상 추가 시 적용) |

**상태 결정 로직** (페이즈 4 #2 `CraftingService` 입력):
1. 해금 조건(`unlock_condition_json`) 평가 → 미충족이면 (a)
2. 입력 재료 보유량 평가 → 부족이면 (b)
3. 모두 충족이면 (c)

#### 2-3. (a) 잠김 카드 — 해금 조건 표시

```
┌────────────────────────────────────────────┐
│ 🔒 [회색 톤 카드]                            │
│   ??? — ??? (T?)                           │
│                                            │
│   해금 조건:                                │
│   • 폐광 재개방 사건 step 6 완료            │
│                                            │
│   (해금되면 폐광의 유물 조각 제작 가능)      │
└────────────────────────────────────────────┘
```

**잠김 카드 표시 정책**:
- 결과물 이름·tier·서사는 **`???` 처리** (스포일러 회피) — 단, 이미 step 6 클라이맥스 보상으로 #10 고대 인장 조각을 입수한 시점에는 자연스럽게 노출
- 해금 조건은 명확하게 표시: "폐광 재개방 사건 step 6 완료" / "마을 신뢰도 3단계 진입" / "폐광의 유물 파편 첫 입수"
- 잠김 카드는 정렬상 **목록 하단**에 배치 (해금된 카드 우선 노출)

**페이즈 1 #3 §1-4 해금 정책에 따른 M5 잠김 분포** (신뢰도 1단계 → 신뢰도 4단계 + step 6 완료까지 진행):

| 신뢰도 | 노출 (해금) | 잠김 |
|---|---|---|
| 1단계 | 0개 | 10개 (대장간 자체 비활성) |
| 2단계 진입 | 6개 (#1·#4·#5·#6) — step 1 미완료 시 #2·#7도 잠김 | 4개 (#3·#8·#9·#10) |
| step 1 완료 후 | 8개 (+#2·#7) | 2개 (#3·#8 — 신뢰도 3단계 미진입 / #10 — step 6 미완료) |
| 신뢰도 3단계 + #8 첫 입수 | 9~10개 (+#3·#8·#9 추가) | 1개 (#3 폐광의 유물 조각 — step 6 미완료) |
| step 6 완료 | 10개 모두 해금 | 0개 |

**참고**: #10 광부의 부적은 "#8 폐광의 유물 파편 첫 입수"가 해금 조건 (페이즈 1 #3 §3-7). #8 입수 시점이 step 4 또는 폐광 숨겨진 발견이므로 step 1보다는 늦지만 step 6보다는 빠름.

#### 2-4. (b) 재료 부족 카드 — 빨강 표기 + 출처 힌트

```
┌────────────────────────────────────────────┐
│ [반투명 60% 카드 — 비활성]                   │
│   광부의 단검 — STR +2 / AGI +1            │
│   personal_equipment / weapon / T2         │
│                                            │
│   필요 재료:                                │
│   • 녹슨 쇳조각      [초록] 5/3   ✓        │
│   • 마른 가죽끈      [초록] 8/1   ✓        │
│   • 녹슨 곡괭이 머리  [빨강] 0/1   ✗        │
│                                            │
│   [제작]  ← 비활성                          │
│                                            │
│   ▎부족: 폐광 입구 정찰 보상에서 입수 가능    │
│                                            │
└────────────────────────────────────────────┘
```

**색상 규칙** (Path of Exile 패턴):
- 충족 재료: 초록 — 보유 ≥ 필요 — `✓` 아이콘 병기
- 부족 재료: 빨강 — 보유 < 필요 — `✗` 아이콘 병기
- 회색은 사용하지 않음 (잠김 카드와 시각 충돌 회피)
- X/Y 형식: 보유/필요 (Path of Exile 동일)

**부족 재료 출처 힌트 펼침**:
- 카드 클릭 시 카드 하단에 부족 재료 1~3개의 출처 힌트 1줄씩 표시
- 텍스트는 §1-6 인벤토리 재료 탭 출처 힌트와 **동일 데이터 소스** 사용 (이중 작성 방지)
- 출처 힌트 1줄 우측에 [인벤토리에서 보기] 작은 텍스트 링크 → 인벤토리 재료 탭 점프 (해당 재료 카드로 스크롤)

**[제작] 버튼 비활성**:
- 비활성 상태 시각: 회색 톤 + 클릭 무반응
- 호버(터치 길게) 시 "재료 부족" 토스트 1초

**페이즈 1 #3 §3-5의 차용**:

#### 2-7. 제작 결과 피드백 — 즉시 토스트 + 활동 로그

페이즈 1 #3 §1-2 즉시 제작 정책에 맞는 짧은 연출.

| 단계 | 시각 |
|---|---|
| 1. [제작] 버튼 클릭 | 버튼 일시 비활성 (50ms) |
| 2. 입력 재료 차감 | 카드 내 재료 X/Y 즉시 갱신 (예: 5/3 → 2/3) |
| 3. 결과물 인벤토리 추가 | InventoryItem 생성 또는 수량 증가 (M2a 모델) |
| 4. 토스트 표시 | 화면 하단 1.5초 "낡은 용병단 깃발 제작 완료 ✨" |
| 5. 활동 로그 기록 | `ActivityLogType` 신규 enum 추가 (페이즈 4 #2) — 예: `craftCompleted` |
| 6. 카드 상태 갱신 | 재료 차감 후 충족 → 부족 전이 시 카드 재계산 (즉시) |

**페이즈 4 #2 위임**:
- `ActivityLogType` 다음 HiveField 27 사용 (CLAUDE.md typeId 6 enum, 다음 HiveField 27)
- 토스트 카피 정확한 텍스트
- 결과물 추가 후 카드 갱신 정렬 (충족 카드 → 카드 자체는 그대로 유지, 재료만 X/Y 갱신)

**M4 약초상 즉시 회복 패턴 차용**:
- M4의 약초상 즉시 회복 토스트와 동일 디자인 패턴 — UI 일관성
- 약초상은 게임 시간 미소모 + 골드 결제 + 쿨다운 / 대장간 제작은 게임 시간 미소모 + 재료 결제 + 쿨다운 없음
- 차이점: 약초상의 24시간 쿨다운 같은 후속 디스인센티브 없음 (재료가 자연 디스인센티브)

#### 2-8. 시간 미소모 정책 — M4 마을 방문 원칙 계승

페이즈 1 #3 §1-2가 명시한 "즉시 제작 — 게임 시간 0초" 정책은 starting-settlement.md §"마을 방문 시간 미소모 원칙"의 자연 계승이다.

| 활동 | 게임 시간 | 결제 |
|---|---|---|
| 약초상 즉시 회복 (M4) | 0 | 골드 + 쿨다운 |
| 촌장 보상 받기 (M4) | 0 | (없음 — 누적 신뢰도 임계값) |
| **대장간 제작 (M5)** | **0** | **재료 + 해금 조건** |

**플레이어 인지**:
- 대장간 진입 시 "여기서는 시간이 흐르지 않아요"의 명시적 안내는 불필요 (M4에서 학습된 패턴)
- 대신 [제작] 버튼 호버 시 "재료를 소비하여 즉시 제작합니다" 같은 짧은 툴팁 1줄

#### 2-9. 정렬 정책 — 4상태 우선순위 + 그룹 우선

레시피 목록 카드 정렬:

1. **(c) 충족 → (b) 부족 → (a) 잠김** (상태 우선)
2. 동 상태 내 **slot 그룹** (banner / weapon / armor / accessory / artifact / material 정제) — 그룹 헤더(2-6절)
3. 동 그룹 내 **결과물 tier desc** (T3 → T2)
4. 동 tier 내 **recipe_id asc** (안정 정렬)

**M5 신규 유저 첫 진입 시 (신뢰도 2단계 진입 직후)**:
- 충족: 0~1개 (이미 입수한 재료에 따라)
- 부족: 5~6개
- 잠김: 4개

→ 학습 부담 분산: 충족 카드 1개 위에 부족 카드 5~6개 + 잠김 4개. 첫 클릭이 [제작]에 자연스럽게 도달.

### 3. 인벤토리 ↔ 대장간 양방향 점프

#### 3-1. 점프 시나리오 4가지

| 시나리오 | 출발 | 도착 | 컨텍스트 전달 |
|---|---|---|---|
| (1) 인벤토리 → 대장간 | 재료 카드 [🔨 ×N 배지] | 대장간 [레시피 목록] | "이 재료 사용 레시피만 필터" 자동 적용 |
| (2) 대장간 → 인벤토리 | 부족 재료 [인벤토리에서 보기] 링크 | 인벤토리 [재료 탭] | 해당 slot chip 자동 선택 + 재료 카드로 스크롤 |
| (3) 대장간 → 출처 힌트 | 부족 재료 텍스트 | 대장간 카드 펼침 영역 (출처 1줄) | 화면 이동 없음 — 인라인 펼침 |
| (4) 인벤토리 빈 상태 → 출처 가이드 | 빈 slot chip의 [출처 가이드 보기] 버튼 | 인벤토리 인라인 시트 — 5종 slot 출처 가이드 | M4 재료 힌트 보기와 동일 정보 (정적 텍스트) |

#### 3-2. 거리 정책 — 마을 방문 진입 게이트

**중요**: 인벤토리 재료 탭의 "🔨 ×N → 대장간 점프"는 **마을 방문 진입 상태**가 전제다.

| 마을 방문 상태 | 클릭 동작 |
|---|---|
| (a) 마을 방문 진입 (region 3 sector 1 village) | 대장간 화면으로 즉시 점프 |
| (b) 마을 방문 미진입 (다른 region/sector) | 토스트 "낡은 대장간에서 확인할 수 있습니다 — 마을 방문 시 진입 가능" 1.5초 |

**인지 부담 회피**:
- 배지 자체는 항상 보이게 함 (정보 가치 = "이 재료 쓰임새 있음")
- 클릭 동작만 마을 진입 상태에 따라 분기 (행동 가능성 분기)
- M5+ 다중 거점 도입 시: 거점별 대장간이 다르면 점프 우선순위 결정 정책 추가 필요. M5 단일 거점 시점에는 일관

#### 3-3. M2a 인벤토리 박스 + 장비 슬롯 호환

페이즈 1 #1 §"Hive 박스 영향" + 페이즈 1 #3 §"Hive 박스 영향"이 명시한 정책 재확인:

| 항목 | M2a 신설 예정 | M5 추가 |
|---|---|---|
| `inventory` 박스 (Hive) | M2a에서 신설 | M5는 같은 박스에 category=material 행 추가만 |
| `Mercenary` 장비 슬롯 (weapon/armor/helmet/boots/accessory) | M2a 신설 | M5 결과물 #2·#6·#7·#9가 장착 대상 |
| `UserData` banner/artifact 슬롯 | M2a 신설 | M5 결과물 #1·#3·#8·#10이 장착 대상 |
| 인벤토리 4탭 UI | M2a에서 3탭(개인장비/용병단장비/정수) | **M5에서 재료 탭 추가 → 4탭 완성** |

**M2a vs M5 진행 순서 가정**:
- M2a 완료 후 M5 진입 — 본 문서가 가정하는 표준 케이스
- M2a-M5 동시 진행 시: 4탭 UI를 M5에서 한 번에 신설 가능 (페이즈 4 #2에서 자율 판단)
- M5가 M2a에 선행 진입 시: 본 문서 범위 외 (마일스톤 순서 우선)

**페이즈 4 #2 명세 권고**:
- 인벤토리 화면은 `IndexedStack` + `TabBar` 4탭 구조
- 각 탭은 별도 위젯 (PersonalEquipmentTab / GuildEquipmentTab / EssenceTab / **MaterialTab** 신규)
- MaterialTab 내부에 sub-filter chip 6종 + 재료 카드 그리드
- 카드 펼침은 인라인 ExpansionTile vs 별도 BottomSheet — 페이즈 4 #2 결정

### 5. 특수 케이스 처리

#### 5-1. 빈 인벤토리 (재료 0종 보유) — 신뢰도 1단계 직후

```
[재료 탭]
  [전체]  [광석]  [가죽]  [약초]  [유물 파편]  [몬스터 부산물]
  ─────────────────────────────────────────────────
  
  아직 입수한 재료가 없습니다.
  
  더스트빌의 의뢰·조사·이동 선택지에서 재료를 얻을 수 있습니다.
  
  [출처 가이드 보기]
```

**[출처 가이드 보기] 인라인 시트** (페이즈 1 #2 §2-1 출처 매핑표 차용):

```
🔨 광석 — 폐광 labor 의뢰, 폐광 일반 조사
🐺 가죽 — 마른 초원 hunt 의뢰, 도적 의뢰
🌿 약초 — 마른 초원 채집 의뢰, 약초상 채집 의뢰 (신뢰도 2 이후)
🏛️ 유물 파편 — 폐광 숨겨진 발견, 폐광 사건 단계 보상
🦴 몬스터 부산물 — 거대 박쥐 등 엘리트 처치 보상
```

#### 5-2. 빈 대장간 (잠김 100%) — 신뢰도 1단계

```
[낡은 대장간]
  하겐 — "(고개를 한 번 끄덕이고 다시 모루를 두드린다)"
  
  대장장이가 아직 일거리를 주지 않았습니다.
  마을 신뢰도를 쌓으면 제작이 시작됩니다.
  
  [돌아가기]
```

**M4 starting-settlement.md §2.2 단계 1 인사말 그대로**: 대장간 활성 자체가 신뢰도 2단계 진입 트리거. M5에서도 이 잠금 정책 그대로 유지.

#### 5-3. 인벤토리 재료 999 도달 시

페이즈 1 #1 §6-1 stack_max 999:

```
재료 카드 우하단 ×999 (3자리 고정)
─ 추가 입수 시도 시 토스트 "재료 보유량이 가득합니다 (999/999)" 1초
```

**대처 정책** (M5 MVP):
- 999 초과 시 더 이상 입수 안 됨 (드랍 무시)
- 활동 로그에 "재료 보유량 가득으로 X 1개 폐기" 1행
- M5 시점에 첫 장비 완성에 30~50개 누적 예상이므로 999 도달 가능성 낮음 (페이즈 1 #1 §6-3 근거)

#### 5-4. 제작 직후 충족 → 부족 전이

```
1. 깃발 복원 카드 충족 (재료 8/3 / 4/2 / 1/1)
2. [제작] 클릭
3. 재료 차감 (5/3 / 2/2 / 0/1)
4. → 빛바랜 천 조각 0/1 → 카드 자동 (b) 부족 상태 전이
5. 토스트 "낡은 용병단 깃발 제작 완료 ✨"
6. 카드 정렬 재계산 — 충족 → 부족 그룹 이동 (시각적 카드 위치 변화)
```

**시각 정책**:
- 카드 위치 변화 애니메이션은 M5 MVP 미적용 — 단순 즉시 재정렬 (페이즈 4 #2 결정)
- 단, 재료 X/Y 갱신은 즉시 (카드는 그대로 유지하면서 숫자만 변화)

#### 5-5. 1회성 재료 #6 빛바랜 천 조각·#10 고대 인장 조각 처리

페이즈 1 #2 §1-2가 식별한 **1회성 입수 재료** 처리:

| 재료 | 1회성 입수 | 재획득 경로 |
|---|---|---|
| #6 빛바랜 천 조각 | 신뢰도 2단계 진입 보상 1회 | 광장 잡동사니 허드렛일 (드물게) — 페이즈 1 #2 §2-2 |
| #10 고대 인장 조각 | step 6 클라이맥스 보상 1회 | 폐광 최심부 숨겨진 발견 (재시작 후 reset 시) |

**UI 표시 정책**:
- 인벤토리 재료 탭에 0개 보유 시 카드 자체 비표시 (다른 재료와 동일)
- 페이즈 1 #4 본 문서는 **"이 재료는 1회성"이라는 시각 표식 없음** 권고 — 플레이어 학습으로 자연 인지
- 만약 후속 마일스톤에서 1회성 재료가 늘어나면 별도 라벨 신설 검토
