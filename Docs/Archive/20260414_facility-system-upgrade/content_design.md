# 시설 시스템 고도화 기획서

> 작성일: 2026-04-14
> 유형: 고도화
> 상태: 기획 확정

---

## 개요

현재 4종 시설(훈련소, 의무실, 주둔지, 정보망)은 최대 Lv5의 수동 보너스만 제공하며, 상한 도달 후 상호작용할 이유가 없다. 이를 **12종 시설, 최대 Lv25, 건설 시간 도입, 기능 해금 이정표** 구조로 확장하여 시설을 게임 전반의 장기 성장축이자 전략적 의사결정 포인트로 격상한다.

**기대 효과:**
- "어떤 시설을 먼저 올릴까?"라는 상시 전략적 선택
- 레벨업마다 수치 향상 + 주요 레벨에서 새 기능 해금 → 지속적 동기 부여
- 기획된 컨텐츠(트레잇 Phase B, 세계 확장, 아이템 시스템)의 자연스러운 해금 게이트

**핵심 감성:** OGame의 건물 시스템 — 레벨이 올라갈수록 비용과 시간이 기하급수적으로 증가하지만, 그만큼 강력한 효과와 새로운 기능이 열린다.

## 시스템 구조

### 핵심 규칙

| 항목 | 설계 |
|------|------|
| 시설 수 | 12종 (기존 4종 재설계 + 신규 8종) |
| 최고 레벨 | 25 |
| 건설 큐 | **1개** — 한 번에 하나의 시설만 건설/업그레이드 가능 |
| 건설 중 효과 | 건설 중에도 해당 시설의 **현재 레벨 효과는 유지**. 완료 시 새 레벨 적용 |
| 효과 스케일링 | 로그 스케일 수치 증가 + 주요 레벨에서 기능 해금 |
| 건설 시간 단축 | 향후 용병 투입 시스템으로 확장 예정 |

### 건설 시간 공식

초반 온보딩을 위해 Lv1~2는 고정값, Lv3부터 지수 공식을 적용한다.

```
Lv1:     5분 (고정)
Lv2:     10분 (고정)
Lv3~25:  25 × 1.45^(레벨-3) 분
```

| 레벨 | 건설 시간 | 체감 구간 |
|------|----------|----------|
| 1 | 5분 | 즉시 체험 |
| 2 | 10분 | |
| 3 | 25분 | 일상 플레이 |
| 4 | 36분 | |
| 5 | 52분 | |
| 7 | 1시간 50분 | |
| 10 | 5시간 37분 | 수면/외출 중 건설 |
| 12 | 11시간 49분 | |
| 14 | 1일 | 일 단위 대기 |
| 17 | 3.1일 | |
| 20 | 9.6일 | 주 단위 대기 |
| 23 | 29일 | 월 단위 대기 |
| 25 | 61일 (~2개월) | 최종 목표 |

**체감 구간:**
- **Lv1~2**: 온보딩. 시설 시스템을 빠르게 체험
- **Lv3~10**: 일상 플레이. 수 시간 단위로 성장 체감
- **Lv11~17**: 건설 큐 1개의 우선순위 판단이 본격화
- **Lv18~25**: 장기 목표. 용병 투입 시간 단축 기능이 빛나는 구간

### 골드 비용 공식

건설 시간과 동일한 구조. 시설별 기본비용은 차등 적용한다.

```
Lv1~2:   시설별 고정값
Lv3~25:  기본비용 × 비용배율^(레벨-3)
```

시설별 기본비용/배율은 `/balance-designer`로 별도 조정한다. 아래는 방향성 참고용 초안이다.

| 시설 그룹 | 기본비용 방향 | 근거 |
|----------|-------------|------|
| 핵심 시설 (훈련소, 주둔지, 의무실) | 중간 | 모든 플레이어에게 필수 |
| 정보/탐색 시설 (정보망, 연구소, 게시판) | 중~높음 | 정보 = 고가치 |
| 경제 시설 (금고, 대장간) | 높음 | 골드 순환에 직접 영향 |
| 전투/이동 시설 (방어시설, 야전병원, 이동수단) | 중간 | 전투 효율 개선 |
| 생활 시설 (주점) | 낮~중간 | 초반 접근성 보장 |

### 효과 스케일링

수치 효과는 **로그 스케일**로 증가한다. 초반 레벨은 체감이 크고, 후반 레벨은 미세한 차이를 위한 투자.

```
효과(레벨) = 최대효과 × ln(1 + 레벨 × α) / ln(1 + 25 × α)
```

α는 곡선의 초반 가파름을 조절하는 파라미터. `/balance-designer`로 시설별 최적값 도출.

**예시 — 훈련소 XP 보너스 (최대효과 80%, α = 0.5):**

| 레벨 | XP 보너스 |
|------|----------|
| 1 | +10% |
| 5 | +30% |
| 10 | +48% |
| 15 | +60% |
| 20 | +70% |
| 25 | +80% |

기능 해금은 특정 레벨에 고정된 이정표로, 수치와 별도로 새로운 게임플레이를 개방한다.

## 시설 상세 설계

### 기존 시설 (4종 재설계)

#### 1. 훈련소

> 용병의 성장을 가속하는 시설

| 항목 | 내용 |
|------|------|
| 수치 효과 | 퀘스트 완료 시 XP 보너스 +10% → +80% (로그) |
| 핵심 역할 | 경험치 가속 + 훈련 배치를 통한 트레잇 촉발 (Phase B) |

**기능 해금 이정표:**

| 레벨 | 해금 기능 | 설명 |
|------|----------|------|
| Lv5 | 훈련 배치 | 파견하지 않는 용병을 훈련소에 배치. 시간 경과 → 훈련 관련 행동 지표 누적 → 트레잇 획득 조건 충족 촉발. 트레잇 Phase B의 핵심 메커니즘 |
| Lv10 | 집중 훈련 | 특정 스탯(ATK, DEF 등) 집중 훈련 모드. 해당 계열 행동 지표 가중 누적 |
| Lv20 | 동시 훈련 슬롯 +1 | 기본 1명 → 2명 동시 훈련 배치 가능 |

**훈련 배치 메커니즘 (Lv5 해금):**

```
훈련소에 용병 1명 배치 → 시간 경과 (방치형)
→ 완료 시 훈련 관련 행동 지표(training_count, focused_training 등) 누적
→ TraitAcquisitionService가 조건 체크 → 트레잇 자동 획득
```

- 배치 중 용병은 퀘스트 파견 불가
- 훈련 1회 소요 시간: 기본 10분 (연구소 레벨로 효율 개선 가능)
- 지역 조사와 독립된 별도 슬롯

| 수치 효과 | 방치형 보상 상한 +60G → +960G (로그). 기본 480G에 합산 |

**기능 해금 이정표:**

|------|----------|------|
| Lv5 | 이자 시스템 | 보관 중인 골드에 일정 비율 이자 부여 (접속 시 정산, 상한 있음) |

| Lv20 | 건설비용 할인 | 시설 업그레이드 골드 비용 10% 할인 |

## 건설 시간 단축 — 용병 투입 (향후 확장)

Lv18 이상 구간에서 건설 시간이 주~월 단위로 증가하므로, 용병을 건설에 투입하여 시간을 단축하는 메커니즘을 향후 추가한다.

**기본 구상:**
- 건설 중인 시설에 유휴 용병을 배치
- 배치된 용병의 스탯(STR 등)에 비례하여 건설 시간 단축
- 투입 중인 용병은 파견/훈련/조사 불가
- OGame의 로봇 공장/나노 공장 역할을 용병이 대체

**상세 설계는 별도 기획으로 진행한다.**

## UI 설계

### 시설 화면 재구성

현재 설정 탭 내 시설 관리를 **독립 화면 또는 홈 화면의 주요 섹션**으로 격상한다.

```
시설 관리 화면
├── 건설 큐 상태 바 (상단 고정)
│   "훈련소 Lv5 → Lv6 건설 중 — 잔여 52분"
│   [건설 취소] 버튼
│
├── 시설 그리드 (2열 또는 리스트)
│   ┌─────────────┐ ┌─────────────┐
│   │ 훈련소 Lv5  │ │ 의무실 Lv4  │
│   │ XP +30%     │ │ 회복 -28%   │
│   │ [업그레이드] │ │ [업그레이드] │
│   └─────────────┘ └─────────────┘
│   ...
│
└── 시설 상세 (탭 시 확장)
    ├── 현재 효과 상세
    ├── 다음 레벨 효과 미리보기
    ├── 해금 이정표 타임라인 (달성/미달성 구분)
    ├── 업그레이드 비용 + 소요 시간
    └── [업그레이드 시작] 버튼
```

### 건설 큐 표시

- 홈 화면에 건설 진행 상태 미니 위젯 표시
- 건설 완료 시 팝업 알림 + 활동 로그 기록
- 건설 큐가 비어 있을 때 "건설 대기 중" 안내 (업그레이드 유도)

## 기존 시스템과의 연관

### 영향받는 시스템

| 기존 시스템 | 변경 내용 | 영향도 |
|-----------|----------|--------|
| UserData (Hive) | `facilities` Map 확장 (4종 → 12종), 건설 큐 상태 추가 | 중 |
| FacilityService | 12종 시설 대응, 로그 스케일 효과 계산, 건설 시간 공식 | 높음 |
| FacilityScreen | 전면 재설계 (그리드/리스트, 건설 큐, 이정표 타임라인) | 높음 |
| QuestCompletionService | 훈련소/야전병원/게시판 효과 반영 | 중 |
| MovementProvider | 이동수단 시간 단축 효과 반영 | 낮음 |
| TravelEventService | 방어시설/이동수단 이벤트 확률 보정 | 낮음 |
| RecruitmentService | 주점 모집 품질/필터 기능 반영 | 중 |
| IdleRewardService | 금고 방치형 보상 상한 반영 | 낮음 |
| HomeScreen | 건설 큐 미니 위젯 추가 | 낮음 |

### Supabase 데이터 변경

`facilities` 테이블 확장:

| 현재 | 변경 후 |
|------|--------|
| 4행 (training, infirmary, barracks, intelligence) | 12행 (8종 추가) |
| max_level: 3~5 | max_level: 25 (전 시설 통일) |
| costs: List<int> (레벨별 고정) | base_cost, cost_multiplier 컬럼 추가 (공식 기반) |
| values: List<double> (레벨별 고정) | max_effect, alpha 컬럼 추가 (로그 공식 기반) |

**신규 컬럼:**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| base_cost | int | 골드 비용 공식의 기본값 |
| cost_multiplier | double | 골드 비용 배율 |
| base_time | int | 건설 시간 공식의 기본값 (분) |
| time_multiplier | double | 건설 시간 배율 |
| max_effect | double | 수치 효과 최대값 |
| alpha | double | 로그 곡선 파라미터 |
| milestones | JSONB | 기능 해금 이정표 목록 `[{level, type, description}]` |

### Hive 변경

`user` 박스의 UserData에 건설 큐 필드 추가:

```
UserData {
  ...
  Map<String, int> facilities          // 기존 — 12종으로 확장
  String? constructionFacilityId       // 신규 — 건설 중인 시설 ID
  DateTime? constructionStartTime      // 신규 — 건설 시작 시각
  DateTime? constructionEndTime        // 신규 — 건설 완료 예정 시각
}
```

### 호환성

- 기존 플레이어의 시설 레벨(Lv0~5)은 그대로 유지
- 신규 8종 시설은 초기 Lv0으로 시작
- 기존 `costs`/`values` 배열 방식 → 공식 기반으로 전환. 마이그레이션 필요
- 기존 시설 효과 적용 코드(`QuestCompletionService`, `MercenaryProvider` 등)를 새 로그 공식으로 교체

## 구현 우선순위 제안

### Phase 1: 핵심 인프라 (높음)

건설 시간 시스템 + 기존 4종 시설 Lv25 확장

- 건설 큐 모델, 타이머, UI
- 기하급수 비용/시간 공식 적용
- 로그 스케일 효과 전환
- 기존 4종 시설의 Lv6~25 효과 적용
- 시설 화면 재설계

### Phase 2: 신규 시설 기본 (높음)

8종 신규 시설 추가 + 수치 효과

- Supabase `facilities` 테이블 8행 추가
- 각 시설의 수치 효과(패시브 보너스) 구현
- 시설 화면에 12종 표시

### Phase 3: 기능 해금 이정표 (중)

시설별 주요 레벨 해금 기능 구현

- 훈련소 훈련 배치 (Phase B 연계)
- 주점 모집 필터
- 의무실 응급치료/사망 방지
- 기타 해금 기능 순차 구현

### Phase 4: 건설 시간 단축 (낮음)

용병 투입 시간 단축 메커니즘

- 별도 상세 기획 필요
- Lv18 이상 구간이 의미 있어지는 시점에 도입

## 밸런스 고려사항

- 시설별 기본비용, 비용배율, 효과 수치는 이 기획서에서 방향성만 제시하며, `/balance-designer`로 시뮬레이션 후 확정한다
- 12종 시설의 골드 소비 총량이 현재 경제 시스템의 골드 공급과 균형을 이루는지 검증 필요
- 기능 해금 이정표가 게임 진행 속도와 맞는지 (너무 이른 해금 / 너무 늦은 해금) 플레이테스트 필요
- 건설 큐 1개 제한 하에서 12종 시설의 평균 레벨 진행 속도 시뮬레이션 권장
