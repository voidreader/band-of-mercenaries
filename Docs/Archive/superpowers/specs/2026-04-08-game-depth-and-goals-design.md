# 게임 깊이 및 장기 목표 시스템 설계

## 개요

MVP 핵심 루프(용병 모집 → 이동 → 퀘스트 파견 → 결과)의 깊이를 강화하고, 장기 플레이 동기를 부여하는 4개 시스템을 추가한다.

**접근법**: 상위 3개 시스템은 깊이 있게, 명성/등급은 기본 골격만 구현.

**범위 외**: 퀘스트 후속 효과, 이동 선택지 이벤트, 서사/보상 다양화는 이번 개발에서 제외.

---

## 시스템 1: 이동 이벤트 (자동 조우)

### 목적

이동 구간에 변화를 줘서 "그냥 기다리기"가 아닌 "무슨 일이 있었는지" 확인하는 재미를 추가한다.

### 발생 규칙

- 이동 시작 시 이벤트 발생 여부와 종류를 미리 판정 (출발 시 롤)
- 이동 거리에 비례하여 발생 확률 증가: 거리 1당 15%, 최대 80%
- 한 번 이동에 최대 1개 이벤트
- 날씨 이벤트는 출발 시 판정되어 이동 시간에 반영됨
- 그 외 이벤트는 이동 완료 시점에 "이동 중 이런 일이 있었다"로 결과 표시 (이동을 멈추지 않음)

### 이벤트 종류

| 타입 | 예시 | 효과 |
|------|------|------|
| 발견 | 금화 주머니 발견 | +골드 |
| 습격 | 산적 습격 | 50% 확률로 -골드, 50% 확률로 랜덤 용병 1명 부상 |
| 날씨 | 폭풍으로 지연 | 이동시간 +20~50% (출발 시 반영) |
| 행운 | 떠돌이 상인과 거래 | +골드 or 용병 피로 회복 |
| 조우 | 다른 용병단과 조우 | +명성(소량) |

### 데이터

- `TravelEvent.json`: 정적 이벤트 풀
- 지역 티어별 이벤트 필터링 (Tier1: 가벼운 이벤트, Tier5: 위험한 이벤트)
- 각 이벤트에 `tier_range`, `type`, `effect` (골드/상태/명성/시간), `magnitude`, `description` 필드

### UI

- 이동 완료 시 결과 다이얼로그에 이벤트 내용 포함
- 예: "이동 완료 — 도중에 산적의 습격을 받아 골드 30을 잃었다"
- 이벤트가 없었으면 기존처럼 조용히 이동 완료

### 기존 시스템 영향

- `movement_provider`: 이동 완료 처리 시 이벤트 판정 로직 추가
- `MovementRepository`: 이벤트 결과를 유저 데이터에 반영 (골드, 용병 상태)
- `home_screen`: 이동 완료 다이얼로그에 이벤트 표시 추가

## 시스템 2: 파견 리스크-리워드

### 목적

파견 인원수 선택에 전략적 무게를 부여한다. 적게 보내면 수익이 크지만 위험하고, 많이 보내면 안전하지만 수익이 줄어든다.

### 수익 분배 (인건비)

- 퀘스트 총 보상에서 용병 인건비를 차감한 것이 순수익
- 용병 단가 = 티어 기반
  - T1: 10G, T2: 25G, T3: 50G, T4: 100G, T5: 200G
- **순수익 = 총 보상 - 파견 용병 단가 합산**
- 순수익이 음수가 될 수 있음 (고티어 용병을 저난이도에 보내면 손해)

### 파견 비용 (선불)

- 퀘스트 파견 시 선불 비용 발생, 실패해도 반환 안 됨
- 난이도별 기본 비용:
  - D1: 20G, D2: 50G, D3: 100G, D4: 200G, D5: 400G
- 무분별한 파견 방지 역할

### 위험도

- 기존 성공률 공식 그대로 활용
- 인원이 적으면 → 총 전투력 감소 → 성공률 하락 → 실패/대실패 확률 증가
- 별도의 새 공식 없이 인원 선택의 무게가 자연스럽게 커짐

### 데이터

- `Difficulty.json`에 `dispatch_cost` 필드 추가
- `Job.json`에 이미 tier 정보 있으므로 인건비 계산에 활용
- 인건비 단가는 `MercenaryWage.json`으로 관리 (티어별 단가)

### UI 변경

- 파견 화면에 예상 수익 표시:
  - 총 보상 ~300G / 인건비 -150G / 파견비용 -50G / **예상 순수익 ~100G**
- 성공률과 함께 표시하여 리스크-리워드 판단 가능
- 순수익이 낮거나 음수일 때 경고 색상 표시

### 기존 시스템 영향

- `quest_calculator.dart`: 보상 계산에 인건비 차감 로직 추가
- `quest_provider`: 파견 시 선불 비용 차감
- `dispatch_screen.dart`: 예상 순수익 및 파견 비용 표시

## 시스템 3: 성장 시스템 (혼합형)

### 목적

용병 개인 성장으로 애착을 만들고, 조직 성장으로 영구적 진행감을 제공한다.

### A. 개인 성장 — 용병 경험치/레벨업

**경험치 획득**:
- 퀘스트 완료 시 파견된 용병에게 경험치 부여
- 경험치량 = 퀘스트 난이도 레벨 × 기본XP(20) × 결과 보정
- 결과 보정: 대성공 2.0, 성공 1.0, 실패 0.5, 대실패 0

**레벨업 테이블**:

| 레벨 | 필요 누적 XP | 능력치 보너스 |
|------|-------------|--------------|
| 1 (기본) | 0 | 없음 |
| 2 | 100 | ATK/DEF/HP +10% |
| 3 | 350 | ATK/DEF/HP +20% |
| 4 | 850 | ATK/DEF/HP +30% |
| 5 (최대) | 1,850 | ATK/DEF/HP +40% |

- 능력치 보너스는 기본 스탯 기준 비례 증가 (중첩이 아닌 기본값 대비)
- **죽으면 레벨/경험치 전부 소멸** → 고레벨 용병의 죽음이 뼈아픔
- 고티어 + 고레벨 = 핵심 자산, 저티어 = 소모적 운영 가능

### B. 조직 성장 — 용병단 시설

골드를 투자하여 시설을 업그레이드. 시설은 **영구적** (용병이 죽어도 유지).

| 시설 | 효과 | 최대 단계 | 업그레이드 비용 |
|------|------|-----------|----------------|
| 훈련소 | 획득 경험치 +10%/단계 | 5 | 500 → 1,000 → 2,000 → 4,000 → 8,000G |
| 의무실 | 부상 회복시간 -10%/단계 | 5 | 300 → 600 → 1,200 → 2,400 → 4,800G |
| 주둔지 | 최대 보유 용병 수 +2/단계 (기본 10명) | 5 | 400 → 800 → 1,600 → 3,200 → 6,400G |
| 정보망 | 퀘스트 생성 시 +1개/단계 | 3 | 1,000 → 3,000 → 9,000G |

### 데이터

- 용병 모델에 `xp`, `level` 필드 추가 (Hive 저장)
- `Facility.json`: 시설 정보 (이름, 효과 타입, 단계별 비용, 단계별 수치)
- 유저 데이터에 시설 레벨 맵 저장: `{ "training": 2, "infirmary": 1, ... }`

### UI

- 용병 카드에 레벨 배지 + 경험치 진행바 표시
- 홈 화면 또는 설정 탭을 시설 관리 화면으로 활용 (설정은 현재 스텁)
- 시설 업그레이드 확인 다이얼로그: 비용, 현재→다음 효과 표시

### 기존 시스템 영향

- `mercenary_model`: xp, level 필드 추가, 능력치 계산에 레벨 보너스 반영
- `quest_provider`: 퀘스트 완료 시 경험치 분배 로직 추가
- `mercenary_provider`: 레벨업 판정 로직
- `recruitment_service`: 주둔지 시설에 따른 최대 용병 수 제한 반영
- `quest_generator`: 정보망 시설에 따른 퀘스트 생성 개수 반영
- `settings_screen` → 시설 관리 화면으로 전환 (또는 새 탭)

## 시스템 4: 명성/등급 (기본 골격)

### 목적

장기 플레이 동기 부여. "다음 등급"을 향한 목표감 제공 및 지역 해금 게이트.

### 명성 획득

- 퀘스트 성공: 난이도 레벨 × 10 명성
- 퀘스트 대성공: 난이도 레벨 × 20 명성
- 이동 이벤트(조우 타입): +5~15 명성
- 실패: 명성 변동 없음

### 등급 테이블

| 등급 | 이름 | 필요 명성 | 해금 효과 |
|------|------|-----------|-----------|
| F | 무명 | 0 | 시작 상태, Tier1 지역만 |
| E | 신출내기 | 500 | Tier2 지역 해금 |
| D | 일반 | 2,000 | Tier3 지역 해금 |
| C | 숙련 | 8,000 | Tier4 지역 해금 |
| B | 정예 | 25,000 | Tier5 지역 해금 |
| A | 전설 | 80,000 | 최종 등급 (향후 확장 예정) |

### 이번 구현 범위

- 명성 포인트 누적 및 저장
- 등급 자동 승급 판정
- 등급에 따른 이동 가능 지역 제한
- 등급별 추가 보너스는 구현하지 않음 (향후)

### 데이터

- 유저 데이터에 `reputation` 필드 추가
- `Rank.json`: 등급 테이블 (이름, 필요 명성, 해금 티어)

### UI

- 홈 화면 상단에 용병단 등급 배지 + 명성 진행바
- 등급 승급 시 축하 다이얼로그
- 이동 화면에서 해금되지 않은 티어 지역은 잠금 표시

### 기존 시스템 영향

- `game_state` / `userDataProvider`: reputation 필드 관리
- `quest_provider`: 퀘스트 완료 시 명성 부여
- `movement_provider`: 이동 이벤트 조우 시 명성 부여
- `movement_screen`: 미해금 지역 잠금 표시 및 이동 차단

---

## 시스템 간 연결

```
이동 이벤트 → 골드/상태/명성 변동
퀘스트 파견 → 비용 차감 → 결과 → 보상(인건비 차감) + 경험치 + 명성
경험치 → 레벨업 → 전투력 상승 → 성공률 향상
골드 → 시설 투자 → 경험치/회복/용병수/퀘스트수 보너스
명성 → 등급 → 고티어 지역 해금 → 고난이도 퀘스트 → 더 많은 보상/명성
```

## 신규 정적 데이터 파일

| 파일 | 내용 |
|------|------|
| TravelEvent.json | 이동 이벤트 풀 (타입, 효과, 티어 범위, 설명) |
| Facility.json | 시설 정보 (이름, 효과, 단계별 비용/수치) |
| Rank.json | 등급 테이블 (이름, 필요 명성, 해금 티어) |
| MercenaryWage.json | 티어별 용병 인건비 단가 |

## 기존 데이터 변경

| 파일/모델 | 변경 |
|-----------|------|
| Difficulty.json | dispatch_cost 필드 추가 |
| Mercenary 모델 | xp, level 필드 추가 |
| User 데이터 | reputation, facilities 맵 필드 추가 |
