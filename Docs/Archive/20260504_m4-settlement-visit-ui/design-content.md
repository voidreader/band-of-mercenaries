## 개요

M4의 시작 거점은 region_id 3 더스트플레인의 마을 더스트빌이다. 본 문서는 신규 유저가 첫 30분 동안 머물 4섹터의 분위기·서사를 확정하고, 마을 내 방문 거점 3종(촌장 집·낡은 대장간·약초상)의 MVP 기능을 정의하며, 4~6명의 NPC와 8~12개의 상태 변화 문구를 명세한다. 이 문서는 컨텐츠 설계의 산출물이며, 수치(약초상 비용·쿨다운, 신뢰도 임계값)는 페이즈 2 밸런스 단계에서 확정한다.

핵심 원칙 3가지:
1. **시간 소모 없음** — 마을 방문 거점은 정보·보상·해금만 제공한다. 약초상의 즉시 회복도 게임 시간을 소모하지 않는다.
2. **의뢰소 별도 거점 미생성** — 파견 메뉴가 의뢰 기능을 담당한다. 거점에 의뢰 기능을 중복 배치하지 않는다.
3. **약초상 ≠ 의무실** — 약초상(즉시·골드·쿨다운) vs 의무실(자동·시간 단축·시설 레벨). 두 시스템은 동시에 작동하며 역할이 분리된다.

## 상세 설계

### 1. 더스트플레인 분위기·서사 (region_id 3 mountain)

#### 1.1 지역 컨셉

**한 줄 요약**: 광산을 품은 변방 산악 지역. 신참 용병들이 첫 발을 떼는 곳.

**분위기 키워드**: 먼지 / 메마름 / 흙먼지 바람 / 폐광의 그늘 / 변방의 인정

**서사 톤**:
- 더스트플레인은 한때 광산이 번성했던 지역이지만, 지금은 광산이 닫히고 마을 사람들이 변방 생활에 적응한 상태다.
- 외부에서 온 용병에 대한 마을의 첫 인상은 **의심과 무관심**이다. 용병단은 허드렛일부터 시작하여 신뢰를 쌓아야 한다.
- 산악 지형이지만 험준하지는 않다. 광산이 닫힌 후 갈색 먼지가 산자락을 덮고 있어 "더스트(Dust)"라는 이름이 붙었다.

**`regions.description` 권장 값**:
> 광산을 품은 변방 산악 지역. 한때 광부들로 북적였으나 광산이 닫힌 뒤로는 흙먼지만이 산자락을 덮는다. 신참 용병들이 첫 발을 떼는 곳.

**`environment_tags`**: `["mountain"]` (페이즈 1 #1 확정)

**`sector_count`**: `4` (페이즈 1 #2 확정)

#### 1.2 4섹터 구성 확정

페이즈 1 #2의 권장안을 본 문서에서 최종 확정한다.

| sector_index | 이름 | sector_type | environment_tags | 분위기 키워드 | 역할 |
|--------------|------|-------------|------------------|---------------|------|
| 1 | 더스트빌 | village | `["mountain","village"]` | 따뜻한 흙벽·작은 광장·낮은 건물 | 시작 마을. 3개 방문 거점 + 광장 NPC |
| 2 | 폐광 | dungeon | `["mountain","dungeon"]` | 닫힌 갱도·박쥐 그림자·녹슨 도구 | 첫 위험 지역. 조사·재료·고정 사건·첫 엘리트 후보 |
| 3 | 마른 초원 | field | `["mountain","plains"]` | 갈색 풀밭·바람·약초 무리 | 허드렛일 무대. 약초 채집·야간 순찰 |
| 4 | 먼지로 덮인 길 | field | `["mountain","road"]` | 흙먼지 길·외부 이정표·여행자 흔적 | 호위·여행자 조우 의뢰 무대 |

#### 1.3 섹터별 상세 설명 (`region_sectors.description` 입력값 권장)

**1번 섹터 — 더스트빌**
> 산기슭의 작은 마을. 흙벽 집들이 광장을 둘러싸고 있고, 촌장 집·낡은 대장간·약초상이 한 자리에 모여 있다. 마을 사람들은 외지인을 의심스럽게 본다.

**2번 섹터 — 폐광**
> 한때 마을의 생계였던 광산. 지금은 입구가 무너져 있고 박쥐와 도굴꾼이 자리 잡았다. 마을 사람들이 가장 두려워하는 곳이지만, 가장 중요한 사건이 시작되는 곳이기도 하다.

**3번 섹터 — 마른 초원**
> 마을 외곽의 거친 풀밭. 약초 무리와 들개 흔적이 섞여 있다. 신참 용병이 처음 가는 곳이며, 야간이면 마을 경계가 약해져 순찰 의뢰가 자주 나온다.

**4번 섹터 — 먼지로 덮인 길**
> 외부와 더스트플레인을 잇는 유일한 산길. 흙먼지가 발자국을 덮어버리는 곳이다. 호위가 필요한 행상이나 외지에서 온 여행자가 자주 마주친다.

### 2. 마을 방문 거점 3종 MVP 기능

`sector_type == 'village'` 섹터(1번 섹터 더스트빌)에서만 노출되는 마을 내 방문 영역. 이동 화면 하단에 추가된다.

#### 2.1 촌장 집 (Chief House)

**역할**: 고정 사건 + 마을 신뢰도의 **단일 서사 허브**

**MVP 기능 (3개 버튼)**:

| 버튼 | 동작 | 표시 정보 |
|------|------|----------|
| `상황 듣기` | 현재 활성화된 고정 사건 단계의 짧은 대화 또는 브리핑을 표시. 단계 미진행 시 마을 일상 대화 노출 | 현재 사건 단계 / 다음 목표 / 진행도 (예: "2/5 단계") |
| `신뢰도 확인` | 마을 신뢰도 4단계 진행 바 + 다음 단계 도달까지 누적 임계값 표시 | 현재 단계 / 누적 신뢰도 / 다음 단계 보상 미리보기 |
| `보상 받기` | 미수령 단계 보상이 있을 때만 활성화. 단계 진입 시 자동 지급이 아니라 명시적 수령 | 보상 항목(골드/XP/거점 해금 등) |

**시간 소모**: 없음. 3개 버튼은 모두 정보 표시 또는 즉시 보상 지급.

**데이터 의존**:
- 페이즈 1 #4의 고정 사건 라인 1개 (예: 폐광길 재개방 / 실종된 광부 수색 / 야간 습격 방어 중 1택 — 페이즈 1 #4에서 확정)
- 페이즈 2 #1의 신뢰도 4단계 누적 임계값 + 단계별 보상 수치

**상태 변화 트리거**: 신뢰도 단계가 오를 때마다 촌장 집 화면 상단의 인사말과 배경 톤이 변화 (4.1절 상태 변화 문구 참조).

#### 2.2 낡은 대장간 (Old Smithy)

**역할**: M5 제작 시스템의 **진입점 예고 + 수리 의뢰 보상**

**MVP 기능 (3개 버튼)**:

| 버튼 | 동작 | 표시 정보 |
|------|------|----------|
| `제작 목표 보기` | M5에서 제작 가능할 첫 목표(`광부의 단검` 또는 `수습대장의 검`)와 필요 재료 + 현재 보유량 표시. 실제 제작 버튼은 비활성 (M5에서 활성) | 목표 아이템 이름 + 재료 목록 (예: 녹슨 쇳조각 ×3 / 마른 가죽끈 ×1) + 보유량 0/3 형식 |
| `수리 의뢰 확인` | 마을에서 들어온 수리 의뢰 1건을 단순 골드 보상으로 처리 (M4 허용 — 풀제작 없이 보상만 지급). 의뢰 미존재 시 "오늘은 수리할 게 없네" 메시지 | 수리 대상(예: 호미·쟁기) + 보상 골드 + 진행 버튼 |
| `재료 힌트 보기` | 더스트플레인 4섹터에서 얻을 수 있는 재료 힌트 텍스트 (폐광=녹슨 쇳조각·낡은 곡괭이 / 마른 초원=마른 가죽끈 / 먼지로 덮인 길=여행자 짐 더미 등) | 섹터별 재료 후보 텍스트 4줄 |

**잠금 정책**:
- 신뢰도 1단계: 대장장이가 "신참 용병에겐 아직 줄 일이 없네"로 거절. 모든 버튼 비활성.
- 신뢰도 2단계: `제작 목표 보기` + `재료 힌트 보기` 활성화.
- 신뢰도 3단계: `수리 의뢰 확인` 활성화 (수리 의뢰 1건 노출).
- 신뢰도 4단계: 수리 의뢰 보상 +20% 또는 추가 의뢰 슬롯 (페이즈 2 #1 수치 확정).

**시간 소모**: 없음. 수리 의뢰는 M4 시점에서 즉시 보상 지급(스튜브성 메커니즘 — M5에서 진짜 제작 절차로 확장).

**데이터 의존**:
- 페이즈 4 #4 명세에서 첫 제작 목표 정의 (M5와 분리하여 노출만 구현)
- 재료 시스템 자체는 M5 — M4에서는 텍스트로만 표시, Hive 박스 미신설

#### 2.3 약초상 (Herbalist)

**역할**: **1회성 즉시 회복** + 약초 채집 허드렛일 안내

**MVP 기능 (3개 버튼)**:

| 버튼 | 동작 | 표시 정보 |
|------|------|----------|
| `즉시 회복` | 부상 또는 피로 상태인 용병 1명을 즉시 정상 상태로 복구. 골드 비용 + 24시간 쿨다운(또는 페이즈 2 #2에서 확정) | 회복 대상 용병 선택 + 비용(예: 50G) + 다음 사용 가능 시각 |
| `채집 정보 보기` | 현재 더스트플레인에서 채집 가능한 약초 종류 + 출현 섹터 + 신뢰도별 채집 의뢰 보상 변화 텍스트 | 약초 종류 ×3~4 (예: 마른 약초·산버섯·접착 수액) + 출현 섹터 표시 |
| `재료 힌트 보기` | 회복 재료(약초·접착 수액)와 회복 외 잡재료(파견용 식량 등)의 입수 방법 힌트 | 재료별 1줄 힌트 텍스트 |

**의무실(시설)과의 역할 분리** (M4 핵심 결정):

| 항목 | 약초상 (M4 신규) | 의무실 (기존 시설) |
|------|------------------|-------------------|
| **회복 방식** | **즉시 회복** (1회성) | 자동 회복 (시간 단축) |
| **트리거** | 플레이어가 명시적 클릭 + 골드 결제 | 부상 시 자동 적용, 시설 레벨에 따라 시간 단축 |
| **비용** | 골드 (페이즈 2 #2에서 확정 — 예: 50~100G) | 시설 건설비(`facilities.cost`) — 영구 효과 |
| **쿨다운** | 24시간 (페이즈 2 #2에서 확정) | 없음 (항상 작동) |
| **신뢰도 영향** | 신뢰도 상승 시 비용 감소 또는 쿨다운 단축 | 영향 없음 (시설 레벨로 결정) |
| **연결 시스템** | 마을 신뢰도, 채집 허드렛일 | 시설 시스템, 부상 자동 회복 |
| **언제 사용?** | "급할 때 골드로 즉시 해결" | "장기적 회복 효율" |

두 시스템은 **동시에 작동**한다. 의무실 자동 회복으로 30분 대기 중인 용병에게도 약초상의 즉시 회복을 적용할 수 있다 (자동 회복 타이머가 즉시 종료된다).

**잠금 정책**:
- 신뢰도 1단계: 약초상이 "처음 보는 얼굴이군. 비싸게 받지" → 즉시 회복 비용 +50%, 쿨다운 24h.
- 신뢰도 2단계: 비용 정상화. 채집 의뢰 노출 시작.
- 신뢰도 3단계: 쿨다운 12h로 감소.
- 신뢰도 4단계: 채집 의뢰 보상 +20% (페이즈 2 #1·#2 수치 확정).

**시간 소모**: 없음. 즉시 회복은 게임 시간 미소모로 즉시 적용.

**데이터 의존**:
- 페이즈 2 #2의 약초상 vs 의무실 수치 분리표 (비용 / 쿨다운 / 신뢰도별 곡선)
- 채집 의뢰 텍스트는 페이즈 1 #4 또는 페이즈 3에서 확정 (허드렛일 quest_pools 풀에 포함)

### 3. NPC 4~6명 정의

거점 NPC 3명(거점당 1명) + 거리 NPC 2명 = 총 **5명**.

| ID | 이름 | 역할 | 위치 | 짧은 성격 | 기본 인사말 (신뢰도 1단계) |
|----|------|------|------|----------|--------------------------|
| `npc_chief_parson` | **파슨** | 촌장 | 촌장 집 | 의심 많은 노인. 외지인에게 까칠하지만 한 번 신뢰하면 끈질기게 챙긴다 | "용병이라고? 쓸 만한지부터 보자고." |
| `npc_smith_hagen` | **하겐** | 대장장이 | 낡은 대장간 | 손이 거친 중년 남자. 말수가 적고 눈으로 사람을 살핀다 | "(고개를 한 번 끄덕이고 다시 모루를 두드린다)" |

**규모 결정 사유**: 4명 미만이면 마을이 비어 보이고, 6명 이상이면 신뢰도 4단계 동안의 인사말 변주(20+종)를 페이즈 3·4에서 작성하는 부담이 커진다. **5명**이 신뢰도 단계당 5인분 = 4단계 × 5인 = 20개 인사말 변주로 페이즈 3 텍스트 부담과 마을 분위기의 균형점.

**위치 데이터**:
- 거점 NPC 3명은 거점 화면 상단에 고정 노출.
- 거리 NPC 2명은 마을 광장(섹터 1 더스트빌의 별도 영역 또는 마을 방문 UI 상단의 "광장 풍문" 영역)에 노출.
- 광장 풍문은 신뢰도 단계와 고정 사건 진행에 따라 단순 텍스트가 변화 (4.2절 참조).

**확장성 (M5 이후)**:
- 도라 할멈은 미래에 "마을 신문" 역할로 확장 가능 (활동 로그의 마을 시점 보완).
- 레미는 폐광 관련 고정 사건의 단서 제공자로 확장 가능.

#### 4.1 거점별 신뢰도 단계 인사말 (총 12개 = 3거점 × 4단계 — 단계 1만 위 3.절에서 정의됨, 여기서는 단계 2~4의 9개 + 사건 완료 후 1개)

> 단계 1 인사말은 3절 NPC 표의 "기본 인사말"로 이미 정의되었다. 본 절은 단계 2·3·4 변주 + 광장 풍문 + 사건 직후 반응.

| 2 | "...당분간 자네에게 줄 일은 없네. 하지만 손이 빈 거 같으면 다시 와봐." — 제작 목표 + 재료 힌트 활성 |

#### 4.2 광장 풍문 (도라 할멈 + 레미 — 신뢰도 단계별 4개 변주)

**총 문구 수 정리**:
- 거점 단계별 인사말: 3거점 × 4단계 = **12개**
- 광장 풍문: **4개**
- 사건 완료 반응: **1개**
- **합계 17개** (요청 범위 8~12개 초과 — 거점별 4단계 변주가 의무이므로 자연스러운 결과)

페이즈 3 또는 페이즈 4 인라인에서 본 표를 그대로 입력 데이터로 사용.

### 5. 마을 방문 UI 흐름 (개념도)

```
이동 화면
├── 현재 위치: 더스트플레인 / 더스트빌
├── [지역 선택]  (다른 region_id로 이동)
├── [섹터 선택]  (1.더스트빌 / 2.폐광 / 3.마른 초원 / 4.먼지로 덮인 길)
└── [마을 내 방문]  ← sector_type == 'village' 일 때만 노출
    ├── 광장 풍문 영역 (도라 할멈 / 레미 텍스트 1~2줄)
    ├── 촌장 집      [상황 듣기] [신뢰도 확인] [보상 받기]
    ├── 낡은 대장간  [제작 목표 보기] [수리 의뢰 확인] [재료 힌트 보기]
    └── 약초상       [즉시 회복] [채집 정보 보기] [재료 힌트 보기]
```

**중요 원칙**:
- 마을 내 방문 영역은 **파견 메뉴를 대체하지 않는다**. 의뢰는 파견 탭이 담당.
- 폐광은 거점이 아닌 섹터(2번)이므로 마을 내 방문 영역에 표시되지 않는다. 폐광 의뢰는 파견 탭에서 sector_type='dungeon' 풀로 노출.
- 거점 클릭 시 별도 화면으로 전환되거나 인라인 펼침 — UI 형식은 페이즈 4 #4 명세에서 결정.

## 현재 시스템과의 연관

### 영향받는 시스템

| 영역 | 영향 | 마이그레이션 범위 |
|------|------|------------------|
| `regions` 테이블 (region 3) | UPDATE: region_name="더스트플레인" / environment_tags=`["mountain"]` / sector_count=4 / description 갱신 | 페이즈 4 #1 |
| `region_sectors` 테이블 | INSERT 4행 (위 6.1) | 페이즈 4 #2 |
| 마을 NPC 데이터 | **신규 테이블 또는 인라인 JSON** — 페이즈 4 #4에서 결정 | 페이즈 4 #4 |
| 거점 상태 변화 문구 | **신규 테이블 또는 인라인 JSON** — 페이즈 4 #4에서 결정 | 페이즈 4 #4 |
| `MovementScreen` | sector_type=='village' 분기 → "마을 내 방문" 영역 추가 | 페이즈 4 #4 |
| `HerbalistService` (신규) | 1회성 회복 + 쿨다운 + 골드 결제 로직 | 페이즈 4 #4 |
| `FacilityService` (의무실) | **변경 없음** — 자동 회복 효과 유지 | — |
| 마을 신뢰도 시스템 | 본 문서의 거점 잠금 정책 + 인사말 변주를 trust_level로 분기 | 페이즈 4 #5 |
| 고정 사건 라인 1개 | 촌장 집의 [상황 듣기] [보상 받기]가 진행 표시 + 보상 지급 | 페이즈 1 #4 + 페이즈 4 #5 |

### 호환성 검토

- **기존 사용자 세이브**: region 3을 보유한 세이브는 regions UPDATE 후에도 region_id 유지로 호환. region_id가 매핑표 외 값이면 페이즈 1 #1 정책에 따라 NewGame 강제.
- **약초상 vs 의무실 분리**: 의무실 시설은 기존 효과 그대로. 약초상은 신규 서비스로 추가되며 의무실과 독립적으로 작동. 두 시스템이 충돌하지 않도록 페이즈 4 #4에서 단위 테스트 권장.
- **거점 잠금 정책**: 신뢰도 1단계에서 모든 거점 진입은 가능하지만 일부 버튼이 비활성화. 진입 자체를 막지 않는 이유는 NPC 인사말로 마을 분위기를 학습시키기 위함.
- **"마을 내 방문" 시간 미소모 원칙**: 파견·이동·건설·조사·의무실 자동 회복만 시간을 소모. 약초상의 즉시 회복은 게임 시간 미소모(대신 골드+쿨다운으로 페널티). M5 제작 진입 시에도 이 원칙은 유지 권장.

### data-generator 적용 여부

본 문서의 산출물은 다음과 같이 분류된다.

| 항목 | 수량 | data-generator 적용 권장 | 사유 |
|------|------|------------------------|------|
| region_sectors 4행 (region 3) | 4 | ❌ (본 문서에서 100% 명시) | 페이즈 1 #2의 region_sectors ~164행 일괄 생성에 포함 |
| NPC 5명 | 5 | ❌ (본 문서에서 100% 명시) | 시작 거점 한정 — 다른 마을 NPC가 늘어날 때 types/npc-line.md 작성 검토 |
| 거점 상태 변화 문구 17개 | 17 | ❌ (본 문서에서 100% 명시) | 시작 거점 한정 — 다른 마을 거점 추가될 때 types/village-facility.md 작성 검토 |
| 채집 의뢰 / 수리 의뢰 텍스트 | (M4 시점) 0~5 | △ (페이즈 1 #4에서 결정) | 허드렛일 풀 10~15개 생성 시 함께 처리 |

본 문서 자체로는 신규 data-generator 타입 스펙이 필요하지 않다. 페이즈 3 진입 시 `region_sectors` ~164행 / NPC·문구는 페이즈 4 인라인 처리를 권장.

## 후속 작업

페이즈 1 #4 "마을 신뢰도 4단계 + 고정 사건 라인 1개"가 본 문서를 입력으로 받아 다음을 결정한다.

- 마을 신뢰도 4단계 누적 임계값 + 단계별 보상 (수치는 페이즈 2 #1)
- 고정 사건 라인 1개 (예: 폐광길 재개방 / 실종된 광부 수색 / 야간 습격 방어 중 1택)
- 단계별 trust_threshold 매핑 — quest_pools 신규 컬럼 4개에 반영
- 고정 사건 진행 상태 저장소 결정 (chainQuestProgress 재사용 vs settlement_event_progress 신설)

페이즈 1 #5 "초반 2시간 플레이 흐름"이 본 문서의 4섹터·NPC·거점 인터랙션을 분 단위 흐름에 배치한다.

페이즈 2 #2 "약초상 vs 의무실 수치 분리"가 본 문서 2.3절의 약초상 1회성 회복 컨셉을 다음 수치로 정량화한다.
- 즉시 회복 골드 비용 곡선 (단계 1: +50% / 단계 2: 정상 / 단계 3: -10% / 단계 4: -20% 등)
- 쿨다운 곡선 (단계 1·2: 24h / 단계 3: 12h / 단계 4: 8h 등)
- 의무실 자동 회복 시간 단축 곡선과의 교차점 (어느 시점에 약초상 사용이 의무실보다 효율적인가?)

페이즈 4 #4 "마을 방문 UI + 거점 3종 + 약초상/의무실 분리"가 본 문서 전체를 명세 입력으로 받아 구현한다.
