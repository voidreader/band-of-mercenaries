# 세력(Faction) 시스템 컨텐츠 기획서

> 작성일: 2026-04-16
> 유형: 신규 컨텐츠
> 선행 구현: Phase 6 (세력 발견 시스템) 완료

---

## 개요

하나의 대륙 안에 공존하는 14개 이해집단을 설계한다. 플레이어는 세력을 발견하고, 평판을 쌓고, 가입함으로써 전용 시설·트레잇·상점을 얻는다. 세력 간 이해충돌 구조가 가입의 전략적 선택을 만들며, "어느 세력과 손을 잡을 것인가"가 용병단 아이덴티티를 결정하는 장기 목표가 된다.

**레퍼런스 비전**: 용마검전(세력 진영) — 장기 소속감 + 배타적 혜택이 결합된 경험

---

## 레퍼런스 분석

| 레퍼런스 | 차용 메커니즘 | 적용 방식 |
|---------|-------------|----------|
| 용마검전 — 세력 진영 | 세력 소속이 캐릭터의 정체성을 형성 | 세력 가입 → 전용 시설·트레잇으로 용병단 방향성 결정 |
| Melvor Idle — 스킬 마스터리 | 반복 행동이 누적되어 전문성 강화 | 세력 퀘스트 반복 → 평판 상승 → 상점·트레잇 확장 |
| Darkest Dungeon — 길드 잠금 | 특정 조건 달성 후 고급 기능 해금 | 평판 구간별 상점 범위 확장, 트레잇 획득 조건 |
| OGame — 연구 의존 순서 | 선행 조건 달성이 다음 단계를 여는 구조 | 발견(clue) → 가입 → 평판 → 상점/트레잇 순차 해금 |

## 상세 설계

### 1. 세력 분류 체계

| 대분류 | 수량 | 발견 방식 | 충돌 범위 |
|-------|------|----------|----------|
| **공개 세력** | 6개 | 도감에 처음부터 이름 표시 | 공개 세력 내에서만 |
| **비밀 세력** | 4개 | clue_level 1→2→3 발견 후 공개 | 비밀 세력 내에서만 |
| **지역·종족 세력** | 4개 | clue_level 1→2→3 발견 후 공개 | 지역·종족 세력 내에서만 |

**설계 원칙:** 이해충돌은 동일 대분류 내에서만 발생한다. 공개 세력이 비밀 세력과 충돌하지 않으므로, 공개·비밀·지역 세력을 동시에 최대 3개 가입해도 충돌하지 않는 조합이 가능하다.

### 2. 가입 및 평판 시스템

#### 평판 구조

```
평판 범위: -100 ~ +100
기본값: 0
가입 전 획득 가능 상한: +10 (세력 태그 퀘스트로 자연 누적)
가입 조건: 평판 > 0 (0 이하면 가입 불가)
적대 기준: 평판 < 0
동시 가입 제한: 최대 3개 (대분류 무관, 이해충돌 제외)
```

#### 이해충돌 처리

- 충돌 세력에 가입하면 상대 세력 평판이 **-100으로 강제 설정** (고정 페널티)
- -100은 이후 다시 올릴 수 있음
- 회복 경로: 적대 세력 **탈퇴 후** 세력 퀘스트로 평판 재상승
- 탈퇴 시: 평판 수치 유지, 전용 시설 잠금(레벨 보존), 전용 트레잇 유지(신규 획득만 차단)
- **적대 상태에서는 세력 퀘스트 발생 안 함, 평판 획득 불가**

#### 평판 구간별 효과

| 구간 | 상태 | 효과 |
|------|------|------|
| 0 (초기) | 미접촉 | 세력 태그 퀘스트 발생 가능 (단서 보유 시) |
| 1~10 | 관심 | 가입 창구 해금 가능 |
| 11~30 | 신참 | 세력 상점 기본 품목 |
| 31~60 | 인정 | 세력 상점 중급 품목, 기본 트레잇 |
| 61~90 | 신뢰 | 세력 상점 고급 품목, 고급 트레잇 |
| 91~100 | 핵심 | 세력 상점 전품목, 최상위 트레잇 |

### 3. 퀘스트 세력 태그 시스템

#### 기본 태그 퀘스트

일반 퀘스트에 낮은 확률로 세력 태그 부여.

```
퀘스트 생성 시:
  → 현재 리전에서 단서를 보유한 세력 목록 조회
  → 각 세력별 태그 확률 계산 (기본 10%, 거점 근접 보정)
  → 태그 붙은 퀘스트 완료 시 해당 세력 평판 +1~2 획득
```

**거점 근접도 보정:**

| 거점까지 리전 거리 | 태그 발생 확률 |
|-----------------|-------------|
| 1~3 | +20% (최대 30%) |
| 4~7 | +10% (최대 20%) |
| 8~15 | 기본 10% |
| 16+ | 5% |

**단서 없는 세력의 태그는 절대 발생하지 않음.** 공개 세력도 단서(도감 표시) 자체가 조건을 충족한 것으로 간주.

#### 세력 전용 퀘스트 (가입 후 전용)

| 항목 | 일반 세력 태그 퀘스트 | 세력 전용 퀘스트 |
|------|-------------------|--------------|
| 발생 조건 | 단서 보유 | 가입 + 평판 11 이상 |
| 난이도 | 현재 지역 기준 | 현재 지역 +1~2 난이도 |
| 파견 인원 | 자유 | 최소 2명 권장 표시 |
| 지역 이동 | 없음 | 타 지역 이동 포함 가능 |
| 평판 보상 | +1~2 | +5~10 |
| 금전 보상 | 기본 배수 | 기본 배수 × 1.3 |

### 4. 가입 혜택 구조

```
가입 즉시
├── 패시브 보너스 활성화
└── 전용 시설 건설 가능 해금

평판 상승에 따라 (거점 방문 필요)
├── 세력 상점 구매 범위 확장
└── 세력 전용 트레잇 획득 가능
```

**거점 방문 필요 기능:**
- 세력 상점 이용
- 세력 전용 트레잇 획득

**거점 구현은 향후 Phase.** 거점 구현 전까지는 상점과 트레잇은 stub 상태로 UI만 표시.

##### 1. 모험가 길드 (Adventurers' Guild)
```
id: faction_adventurers_guild
tier_range: [1, 3]
color: #4CAF50
visibility_type: public
```

**설명:** 대륙 전역에 지부를 둔 가장 오래된 용병·탐험가 조합. 출신과 종족을 가리지 않고 누구에게나 열린 문을 자랑하며, 대륙 초보자들이 처음 이름을 올리는 곳이다.

**철학:** 용기와 탐험이 세계를 바꾼다. 두려움 없이 나아가는 자만이 진정한 모험가다.

|------|------|
| 가입 조건 | 랭크 E + 평판 > 0 |
| 이해충돌 | 도둑 길드 (법질서 vs. 지하세계) |
| 패시브 보너스 | 탐험(explore) 퀘스트 보상 +15% |
| 전용 시설 | **의뢰 중개소** — 고급 의뢰 생성 확률 상승, 의뢰 갱신 주기 단축 |

**설명:** 마법과 기계를 융합한 금지 기술을 연구하는 비밀 결사. 인체 개조와 금지된 자동화 장치를 통해 인간의 한계를 초월하는 것을 목표로 한다. 구성원들은 스스로를 실험 대상으로 삼는 것도 마다하지 않는다.

**철학:** 한계는 극복하기 위해 존재한다. 금지된 기술이 인간을 신의 경지로 이끈다.

| 가입 조건 | clue_level 3 + 평판 > 0 |
| 이해충돌 | 뿌리의 맹세단 (기술로 자연 초월 vs. 자연 질서 수호) |
| 패시브 보너스 | 시설 건설 비용 -10%, 시설 효과 +5% |

### 6. 이해충돌 매트릭스

```
공개 세력:    모험가 길드   ↔  도둑 길드
              상인 연합     ↔  전사 길드
              마탑 연합     ↔  태양 교단

비밀 세력:    균형 감시자   ↔  금지된 서고
              뿌리의 맹세단 ↔  황혼 공학회

지역·종족:    심층 망치단   ↔  화산 심장단
              혈계 귀족회   ↔  송곳니 결사
```

**동시 가입 예시 (최대 3개):**
- 모험가 길드 + 균형 감시자 + 심층 망치단 → 가능 (각 대분류 1개)
- 모험가 길드 + 전사 길드 + 균형 감시자 → 가능 (공개 세력 내 충돌 없음)
- 모험가 길드 + 도둑 길드 → **불가** (이해충돌)
- 마탑 연합 + 금지된 서고 → 가능 (다른 대분류, 충돌 없음)

### 7. Supabase 데이터 설계

#### `factions` 테이블 확장 필드

현재 `id, name, description, philosophy, tier_range, color` 에 추가:

| 신규 컬럼 | 타입 | 설명 |
|---------|------|------|
| `visibility_type` | TEXT | `public` / `secret` / `regional` |
| `join_rank_min` | TEXT? | 최소 랭크 코드 (`E`/`D`/`C`/`B`/null) |
| `join_needs_clue` | BOOLEAN | clue_level 3 필요 여부 |
| `passive_bonus_json` | JSONB | 패시브 보너스 정의 |
| `facility_id` | TEXT? | 전용 시설 ID (faction_facilities 참조) |

#### `faction_conflicts` 테이블 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `faction_id_a` | TEXT | FK → factions.id |
| `faction_id_b` | TEXT | FK → factions.id |

양방향 충돌이므로 (A, B) 한 쌍만 저장. 조회 시 양방향으로 검색.

**초기 데이터 (7쌍):**
```
(faction_adventurers_guild, faction_thieves_guild)
(faction_merchants_alliance, faction_warriors_guild)
(faction_mage_towers, faction_sun_order)
(faction_balance_watchers, faction_forbidden_archive)
(faction_root_oath, faction_twilight_artificers)
(faction_deep_hammer, faction_volcanic_heart)
(faction_blood_council, faction_fang_brotherhood)
```

#### `faction_facilities` 테이블 (신규)

현재 `facilities` 테이블과 동일한 포맷. 세력 전용 시설 14종.

| 컬럼 | 타입 |
|------|------|
| `id` | TEXT PK |
| `faction_id` | TEXT FK |
| `name` | TEXT |
| `description` | TEXT |
| `effect_type` | TEXT |
| `max_level` | INT |
| `cost_json` | JSONB |
| `effect_json` | JSONB |

#### `faction_states` Hive 박스 확장 (향후)

현재 구현된 `FactionState` 모델에 추가 필요:
```dart
// 현재
factionId: String
clueRecords: List<FactionClueRecord>

// 추가 필요
reputation: int          // -100 ~ 100, 기본 0
joined: bool             // 가입 여부
joinedAt: DateTime?      // 가입 시각
facilityLevels: Map<String, int>  // 세력 시설 레벨 (탈퇴 시에도 보존)
```

## 현재 시스템과의 연관

| 기존 시스템 | 영향 | 처리 방식 |
|-----------|------|----------|
| `factions` Supabase 테이블 | 필드 추가 필요 | 기존 3개 세력 데이터에 신규 필드 마이그레이션 |
| `FactionState` Hive 모델 | reputation 등 필드 추가 | HiveField 번호 이어서 추가, 기존 데이터 호환 |
| `FactionStateRepository` | 평판 CRUD, 충돌 처리 로직 추가 | 기존 processClue()에 평판 로직 병합 |
| 퀘스트 생성 시스템 | 세력 태그 퀘스트 생성 로직 추가 | QuestGenerator에 faction_tag 필드 조건부 추가 |
| 세력 도감 UI | 가입 상태, 평판 바, 전용 시설 표시 | FactionDetailScreen 확장 |
| 시설 시스템 | 세력 전용 시설 추가 | FacilityTabScreen에 세력 시설 섹션 분리 또는 통합 |
