# 생활권 완성도 대시보드 컨텐츠 기획서

> 작성일: 2026-05-20
> 유형: 신규 컨텐츠 (M8.5 페이즈 1 #1)
> 적용 마일스톤: M8.5 "재미 가시화와 폴리싱"
> 적용 리전: region 3 (더스트플레인) 단독 MVP, M9에서 6 리전 확장 위임

---

## 1. 개요

M4~M7에서 분산 구현된 거점 신뢰도·세력 도감·체인 의뢰·M7 인프라 시스템·M8a 세력 수직 절편을 한 화면에서 "현재 어디까지 와 있고, 다음 무엇을 해야 하는가"로 통합 노출하는 대시보드를 추가한다. 플레이어는 홈에서 5초 안에 진행 상황을 훑고, 정보 탭에서 30초 안에 다음 행동을 정한다.

대시보드는 신규 시스템이 아니라 기존 데이터의 가시화다. 데이터 마이그레이션, 신규 hook, 신규 산식은 없다. 변환과 표시만 한다.

핵심 가치 명제:
- **30분 세션 플레이어**: 홈 요약 카드 → 30분 목표 → 즉시 행동
- **8시간 방치 플레이어**: 홈 요약 카드 → 8시간 목표 → 다음 임계값 인지
- **목표 추적 플레이어**: 정보 탭 상세 → 6지표 펼침 + 점프 버튼 → 관련 시스템 화면 직행

## 2. 레퍼런스 분석

### 행성 개요 패턴 (OGame)

OGame은 행성별 6~8 자원/시설 지표를 한 패널에 그리드 형태로 보여주고, 각 지표는 다음 임계값 ETA를 함께 표시한다. 단일 행성 단위 진행도를 한눈에 확인하는 패턴이 region 3 단독 MVP에 정합한다.

→ 차용 포인트: 6지표 풀 그리드 노출, 다음 임계값 ETA 표시, 통합 점수 헤더.

### Skills Overview 패턴 (Melvor Idle)

Melvor Idle의 Skills overview는 30+ 스킬을 그리드로 보여주는 풀 패널. 각 스킬은 마스터리 % 진행 바와 다음 레벨 ETA를 표시한다. 한 화면에서 모든 진행 상태를 훑는 정보 위계를 만든다.

→ 차용 포인트: 미니 진행 바 그리드, 진행률 % 통합 표시, 인라인 펼침.

### Quest Log 패턴 (Darkest Dungeon)

Darkest Dungeon의 town overview는 도시 시설별 발전 단계를 카드 형태로 보여주고, 각 카드는 다음 업그레이드 비용·효과를 즉시 표시한다.

→ 차용 포인트: 단계형 지표 표시(거점 발전 Tier 1~4), 다음 단계 ETA·조건 명시.

### 부적합 패턴 (제외)

- **FTL Pause Screen** (전체 상태 일시정지): 본 게임은 방치형이라 일시정지 불필요
- **Stardew Valley Calendar** (날짜 기반): 본 게임은 실시간 기반이라 캘린더 부적합

## 3. 상세 설계

### 3.1 진입 동선 (D 하이브리드)

```
홈 야영지 화면
├── 야영지 이미지
├── FlagshipHomeCard
├── ChronicleHomeCard
├── [신규] LivingsphereSummarySection  ← 5초 훑기 진입
│   └── [자세히 보기 →] 정보 탭으로 이동
├── 진행 중 의뢰·이동·조사 상태
└── 활동 로그

정보 탭 화면
├── 명성 카드
├── [신규] LivingsphereDetailScreen 진입 카드  ← 30초 탐색 진입
│   ├── 헤더: 더스트플레인 — 통합 완성도 67%
│   ├── 6지표 펼침 카드 (각각 점프 버튼 포함)
│   ├── 다음 30분 목표 (자동 추천 + 수동 핀)
│   └── 다음 8시간 목표 (자동 추천 + 수동 핀)
├── 세력 도감
└── (기존 섹션)
```

홈 요약 카드는 항상 표시되어 즉시성을 확보하고, 정보 탭은 장기 추적용 풀 패널을 제공한다. 두 위치는 동일한 데이터 소스(`RegionState`·`BandAchievement`·`FactionState` 등)를 공유하며, 정보 위계만 다르다.

### 3.2 6지표 정의

각 지표는 0~100% 정규화 또는 단계형 표현 중 하나를 채택한다. 산식의 정량값은 페이즈 2 밸런스에서 정밀화한다.

| 지표 | 표현 | 산정 원천 | 색상 | 점프 대상 |
|------|------|-----------|------|----------|
| **안정도** | 0~100% + 단계 라벨 | `RegionState.dangerScore` (-100~+100, 낮을수록 안전) 역정규화 + 4단계(stable/peaceful/tension/threat) 라벨 | `AppTheme.dangerLevelColor` (파랑/초록/주황/빨강) | `MovementScreen` 생활권 선택 포커스 |
| **거점 발전** | 단계 Tier 1~4 | `RegionState.infrastructureTier` (1~4, 신뢰도는 보조 표시) | `AppTheme.settlementAccent` (0xFFFFA000) | `ChiefHouseScreen` |
| **사건 완료율** | n/m 카운트 + % | 완주 체인·거점 사건 위업·`RegionState.triggeredDiscoveries`·`questPoolCompletionCounts` 기반 M7 상태 의뢰 완료 (region 3 한정 집계) | `AppTheme.chainGold` (0xFFD4AF37) | `DispatchScreen` 체인/고정 의뢰 포커스 |
| **자원·제작** | n/m 카운트 + % | 보유 특산품 종 수(`firstAcquiredMaterialIds`) + 해금 레시피 수(region 3 unlock_condition 만족 레시피) | M5 재료/제작 베이스 색상 | `InventoryScreen` MaterialTab (slot=material_*) |
| **영향력** | 평균 점수 + 단계 | M8a 활성 3 세력(모험가 길드·상인 연합·전사 길드) 평판 평균 / 계산형 `FactionRelationStage` 7단계 매핑 | `AppTheme.namedAccent` (0xFFE91E63) | `FactionCodexScreen` |
| **위업 달성률** | n/m 카운트 + % | region 3 관련 위업 달성 수 / region 3 전체 위업 시드 수 (`region_pacified:region_3`·`infrastructure_tier:tier_4`·`settlement_event_completed:settlement_*`·region 3 한정 `craft_first_rare:*` 등) | `AppTheme.uniqueAccent` (0xFF7b1fa2) | `ChronicleScreen` |

산식 컨셉 (페이즈 2 정밀화 위임):
- **안정도**: `((100 - dangerScore) / 200) × 100%`를 0~100으로 clamp. `dangerScore=-100`은 100%, `dangerScore=+100`은 0%다. 4단계 라벨은 `DangerLevelResolver.resolveLevel`을 그대로 사용. M7 dangerLevel 색상 공유.
- **거점 발전**: `(infrastructureTier - 1) / 3 × 100%`. 신뢰도는 보조 정보로 펼침 시 표시. Tier 4 도달 시 100%.
- **사건 완료율**: 분모는 region 3에 정의된 거점 체인·M7 상태 변화 의뢰·region_discoveries 시드 총합. 분자는 `UserData.completedChains`/`ChainQuestProgress`, `BandAchievement.templateId`, `RegionState.triggeredDiscoveries`, `RegionState.questPoolCompletionCounts`에서 재구성한다. 완료 후 삭제될 수 있는 `ActiveQuest.status == completed`는 장기 달성률 산식에 사용하지 않는다.
- **자원·제작**: 분모는 region 3 unlock_condition을 갖는 레시피 수 + region 3 firstAcquiredMaterial 후보 5종. 분자는 실제 해금/획득.
- **영향력**: M8a 활성 3 세력의 평판을 가입(-100~100)·미가입(-100~10)으로 정규화 후 평균. 가입 가중치 1.5×.
- **위업 달성률**: region 3 관련 위업 시드 수가 분모. `BandAchievement.regionId == 3`을 1차 기준으로 쓰고, `regionId`가 비어 있는 과거/범용 위업은 templateId allowlist(`settlement_event_completed:settlement_3_*`, `infrastructure_tier:tier_4`, region 3 한정 `craft_first_rare:*`)로 보정한다. category(`infrastructure_growth` 등)와 templateId를 혼용하지 않는다.

### 3.3 30분/8시간 목표 정책 (C 하이브리드)

#### 30분 목표 (즉시 행동)

자동 추천 우선순위:
1. **진행 중 작업 완료 임박** (~30분 이내 종료 예정)
   - 진행 중 의뢰 (`ActiveQuest.endTime`) → "의뢰 완료까지 12분"
   - 진행 중 이동 (`UserData.moveEndTime`) → "더스트플레인 도착까지 8분"
   - 진행 중 조사 (`UserData.investigationEndTime`) → "지역 조사 완료까지 15분"
   - 진행 중 건설 (`UserData.constructionEndTime`) → "공방 Lv3 완료까지 24분"
2. **임계값 직전** (다음 단계까지 ~30분 행동으로 도달)
   - 신뢰도 임계값 직전 (예: 28/30)
   - 인프라 단계 flag 1개 부족
   - 명성 다음 랭크까지 ~30분 행동 거리

자동 추천 fallback:
- 진행 중 작업 0개 + 임박 임계값 0개 → "다음 의뢰를 시작하세요" stub (DispatchScreen 점프 버튼)

수동 핀:
- 플레이어가 정보 탭에서 임의 작업·목표를 ★로 핀 가능
- 핀 설정 시 자동 추천보다 우선 노출
- 핀 만료(작업 완료)되면 자동 추천 fallback

#### 8시간 목표 (방치형/장기)

자동 추천 우선순위:
1. **다음 임계값 진입 ETA** (현재 누적 속도 기반 추정)
   - 다음 신뢰도 단계 진입 (예: "신뢰도 4단계까지 142/200")
   - 다음 인프라 단계 진입 (예: "거점 Tier 4까지 flag 3/6")
   - 명성 다음 랭크 진입 (예: "C 랭크까지 5,820/8,000")
2. **활성 체인 완주** (남은 단계 수 표시)
3. **핵심 제작 레시피 해금** (1차 재료 부족 표시)
4. **신규 세력 가입 조건 충족** (평판·랭크 조건 표시)

자동 추천 fallback:
- 모든 임계값에서 멀 때 → "위업 컬렉션을 채워보세요" stub (ChronicleScreen 점프 버튼)

수동 핀:
- 30분과 동일 메커니즘

#### 핀 데이터 구조 (MVP 정책)

```dart
// MVP는 별도 Hive type을 만들지 않고 UserData에 pinId 2개만 저장한다.
// pinId 예: 'quest:abc123' or 'chain:settlement_3_pyegwang_reopen' or 'craft:recipe_id'
enum GoalSlot { short30Min, long8Hour }
```

MVP에서는 `UserData.shortGoalPinId` / `UserData.longGoalPinId`만 추가한다. `pinnedAt`과 사용자 메모는 UI 가치 대비 영속 모델 부담이 크므로 M9+로 미룬다. 핀 대상이 완료·삭제·조건 불만족 상태가 되면 `GoalRecommendationService`가 해당 pinId를 무효 처리하고 자동 추천 fallback을 노출한다.

레이아웃 정책:
- 카드 위치: 홈 야영지 화면의 ChronicleHomeCard 직후 (FlagshipHomeCard → ChronicleHomeCard → LivingsphereSummarySection 순)
- 헤더: 지역명 + 통합 완성도 % (가중치는 페이즈 2 #1에서 확정)
- 6지표: 미니 진행 바 형식. 각 지표는 표현 유형에 따라 % 또는 단계 또는 카운트 표시
- 30분/8시간 목표: 🎯 아이콘 + 한 줄 요약
- 자세히 보기 버튼: 정보 탭 LivingsphereDetailScreen으로 점프

빌드 임계 케이스:
- region 3 외 위치: "더스트플레인 생활권" 헤더는 유지하고 데이터는 region 3 기준으로 표시 (위치 무관 항상 region 3 대시보드)
- 6지표 중 부분 데이터 부족 (예: 신규 유저, FactionState 없음): "—" 표시 + 0% 진행 바
- 진행 중 작업 0개 + 임계값 멀 때: 30분/8시간 목표는 stub fallback

- 모든 지표 카드 기본 접힘. 탭 시 인라인 펼침
- 펼침 상태는 SessionState (영속 X) — 화면 이탈 시 초기화
- 펼침 후 점프 버튼은 카드 하단에 배치. 점프 시 LivingsphereDetailScreen 컨텍스트 보존(뒤로 가기 시 펼침 상태 복원)
- 다중 점프 가능 지표(자원·제작 → 인벤토리/대장간 2개)는 2개 점프 버튼 노출

점프 구현 정책:
- **하단 탭 이동만으로 충분한 대상**: `currentTabProvider`를 사용한다.
- **화면 내부 포커스가 필요한 대상**: 기존 일회성 provider 패턴을 확장한다. 예: `livingsphereMovementTargetProvider(regionId)`, `dispatchFocusQuestPoolIdProvider`, `settlementFacilityTargetProvider`, `factionCodexScrollTargetProvider`, `materialJumpTargetItemIdProvider`, `recipeFilterMaterialIdProvider`.
- **MovementScreen**: 현재 `LivingsphereJumpBar`는 사용자의 현재 위치가 M7 생활권 7리전일 때만 노출된다. 대시보드의 안정도 점프는 이 바 자체가 아니라 `MovementScreen`의 선택 region을 region 3 또는 관련 생활권 region으로 맞추는 포커스 동작으로 정의한다.
- **ChiefHouseScreen/대장간**: 현재 `MovementScreen` 내부 `VillageVisitSection`의 로컬 선택 상태가 필요하므로 "변경 없음" 대상이 아니다. 페이즈 4 명세에서 시설 선택 target provider를 추가한다.

목표 카드 정책:
- 자동 추천 후보 ▶ 표시 (1개)
- 후보 ○ 표시 (최대 3개)
- 핀 설정 시 ★ 표시로 자동 추천 대체
- 핀 토글은 정보 탭에서만 가능 (홈 요약 카드에서는 자동/핀 결과만 노출)

### 3.6 시각 표현 가이드

- 통합 완성도 %: 헤더 우상단 큰 글씨 (16~18sp)
- 6지표 진행 바: 10칸 (▰▱) 또는 Flutter `LinearProgressIndicator` (페이즈 4 명세에서 확정)
- 단계 라벨 (Tier 1~4, Lv 1~4): 진행 바 옆 텍스트로 표시
- 색상: 표 3.2 매핑 그대로 (M7 dangerLevel·M6 chainGold·M4 settlementAccent·M8a namedAccent 등 기존 AppTheme 재사용)
- 아이콘: 🏘️ (지역) / 🎯 (목표) / ★ (핀) / [▼] (펼침) / [→] (점프)

신규 영속 데이터:
- `UserData.shortGoalPinId` / `longGoalPinId` HiveField 28~29
- 주간 기여도 랭킹의 `UserData.weeklyScoreBuffer` 계열 필드는 페이즈 4 #6에서 HiveField 30~33을 사용한다. 생활권 대시보드 목표 핀과 번호를 공유하지 않는다.

### 4.2 영향 받는 시스템

| 시스템 | 변경 사항 |
|--------|----------|
| HomeScreen | LivingsphereSummarySection 신규 카드 삽입 |
| InfoScreen | LivingsphereDetailScreen 진입 카드 신규 |

| InventoryScreen MaterialTab | 기존 `materialJumpTargetItemIdProvider`·필터 provider 재사용 |

| UserData | HiveField 28~29 추가 (shortGoalPinId / longGoalPinId) |

### 4.3 호환성 검토

- **신규 유저 (region 3 미진입)**: 통합 완성도 ~5%, 6지표 대부분 0%, 30분 목표는 "다음 의뢰를 시작하세요" stub. M7 명성 조건(D 랭크 8,000)으로 region 3 도달 가능.
- **기존 유저 (region 3 도달, 인프라 Tier 1~4 진행 중)**: 데이터가 자연스럽게 표시되며 디스럽션 없음.
- **방치 유저 (8시간+ 미접속)**: 8시간 목표가 임계값 도달 여부 표시. 핀 설정 시 임박 알림 가능 (M8.5 #1 범위 외, M9 위임).
- **M9 6 리전 확장 시**: LivingsphereSummarySection·LivingsphereDetailScreen에 리전 선택 드롭다운 추가만 필요. 자료 구조는 region 3 기준으로 설계되었으므로 region 31·127·9·10·146·38은 데이터만 채우면 활성화.

### 4.4 잠재적 우려

- **정보 과부하**: 홈 요약 카드가 6지표 + 2 목표 + 자세히 보기 버튼으로 길어질 수 있음. 페이즈 4 명세에서 카드 높이 cap 검토 (예: max 240dp).
- **자동 추천 알고리즘 자의성**: 30분/8시간 목표의 자동 추천 정책이 게임 의도와 다를 위험. 페이즈 2 밸런스에서 우선순위 가중치 정밀화 필요.
- **통합 완성도 % 산식 자의성**: 6지표 가중평균 가중치가 게임 진행 흐름과 다를 위험. 페이즈 2 밸런스에서 정밀화.
- **region 3 한정 위업 시드 부재**: `band_achievement_templates`에 region 3 한정 시드가 `region_pacified:region_3`·`infrastructure_tier:tier_4`·`settlement_event_completed:settlement_3_*` 일부에 국한. 위업 달성률 분모가 불안정할 수 있음. 페이즈 2에서 templateId allowlist를 확정해야 한다.
- **점프 동선의 로컬 상태 의존**: `MovementScreen`·`VillageVisitSection`·`DispatchScreen` 일부 포커스는 현재 로컬 state라 하단 탭 이동만으로 목표 위치를 보장할 수 없음. 페이즈 4에서 일회성 target provider를 명시해야 한다.

## 5. 페이즈 2/3/4 인터페이스

### 5.1 페이즈 2 (밸런스) 위임 항목

- 통합 완성도 % 산식: 6지표 가중평균 가중치 (안정도 0.2 / 거점 발전 0.2 / 사건 0.2 / 자원·제작 0.15 / 영향력 0.1 / 위업 0.15 같은 초안 + 검증)
- 각 지표의 0~100% 정규화 산식 정밀화 (특히 사건 완료율·위업 달성률의 분모 확정)
- 30분 목표 자동 추천 우선순위 가중치 (시간 임박도 vs 가치 가중치)
- 8시간 목표 자동 추천 우선순위 가중치 (다음 임계값 가까움 vs 보상 가치)
- 임박 임계값 판정 threshold (예: 다음 단계까지 ~30분 행동 거리 = 명성 ~200G 이내 등)

### 5.2 페이즈 3 (데이터) 위임 항목

- 이미지 에셋 메타데이터 (페이즈 4 #5): 더스트플레인 생활권 배경 썸네일 1장 + 6지표 카테고리 아이콘 6종 (안정도·거점·사건·자원·영향력·위업)
- region 3 한정 위업 시드 분모 확정 시 추가 시드가 필요한 경우 페이즈 4 #5에 합산 위임

### 5.3 페이즈 4 (명세) 위임 항목

- HomeScreen 확장 명세 (페이즈 4 #1): LivingsphereSummarySection 위젯 신규
- InfoScreen 확장 명세 (페이즈 4 #1): LivingsphereDetailScreen 신규 + 6 펼침 카드 + 2 목표 카드
- `livingsphereDashboardProvider` (Provider<LivingsphereDashboardSnapshot>) — RegionState·BandAchievement·FactionState·ActiveQuest·UserData watch
- `goalRecommendationService` (GoalRecommendationService) — 자동 추천 알고리즘
- `livingsphereGoalPinProvider` 또는 `UserDataNotifier` 메서드 — `UserData.shortGoalPinId` / `longGoalPinId` 읽기·쓰기·무효화
- `BandAchievementCategory` + templateId allowlist 분류 헬퍼 (region 3 관련 위업 필터)
- 점프 동작 명세 (`currentTabProvider` + 일회성 target provider + 컨텍스트 보존 정책)
- 통합 완성도 % 계산 헬퍼 (페이즈 2 산식 적용)

## 6. 구현 우선순위 제안

**우선순위: 중간**

근거:
- 신규 시스템이 아니라 기존 데이터 가시화이므로 M8.5 핵심 6 시스템 중 가장 적은 코드 변경
- 그러나 "재미 가시화" 마일스톤의 정합도 측면에서 1순위. 다른 5 시스템(간판 용병 솔로/소수정예 의뢰·전투 감정 반응·히든 스탯·UI 폴리싱·주간 기여도 랭킹)이 모두 대시보드에 표시될 데이터를 만들어내므로 대시보드가 통합 점검 도구가 됨
- M9 6 리전 확장의 기반이 되는 구조이므로 region 3 MVP 단계에서 확실히 설계해두는 것이 중요

**1차 범위 (M8.5 페이즈 4 #1 명세 → 구현)**
- HomeScreen LivingsphereSummarySection (6지표 + 30분/8시간 목표 + 자세히 보기)
- InfoScreen LivingsphereDetailScreen (6 펼침 카드 + 2 목표 카드 + 핀 토글)
- 자동 추천 알고리즘 (페이즈 2 가중치 적용)
- 핀 영속 (UserData HiveField 28~29)
- 6 점프 대상 동선 (`currentTabProvider` + 화면별 target provider)

**2차 범위 (M9 위임)**
- 6 리전 확장 (region 31·127·9·10·146·38 데이터 채우기 + 리전 선택 드롭다운)
- 통합 완성도 % 산식 6 리전 가중평균
- 자동 추천 알고리즘 6 리전 가중치
- 임박 임계값 알림 (푸시 또는 인앱)

**3차 범위 (M10+ 위임)**
- 다른 플레이어 비교 (랭킹 시스템과 통합, M8.5 #6 주간 기여도 랭킹과 별도)
- 대시보드 위젯 (홈 화면 외부 — Android 위젯·iOS 위젯)
