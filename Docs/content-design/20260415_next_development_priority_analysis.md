# 다음 개발 우선순위 분석 리포트

> 작성일: 2026-04-15
> 유형: 분석 리포트
> 기반 문서: `docs/content-design/roadmap/trait_enrichment_roadmap.md`, `docs/content-design/20260413_world_expansion_system.md`

## 개요

trait-lifecycle-phase-a와 facility-system-upgrade 완료 후, 트레잇 풍부화 로드맵(Phase B~E)과 세계 확장 시스템(Phase 1~6) 중 다음 개발 대상을 선정하기 위한 우선순위 분석.

## 후보별 평가

### 1. 트레잇 Phase B — 시설 ↔ 트레잇 연계

| 기준 | 평가 |
|------|------|
| 구현 난이도 | **낮음** — `MercenaryStatService`에 지표 추가 + 시설 사용 시 카운트 갱신 |
| 코드베이스 친화도 | **최상** — 시설 시스템과 트레잇 시스템 모두 직전에 작업 완료 |
| 게임 경험 임팩트 | 중간 — 시설이 "골드 넣고 수동 보너스 받기"에서 "트레잇 촉발점"으로 격상 |
| 전략적 가치 | 중간 — 다른 Phase를 해금하지는 않지만, 시설 투자에 새 동기 부여 |

레퍼런스: Melvor Idle의 마스터리 시스템 — 특정 활동 반복이 관련 역량으로 축적되는 구조.

### 2. 세계 확장 Phase 1 — 지역 조사 + 지역 상태

| 기준 | 평가 |
|------|------|
| 구현 난이도 | **중간** — Hive 박스 1개, Supabase 테이블 1개, 새 행동 슬롯, 홈 화면 UI |
| 코드베이스 친화도 | 중간 — 새 feature 모듈이지만, gameTickProvider 연동 패턴은 건설 큐와 동일 |
| 게임 경험 임팩트 | **높음** — "파견만 돌리기"에서 벗어나는 완전히 새로운 행동 축 |
| 전략적 가치 | **최상** — Phase 2~6 전체가 이것에 의존. 세계 확장 전체의 병목 |

레퍼런스: Darkest Dungeon의 탐험 깊이 시스템 — 같은 던전을 반복할수록 숨겨진 방과 비밀이 드러나는 구조.

### 3. 트레잇 Phase C — 용병 간 상호작용

| 기준 | 평가 |
|------|------|
| 구현 난이도 | 중간 — 공동 파견 횟수 추적 + 임계값 트레잇 발생 |
| 코드베이스 친화도 | 높음 — MercenaryStatService에 새 지표, TraitAcquisitionService에 새 조건 |
| 게임 경험 임팩트 | 중~높 — Fire Emblem 지원 시스템처럼 용병 간 관계 형성 |
| 전략적 가치 | 낮음 — 독립적, 다른 Phase를 해금하지 않음 |

### 4. 세계 확장 Phase 6 — 세력 발견

| 기준 | 평가 |
|------|------|
| 구현 난이도 | 중간 — Supabase 1개, Hive 1개, 도감 UI |
| 전략적 가치 | 중간 — Phase 1만 선행하면 됨. 향후 세력 가입/상점의 토대 |

## 권장 순서

```
① 트레잇 Phase B (시설 ↔ 트레잇)     ← 1~2일, 즉시 가능한 Quick Win
      ↓
② 세계 확장 Phase 1 (지역 조사)       ← 핵심 투자, 전체 세계 확장의 뿌리
      ↓
③ 세계 확장 Phase 6 (세력 발견)       ← Phase 1 직후 바로 가능
   또는
③ 트레잇 Phase C (용병 상호작용)      ← 독립 작업, 언제든 가능
```

### 근거

**① Phase B를 먼저 하는 이유:**
- **컨텍스트 비용 최소화** — 시설 시스템과 트레잇 시스템을 연속으로 작업했기 때문에 두 시스템의 코드 구조가 가장 선명한 시점
- **작업량 대비 체감** — `MercenaryStatService`에 시설 관련 지표 추가하고, 시설 사용 지점에서 카운트를 올려주면 기존 `TraitAcquisitionService`의 조건 체크 파이프라인이 자동 처리
- Supabase의 traits 테이블 `acquisition_condition`에 시설 관련 조건만 데이터로 추가하면 코드 변경 최소화

**② Phase 1을 두 번째로 하는 이유:**
- 세계 확장 Phase 2~6 전부가 Phase 1에 의존. 가장 큰 병목을 먼저 해소
- 건설 큐의 "gameTickProvider에서 완료 체크 → 자동 처리" 패턴을 지역 조사에 그대로 재활용 가능

**③ 세 번째를 선택지로 남기는 이유:**
- Phase 6(세력)은 Phase 1 직후 연속성이 좋지만, 아이템 없이는 "발견만 하고 할 게 없는" 상태
- Phase C(용병 상호작용)는 독립적이라 어디에 끼워도 되지만, 세계 확장 흐름을 끊으면 컨텍스트 전환 비용 발생
- Phase 6 권장 — Phase 1에서 만든 discovery 트리거 파이프라인 위에 바로 올라감

## 주의사항

### 트레잇 Phase D와 세계 확장 Phase 4의 중복

트레잇 로드맵의 Phase D(특수 임무 시스템)와 세계 확장의 Phase 4(숨겨진 연계 퀘스트)는 사실상 같은 영역. 둘 다 "특수한 퀘스트 → 트레잇/아이템 보상"을 다룬다. 이 두 Phase에 도달하면 하나의 설계로 통합하는 것이 효율적. 별도로 만들면 퀘스트 생성 로직이 분기되어 복잡도 급증.

### 피해야 할 순서

- **세계 확장 Phase 2(아이템 인프라) 먼저** — 인프라 작업량이 매우 크고, Phase 1 없이는 아이템 드랍 경로가 없음
- **트레잇 Phase E(아이템/스킬북, 티어 업그레이드)** — 세계 확장 Phase 2와 중복, 별도 대형 기획 선행 필요
