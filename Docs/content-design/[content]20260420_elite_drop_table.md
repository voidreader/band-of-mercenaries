## 1. 개요

M2b 엘리트 몬스터 39종(보통 31 + 유니크 8)의 드랍 테이블을 **4개 드랍 카테고리 × 4단계 희소성 등급 × 2계층 구성 규칙**으로 구조화한다. 본 기획서는 수치(정확한 드랍 확률)를 못박지 않으며, 구조·상한·매핑 규칙·구성 원칙만 확정하여 balance-designer 페이즈 2-2의 시뮬레이션 입력을 제공한다.

**핵심 설계 결정 (이 문서가 확정)**:
- 드랍 카테고리 4종 (gold / essence / personal_equipment / guild_artifact)
- 드랍 희소성 등급 4단계 (common / rare / epic / legendary) — 시각적/서사적 구분
- 엘리트당 드랍 항목 수 상한 (보통 4~6 / 유니크 7~9, 전체 평균 5~7)
- 타입 가족 → 정수 축 매핑, 타입 가족 → 장비 슬롯 매핑 (§3.6 상세화)
- 보통 vs 유니크의 드랍 구성 차이 원칙 (3가지 축: 기대 항목 수 / 드랍률 범위 / 고티어 허용 여부)
- 유니크 전용 드랍 풀 (맹주 깃발·유물은 유니크만)
- `elite_loot_tables` 테이블 스키마 초안

## 2. 레퍼런스 분석

| 레퍼런스 | 차용 포인트 |
|---------|------------|
| **Diablo 2 — TC(Treasure Class) 구조** | 각 드랍 항목을 독립 확률로 판정하는 "flat roll table" 모델. 본 기획서도 독립 확률 판정으로 가기로 결정 (§4.2) |
| **Diablo 2 — Super Unique 확정 드랍 풀** | 유니크 엘리트만 추가 접근할 수 있는 "맹주 유물" 카테고리를 둔다. 고정 풀이 아닌 낮은 확률이지만 유니크에서만 나옴 |
| **Melvor Idle — 희소성 태그** | 드랍 아이템에 common/uncommon/rare/very rare 등 희소성 배지가 시각적으로 붙어 드랍 로그에서 한눈에 가치를 인식. 본 기획서의 `rarity_grade` 필드로 차용 |
| **Kingdom of Loathing — 보스 전용 드랍** | 네임드 보스에 "이 보스에서만 나오는 명품 1종"이 존재. 유니크 8종 각각에 "signature drop"(legendary) 1개 필수 배치로 차용 |

**차용하지 않은 요소**:
- Diablo의 가중치 풀 선택식(weighted pick-one) — 독립 확률 판정 대비 UX·디버깅 복잡도만 증가. 배제.
- MMO의 "1주일 쿨다운 확정 드랍" — M2b는 쿨다운 없는 반복 파밍이므로 무용.

## 3. 드랍 카테고리 정의

### 3.1 카테고리 4종

| 카테고리 | `drop_type` | 대상 아이템 풀 | 수량 단위 | M2a 아이템 테이블 참조 |
|---------|-------------|---------------|----------|----------------------|
| **골드** | `gold` | — (아이템 아님) | 정수 (min~max 범위) | 참조 없음 |
| **정수** | `essence` | M2a 정수 20종 | 1개 고정 | `items.category = consumable`, `slot LIKE 'essence_%'` |
| **개인 장비** | `personal_equipment` | M2a 개인 장비 6종 | 1개 고정 | `items.category = personal_equipment` |
| **용병단 유물** | `guild_artifact` | M2a 용병단 장비 4종 (banner 포함) | 1개 고정 | `items.category = guild_equipment` |

**골드를 드랍 테이블에 포함시키는 이유**:
- 구성 단순성: 모든 드랍 결과가 하나의 테이블에서 rolling되므로 UI/로깅이 동일 경로를 탄다.
- 희소성 등급 통합: 골드도 `rarity_grade = common`으로 "일반 드랍"으로 분류되어 퀘스트 완료 팝업의 드랍 리스트가 일관된 모양을 갖는다.
- 다중 골드 행 허용: 예를 들어 유니크는 "기본 골드 드랍(확정) + 보너스 골드 드랍(낮은 확률)" 2행을 두어 "잭팟 골드" 연출이 가능.

### 3.2 카테고리별 드랍 의의

| 카테고리 | 플레이어 체감 | 설계 비중 |
|---------|-------------|----------|
| `gold` | 기본 수익, 인건비·파견비 회수 | **보통 엘리트의 주력** (일반 퀘스트의 1.5~2배) |
| `essence` | 장기 영구 스탯 강화, 가장 바라는 보상 | **유니크 엘리트의 주력** (50~75% 확률). 보통 엘리트는 보조 (10~25%) |
| `personal_equipment` | 즉시 장착 가능한 전력 증강 | 양쪽 모두 중간 비중. 유니크 쪽이 동·상위 티어 포함 |
| `guild_artifact` | 용병단 전체에 거시 영향, 영구성 | **유니크 엘리트 전용** (보통은 0~2% 극희소 노이즈만). 수량 절대 부족 원칙 |

## 4. 드랍 판정 방식

### 4.1 독립 확률 판정 (확정)

각 드랍 행은 **독립적으로 `drop_rate` 확률 판정**. 한 엘리트에 5개 드랍 행이 있고 각각 40%/30%/20%/10%/5%라면, 퀘스트 완료 시 5번의 독립 롤이 수행된다.

**기대 드랍 수 계산 공식**:
```
E[drops] = Σ drop_rate_i   (행별 독립 베르누이 시행)
```

예시:
- 보통 엘리트(드랍 행 5개, 각 100%/20%/15%/10%/5%): 기대값 1.5개 (골드 1 확정 + 0.5 부가)
- 유니크 엘리트(드랍 행 8개, 각 100%/70%/50%/30%/25%/15%/10%/5%): 기대값 3.05개 (골드 확정 + 2~3개 아이템)

**채택 이유**:
- 계산·시뮬레이션·디버깅이 단순 (가중치 기반 pick-one은 0드랍 처리와 복합 상호작용 시 복잡)
- 플레이어 기대값이 예측가능 ("이 엘리트는 대략 정수 1~2개 + 장비 0~1개 나온다")
- balance-designer 페이즈 2-2 시뮬레이션 입력이 단순

### 4.2 드랍 롤 순서

의미적 순서는 없지만 UI 표시 일관성을 위해 다음 순서로 롤·표시:
1. `gold` (rarity 순으로 내림차순)
2. `essence` (tier 내림차순)
3. `personal_equipment` (tier 내림차순)
4. `guild_artifact` (tier 내림차순)

즉, 퀘스트 완료 팝업에서 "골드 → 정수 → 장비 → 유물" 순서로 드랍 리스트가 표시된다.

### 4.3 데이터베이스 레코드 모델 (초안)

`elite_loot_tables` 테이블 컬럼 제안:

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | TEXT PK | 고유 식별자 (예: `elite_golem_rusty_gold_01`) |
| `elite_id` | TEXT FK → `elite_monsters.id` | 소속 엘리트 |
| `drop_type` | TEXT NOT NULL | `gold` / `essence` / `personal_equipment` / `guild_artifact` |
| `item_id` | TEXT FK → `items.id` nullable | drop_type = gold이면 NULL, 그 외는 필수 |
| `gold_min` | INTEGER nullable | drop_type = gold일 때 최소 |
| `gold_max` | INTEGER nullable | drop_type = gold일 때 최대 (min ≤ max) |
| `drop_rate` | REAL NOT NULL | 0.0~1.0 |
| `rarity_grade` | TEXT NOT NULL | `common` / `rare` / `epic` / `legendary` |
| `quantity` | INTEGER NOT NULL DEFAULT 1 | 아이템 개수 (정수·장비는 기본 1, 향후 스택 확장 여지) |

**제약**:
- `drop_type = 'gold'` → `item_id` IS NULL, `gold_min` IS NOT NULL, `gold_max` IS NOT NULL
- `drop_type != 'gold'` → `item_id` IS NOT NULL, `gold_min` IS NULL, `gold_max` IS NULL
- `drop_rate` BETWEEN 0.0 AND 1.0
- 외래키: `item_id`가 NOT NULL일 때 M2a `items` 테이블에 존재해야 함

**인덱스**: `(elite_id)` — 엘리트 완료 시 드랍 테이블 조회 속도용

## 5. 희소성 등급 체계 (`rarity_grade`)

### 5.1 4단계 구조

| 등급 | 드랍률 범위 (가이드) | UI 색상 | 의미 | 보통 엘리트 사용 | 유니크 엘리트 사용 |
|------|---------------------|---------|------|----------------|-------------------|
| **common** | 50% 이상 | 회색 / 흰색 | 확정에 가까운 드랍 | 골드 | 골드 |
| **rare** | 15% ~ 50% | 초록 / 파랑 | 중간 가치 드랍 | 저티어 정수, 저티어 장비 | 중티어 정수, 동·상위 티어 장비 |
| **epic** | 3% ~ 15% | 보라 | 높은 가치 드랍 | 중티어 정수(T3), 중티어 장비 | 고티어 정수(T4), 고티어 장비 |
| **legendary** | 0.1% ~ 3% | 빨강 / 금색 | 희귀 보물 | 거의 없음 (특수 엘리트 예외) | 맹주 유물, T5 정수, T5 장비 |

### 5.2 설계 원칙

- **희소성 등급은 기계적 의미가 아니라 시각적·서사적 구분**. 실제 드랍 판정은 `drop_rate` 필드 값으로 수행한다. `rarity_grade`는 UI 색상·드랍 로그 강조·활동 로그 이모지/톤 결정에 사용.
- **같은 등급도 엘리트마다 다른 `drop_rate`를 가질 수 있다**. 예: T2 보통 엘리트의 "T1 정수 rare = 20%" vs T4 보통 엘리트의 "T2 정수 rare = 18%" — 둘 다 rare 표시이나 수치는 엘리트 티어에 따라 자연 조정.
- **등급·실제 확률의 정합성은 balance-designer가 검증**. 페이즈 2-2에서 "rare로 표시되는데 실제 드랍률이 3%"인 행이 없도록 교차 검증.

### 5.3 활동 로그 연계

퀘스트 완료 시 드랍 결과가 활동 로그에 기록되는데, `rarity_grade`에 따라 톤이 다르게 출력된다 (spec-writer 페이즈 4에서 확정):

- **common**: "전리품: 골드 120개를 획득했다."
- **rare**: "전리품: 오래된 힘의 정수를 발견했다."
- **epic**: "✦ 희귀 전리품: 태고의 지혜의 정수를 발견했다!"
- **legendary**: "✦✦ 전설적 전리품: 늑대왕의 피로 물든 가죽 깃발을 획득했다!"

## 6. 엘리트당 드랍 항목 수 상한

### 6.1 기본 원칙

| 구분 | 드랍 행 수 | 기대 드랍 수(평균) | 필수 골드 행 | 필수 테마 정수 행 |
|------|-----------|-------------------|------------|-----------------|
| **보통 엘리트** | **4~6행** | 1.2 ~ 1.8 | 1행 (common, drop_rate=1.0) | 1~2행 (테마 축 T1~T2/T3) |
| **유니크 엘리트** | **7~9행** | 2.5 ~ 3.5 | 1~2행 (기본 + 잭팟) | 2~3행 (테마 축 T3~T5) |

**구성 비례 (대략)**:
- 보통 엘리트 4~6행 = 골드 1 + 정수 2 + 개인 장비 1 + (선택: 용병단 유물 극희소 1 + 정수 추가 1)
- 유니크 엘리트 7~9행 = 골드 1~2 + 정수 2~3 + 개인 장비 2~3 + 용병단 유물 1~2

**전체 평균 5~7 행** — 39종 × 평균 5.5행 ≈ **총 약 215행**. 페이즈 2-2 시뮬레이션 후 ±30% 조정 여지 두고 **200~270행**을 data-generator 생성 목표로 제시 (페이즈 1-1 §6.2와 일치).

### 6.2 상한의 의미

- **4행 미만**: 드랍 선택지 빈약으로 파밍 재미 저하. 금지.
- **9행 초과**: 드랍 로그 가독성 급락, 기대값 관리 복잡. 유니크 한정 예외 허용(최대 10행, 특별한 경우).
- **보통 엘리트 최대 6행 고정**: 보통 엘리트는 드랍이 단순·예측가능해야 파밍 리듬이 형성됨 (수백 번 반복).

### 6.3 각 카테고리 내 드랍 행 수 상한

한 엘리트 내에서 같은 카테고리가 여러 행 가질 수 있다. 다만 상한 존재:

| 카테고리 | 보통 엘리트 최대 | 유니크 엘리트 최대 |
|---------|---------------|------------------|
| `gold` | 1행 | 2행 (기본 + 잭팟) |
| `essence` | 3행 | 4행 |
| `personal_equipment` | 2행 | 3행 |
| `guild_artifact` | 1행 (극희소 legendary) | 2행 (맹주 유물 + 일반 유물) |

## 7. 드랍 구성 원칙 — 보통 vs 유니크 차이

### 7.1 3가지 차별화 축

| 차별화 축 | 보통 엘리트 | 유니크 엘리트 |
|----------|------------|--------------|
| **1. 기대 항목 수** | 1.2~1.8개 (평균 1.5) | 2.5~3.5개 (평균 3.0) |
| **2. 정수 최고 티어** | 해당 엘리트 티어까지 (T2 엘리트 → 최대 T2 정수) | 해당 엘리트 티어 + 1 (T4 유니크 → 최대 T5 정수 허용) |
| **3. 용병단 유물 접근** | 극희소 legendary 1행만 허용 (드랍률 0.5~2%) | 맹주 전용 유물 1~2행 (rare~legendary, 드랍률 5~15%) |

### 7.2 2계층 드랍 구성 차이 원칙

**보통 엘리트의 역할**: "파밍 리듬 공급기"
- 드랍 종류를 단순하게 유지 → 플레이어가 "이 엘리트 = 저 보상"으로 학습 가능
- 정수는 주로 T1~T2(중간 기준 해당 엘리트 티어의 1~2단계 아래)로 한정
- 장비는 동티어 중심, 상위 티어는 없음
- 용병단 유물은 거의 안 나옴 (0~2% 극희소)

**유니크 엘리트의 역할**: "목표 도전의 보물 창고"
- 드랍 종류를 다양하게 → "오늘은 정수가 3개 나왔다!" 같은 감동 순간 발생
- 정수는 T3~T5 중심 (해당 엘리트 티어 기준 상하 한 단계 허용)
- 장비는 동티어 + 상위 티어 혼합
- **맹주 전용 유물**: 각 유니크마다 "signature drop" 1개 필수 (§8.2)

### 7.3 티어 허용 매트릭스

**정수 드랍 허용 티어 범위 (엘리트 티어별)**:

| 엘리트 티어 | 보통 엘리트 정수 범위 | 유니크 엘리트 정수 범위 |
|------------|---------------------|----------------------|
| T2 | T1 (주) + T2 (부) | T1, T2, T3 |
| T3 | T1, T2 (주) + T3 (부) | T2, T3, T4 |
| T4 | T2, T3 (주) + T4 (부) | T3, T4, T5 |
| T5 | (보통 없음) | T4, T5 |

**개인 장비 드랍 허용 티어 범위**:

| 엘리트 티어 | 보통 엘리트 장비 범위 | 유니크 엘리트 장비 범위 |
|------------|---------------------|----------------------|
| T2 | T2 | T2, T3 |
| T3 | T2, T3 | T3, T4 |
| T4 | T3, T4 | T4, T5 |
| T5 | — | T5 (전설급 포함) |

**설계 의도**:
- 보통 엘리트는 "해당 티어의 안정적 공급"
- 유니크 엘리트는 "한 티어 위까지 꿰뚫는 보상의 정점" → 플레이어가 유니크 토벌을 목표로 삼는 이유 제공

## 8. M2a 아이템 매핑 규칙

### 8.1 정수 20종 매핑

**타입 가족 → 주 드랍 정수 축** (페이즈 1-1 §3.6 재확인·세부화):

| 타입 가족 | 주 드랍 축 | 부 드랍 축 | 미드랍 축 |
|----------|-----------|----------|----------|
| 골렘류 | `essence_str` (60%), `essence_vit` (30%) | — | `essence_int`, `essence_agi` |
| 오크류 | `essence_str` (50%), `essence_int` (30%) | — | `essence_vit`, `essence_agi` |
| 고블린류 | `essence_agi` (50%), `essence_int` (30%) | — | `essence_str`, `essence_vit` |
| 트롤류 | `essence_vit` (50%), `essence_str` (30%) | — | `essence_int`, `essence_agi` |
| 리자드·용족 | `essence_agi` (40%), `essence_str` (30%) | — | `essence_int`, `essence_vit` |
| 언데드류 | `essence_int` (40%), `essence_agi` (30%) | — | `essence_str`, `essence_vit` |
| 정령류 | `essence_int` (60%) | 원소별 부축 (§8.1b) | `essence_str`, `essence_vit` |
| 야수류 | `essence_agi` (50%), `essence_str` (30%) | — | `essence_int`, `essence_vit` |
| 곤충류 | `essence_agi` (40%), `essence_int` (30%) | — | `essence_str`, `essence_vit` |
| 이매·악마류 | `essence_int` (50%), `essence_str` (30%) | — | `essence_vit`, `essence_agi` |
| 유니크 초월형 | 해당 유니크 개체별 서사 반영 | — | (자유) |

**용어**:
- **주 드랍 축 (primary)**: 이 축의 정수가 반드시 1행 이상 포함되어야 함
- **부 드랍 축 (secondary)**: 있으면 좋음, 없어도 무방
- **미드랍 축**: 해당 엘리트에서 절대 드랍하지 않음 (서사 일관성)

#### 8.1.a 정수 드랍 행의 티어 배분 예시

**녹슨 골렘** (T2 보통, 골렘류): 주 STR·부 VIT
- 정수 행 2개:
  1. T1 힘의 정수 (rare, drop_rate = 0.30)
  2. T1 수호의 정수 (rare, drop_rate = 0.15)

**강철 골렘** (T3 유니크, 골렘류 왕): 주 STR·부 VIT
- 정수 행 3개:
  1. T2 오래된 힘의 정수 (rare, drop_rate = 0.50)
  2. T3 고대의 수호의 정수 (epic, drop_rate = 0.40)
  3. T4 태고의 힘의 정수 (legendary, drop_rate = 0.08)

#### 8.1.b 정령류 원소별 부축 매핑

정령류는 원소 특성에 따라 부 드랍 축이 달라진다 (테마 일관성):

| 원소 정령 | 주축 | 부축 |
|---------|------|------|
| 화염 정령 | `essence_int` | `essence_str` (공격적 원소) |
| 물의 정령 | `essence_int` | `essence_agi` (유동적) |
| 대지의 정령 | `essence_int` | `essence_vit` (견고함) |
| 바람 정령 | `essence_int` | `essence_agi` (신속) |

### 8.2 개인 장비 6종 매핑

M2a 개인 장비 6종:

- armor: 사슬 흉갑 (T2~T3)
- helmet: 철 투구 (T2)
- boots: 질풍의 가죽 부츠 (T3)
- accessory 1: 단련의 은반지 (T3~T4)

**타입 가족 → 주 드랍 장비 슬롯** (§3.6 재확인·세부화):

| 타입 가족 | 주 드랍 슬롯 | 드랍 대상 아이템(슬롯→테마) | 미드랍 슬롯 |
|----------|------------|--------------------------|-----------|
| 골렘류 | armor, helmet | 사슬 흉갑, 철 투구 | weapon, boots |
| 트롤류 | armor, helmet | 사슬 흉갑, 철 투구 | accessory |
| 오크류 | weapon | 강철 장검 | armor(낮음), helmet |
| 고블린류 | boots, accessory | 질풍의 가죽 부츠, 단련의 은반지 | armor, helmet |
| 리자드·용족 | weapon, armor | 강철 장검, 사슬 흉갑 | accessory |
| 언데드류 | accessory | 단련의 은반지, 멸혼결(유니크만) | boots(낮음) |
| 정령류 | accessory | 단련의 은반지 | weapon, armor, helmet |
| 야수류 | boots | 질풍의 가죽 부츠 | weapon, armor |

**M2a 개인 장비 6종의 한계**:
- 슬롯별 아이템이 1~2종뿐이므로 장비 드랍은 "슬롯 = 아이템" 관계가 거의 1:1이다. 향후 M4+ 세력 장비·길드 장비 확장 시 매핑 다양화됨.
- 전설급 accessory(멸혼결 등) 1종은 **유니크 엘리트 한정** 드랍 (보통 엘리트는 접근 불가).

#### 8.2.a 전설급 accessory 드랍 할당 (유니크 엘리트 한정)

M2a 확정 전설 1종(기본안: 멸혼결)은 유니크 8종 중 서사적 매칭이 가장 높은 **1~2종에만** 드랍 허용:

### 8.3 용병단 유물 4종 매핑 — 유니크 전용 풀

M2a 용병단 장비 4종:
- banner: 용병단의 깃발 (T3)
- artifact 1: 황금 저울 (T3, 경제)
- artifact 2: 명예의 뿔피리 (T3~T4, 모집)

**원칙**: 용병단 유물은 **유니크 엘리트 전용**. 보통 엘리트는 legendary 극희소(0~2%) 1행에 한해 `guild_artifact` 카테고리를 가질 수 있으나, 기본 구성은 0행.

**유니크 엘리트 → 용병단 유물 매핑 (서사 기반)**:

| 유니크 엘리트 | 드랍 후보 용병단 유물 | 드랍률 가이드 |
|-------------|--------------------|-------------|
| 늑대왕 울부르 (T2, 야수 왕) | 용병단의 깃발 (승리 트로피 감성) | rare, 10~15% |
| 강철 골렘 (T3) | 수호자의 방패 장식 (골렘의 견고함 상징) | epic, 8~12% |
| 습지의 히드라 (T3) | 황금 저울 (고대 유적 보물 감성) | epic, 5~10% |
| 백골의 장군 (T3) | 명예의 뿔피리 (잊혀진 사령관의 뿔) | epic, 5~10% |
| 사막의 파수꾼 (T4) | 수호자의 방패 장식 (파수꾼 서사 직결) | epic, 5~8% |

**signature drop 원칙**:
- 각 유니크에 "이 유니크를 토벌해야만 얻을 수 있는 용병단 유물 1종"을 배정.
- 플레이어가 "이번엔 황금 저울이 필요해 → 습지의 히드라를 토벌하자" 같은 목표 중심 파밍이 가능해짐.
- M2a 유물 4종이 유니크 8종에 2:1로 중복 배치되는 것은 정상 (유물 수량 부족 단계이므로).
- 향후 M4 세력 장비로 확장되면 유니크별 signature drop 재분배.

### 8.4 골드 드랍 매핑

골드는 아이템 풀 참조가 없지만, `gold_min` / `gold_max` 범위를 엘리트 티어·계층별로 다음 가이드:

| 엘리트 티어 | 보통 엘리트 (단일 행) | 유니크 엘리트 기본 행 | 유니크 엘리트 잭팟 행 (선택) |

| T3 | min=150, max=280 | min=400, max=800 | min=1000, max=1800 (drop_rate 8%) |
| T4 | min=280, max=500 | min=800, max=1500 | min=2000, max=3500 (drop_rate 6%) |
| T5 (유니크만) | — | min=1500, max=3000 | min=4000, max=7000 (drop_rate 5%) |

**수치 초안**. balance-designer 페이즈 2-2에서 "일반 퀘스트의 1.5~2배" / "2~3배" 기준으로 재조정.

**잭팟 행 서사**:
- 유니크 엘리트의 "드물게 등장하는 거액 골드" 행은 common이 아닌 **rare** 등급으로 표시 (색상 초록/파랑). 플레이어가 "오늘 대박 났다" 감정 체감.

## 10. 현재 시스템과의 연관

### 10.1 영향받는 기존 시스템

| 시스템 | 영향 | 비고 |
|-------|------|------|
| **M2a `items` 테이블** | 외래키 참조 대상 | `item_id` 30종 모두 FK 검증. M2a 데이터 선행 필수 |
| **M2a `InventoryRepository.addItem`** | 드랍 적재 경로 재사용 | `EliteLootService`가 드랍 결과 rolling 후 각 아이템을 `addItem`으로 투입. 정수/장비/유물 구분 없이 통일 경로 |
| **`UserData.gold`** | 골드 드랍 적재 | `UserDataNotifier.addGold(amount)` 호출 (기존 경로) |
| **`ActivityLog`** | 드랍 결과 로그 | 기존 `ActivityLogType.rewardsReceived`에 세부 드랍 목록 추가 or 신규 `ActivityLogType.eliteDrop` 검토 (spec-writer 페이즈 4 결정) |
| **페이즈 1-1 §4.1 (카탈로그) 시스템 영향 목록** | 본 기획서가 세부화 | `elite_loot_tables` 스키마 확정, `EliteLootService` 책임 범위 확정 |

### 10.2 신규 생성 필요 요소

- `elite_loot_tables` 테이블 (DDL + Supabase 마이그레이션) — 페이즈 4-2
- `EliteLootEntry` Freezed 모델 — 페이즈 4-2
- `EliteLootService` 도메인 서비스 — 페이즈 4-3
  - `Future<List<LootResult>> rollDrops(String eliteId)`
  - 드랍 행 각각 독립 확률 판정
  - 결과를 `LootResult(drop_type, item_id?, gold_amount?, rarity_grade)` 리스트로 반환
- `LootResult` 값 객체 (`features/quest/domain/loot_result.dart` 제안)
- 퀘스트 완료 팝업 드랍 리스트 위젯 — 페이즈 4-4

### 10.3 호환성 검토

- **기존 퀘스트 보상 공식과의 관계**: 엘리트 퀘스트의 기본 보상(`QuestCalculator.calculateReward`)은 그대로 유지. 엘리트 드랍은 **기본 보상 위에 추가**되는 형태. 세력 전용 퀘스트 보상 보너스와 동일한 경로 (공존 가능).
- **인벤토리 용량**: M2a 인벤토리 용량 정책(예: 개인 장비/용병단 장비 개별 상한, 정수 무제한)은 기존 규칙 유지. 드랍 결과가 용량 초과 시 처리는 M2a 규칙 따름 (spec-writer가 재확인).
- **패시브 보너스 상호작용**: 세력 패시브/랭크 패시브의 `gold_reward_multiplier`, `item_drop_rate_bonus`(M2b 신규 가능성) 등과의 상호작용은 페이즈 4-3 개발 명세에서 확정. 기본안: 골드 드랍에는 multiplier 적용, 아이템 드랍률에는 별도 bonus 없음 (패시브 효과 인플레이션 방지).
- **시간 가속 호환**: 드랍 판정은 퀘스트 완료 시점 1회 실행이므로 시간 가속과 무관.
- **활동 로그 100개 상한**: 유니크 엘리트 토벌 시 드랍 행이 8~9개 발생 → 드랍 하나당 로그 1개 생성 시 로그가 빠르게 100개 상한 도달 우려. **대책**: 퀘스트 완료 시 드랍 결과를 **단일 로그**로 요약 ("강철 골렘 토벌. 전리품: 골드 650 + 정수 2개 + 장비 1개 + 유물 1개"). 상세는 퀘스트 완료 팝업에서 확인. spec-writer 페이즈 4-4에서 확정.

## 12. data-generator 지시사항

### 12.1 대상 — 엘리트 드랍 테이블 벌크 생성

- **대상 타입**: `elite-loot-table` (신규 타입 스펙 작성 필요 — 페이즈 3-1에서 `elite-monster`와 통합 또는 분리 결정. 본 기획서는 **분리 권장** — FK 검증 로직이 독립적)
- **대상 테이블**: `elite_loot_tables` (신규)
- **생성 수량**: **약 200~270행** (39종 × 평균 5~7행)

- **구조적 제약**:
  - 각 엘리트의 드랍 행 수: 보통 4~6행 / 유니크 7~9행 (§6.1 준수)
  - 각 엘리트마다 `drop_type = 'gold'` 행 최소 1개 (common, drop_rate=1.0) 필수
  - 각 엘리트마다 `drop_type = 'essence'` 행 최소 1개 (주 드랍 축) 필수
  - 유니크 엘리트 8종은 `drop_type = 'guild_artifact'` 행 최소 1개 (§8.3 매핑) 필수
  - 보통 엘리트는 `drop_type = 'guild_artifact'` 행을 선택적 허용 (0~1개, drop_rate ≤ 0.02, rarity = legendary)
  - 정수·장비 드랍의 item_id는 **§8.1·§8.2·§8.3 매핑 테이블 준수**
  - 정수 티어는 §7.3 티어 허용 매트릭스 준수 (보통 엘리트가 T4~T5 정수 드랍 금지, 유니크 엘리트가 엘리트 티어 +2 정수 드랍 금지 등)
  - `rarity_grade`는 `drop_rate`에 정합 (§5.1 범위 준수)
  - `drop_type = 'gold'` → item_id NULL, gold_min/max 설정 (§8.4 범위)
  - `drop_type != 'gold'` → gold_min/max NULL, item_id 설정

- **톤/세계관 가이드**:
  - 생성 대상이 **구조적 데이터**이므로 서사 텍스트는 없음 (엘리트 몬스터 자체의 lore는 별도 `elite-monster` 타입에서 생성)
  - id 네이밍: `elite_{family}_{variant}_{drop_type}_{nn}` 형식 권장 (예: `elite_golem_rusty_gold_01`, `elite_golem_steel_essence_01`)

- **수치 출처**:
  - 정수·장비 드랍률 범위: `Docs/balance-design/{페이즈 2-2 산출물}` (balance-designer 시뮬레이션 확정값)
  - 골드 min/max: 본 기획서 §8.4 초안 + balance-designer 페이즈 2-2 조정
  - rarity_grade 배분: 본 기획서 §5.1 범위 + 실제 drop_rate 정합 검증

- **특수 요구**:
  - 유니크 8종 각각의 **signature drop** 1개 legendary 필수 보장 (§8.3 표 준수, 드랍률 3~15%)
  - 보통 엘리트 31종은 §8.1 매핑을 엄격히 준수 (미드랍 축 정수는 절대 포함하지 않음 — 서사 일관성)
  - 생성 결과 CSV의 elite_id 참조 무결성 검증: `elite_monsters` 테이블과 교차 체크 필수 (페이즈 3-4 선행)
  - item_id 참조 무결성 검증: M2a `items` 테이블 30종과 교차 체크 필수 (M2a 데이터 선행)

### 12.2 페이즈 3 선행 의존 관계

페이즈 3-5(엘리트 드랍 테이블) 실행 전 완료되어야 할 선행:

## 13. 미결·향후 확정 사항

- **[Q-1] 엘리트 드랍에 패시브 보너스(랭크/세력 `item_drop_rate_bonus`) 적용 여부**: 본 기획서 기본안은 "적용 안 함"(인플레이션 방지). 페이즈 4-3 개발 명세에서 최종 결정. 적용 시 `PassiveBonusService`에 신규 효과 타입 추가 필요.
- **[Q-2] 드랍률 0.0 행 허용 여부**: 데이터 설계 편의를 위해 "드랍 조건 미충족" 상태를 drop_rate = 0.0으로 표현할지, 행 자체를 생성하지 않을지. 추천: **행 자체를 생성하지 않음**. data-generator 검증에 `CHECK (drop_rate > 0.0)` 추가 가능.
- **[Q-3] M2a 전설 accessory 확정안(멸혼결 vs 광란검 vs 이그드라실) 변경 시 §8.2.a 재검토**: data-generator 페이즈 3 시점에 M2a 확정안 확인 후 유니크 매칭 재조정.
- **[Q-4] 드랍 quantity > 1 확장**: 본 기획서는 quantity = 1 고정 (정수도 1개씩 별도 행). 향후 "골드 3개 스택 드랍" 같은 확장 시 quantity > 1 지원 필요. 현 스키마는 컬럼 자체는 존재하므로 확장 여지 있음.
- **[Q-5] 엘리트별 가변 드랍 (난이도·자연환경 조건)**: 예를 들어 "달밤의 늑대왕은 유물 드랍률 2배" 같은 이벤트형 드랍. M2b 범위 외. M3 지역 변형 시점 재검토.
- **[Q-6] 중복 signature drop 방지 장치**: 유니크 8종의 signature drop이 M2a 유물 4종에 중복 배치되는데, 플레이어가 동일 유물을 2개 이상 획득 시 처리(자동 판매 vs 중첩 스택 vs 개수 상한 도달). M2a 인벤토리 정책 따름 (spec-writer 페이즈 4 재확인).
