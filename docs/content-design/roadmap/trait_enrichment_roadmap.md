# 트레잇 경험 풍부화 — 컨텐츠 로드맵

> 작성일: 2026-04-13
> 기반 문서: `Docs/content-design/20260412_trait_system_design.md` 항목 #13~#19
> 선행 완료: Trait-Roadmap Phase 1~5 (메커니즘 완성)

---

## 배경

Phase 1~5를 통해 트레잇 시스템의 메커니즘(획득, 진화, 충돌, 시너지, UI)은 완성되었다. 그러나 트레잇이 발생하는 **촉발점(trigger point)이 퀘스트 완료 한 곳에만 집중**되어 있어, 플레이어 경험이 단조롭다.

이 로드맵은 기획서의 "향후 확장 (트레잇 경험 풍부화)" 항목 #13~#19를 구현 난이도, 기존 시스템 활용도, 게임 경험 임팩트 기준으로 우선순위를 정리한다.

## Phase 구성

### Phase A — 트레잇 라이프사이클 완성

| 항목 | 내용 |
|------|------|
| 포함 | #18 트레잇 삭제 시스템, #13 여행 이벤트 ↔ 선천 트레잇 연계 |
| 목표 | 트레잇의 생성-성장-소멸 흐름을 완성하고, 이동 시스템에 전략적 의미를 부여한다 |
| 핵심 근거 | 현재 후천 트레잇 제거 수단이 없어 잘못된 트레잇이 영구 고착되는 문제(Darkest Dungeon의 Sanitarium 부재와 동일), 선천 빈 슬롯이 영원히 비어 있는 문제 해결 |
| 레퍼런스 | Darkest Dungeon — Sanitarium(특성 제거 비용), Crusader Kings 3 — 이벤트 기반 특성 획득 |

**#18 트레잇 삭제 시스템**

- 후천 트레잇만 삭제 가능 (선천은 면역)
- 삭제 비용/조건 설계 필요 (골드? 시설 레벨?)
- `traitHistory`에 `deleted` 사유 기록
- 용병 상세 오버레이에 삭제 UI 추가
- 구현 난이도: 낮음 — `TraitAcquisitionService` 역방향 로직

**#13 여행 이벤트 ↔ 선천 트레잇 연계**

- 이동 중 이벤트에서 빈 선천 슬롯에 트레잇 부여 기회
- 기존 `TravelEventService`의 12종 이벤트 프레임워크에 새 이벤트 타입 추가
- 먼 거리 이동 → 이벤트 확률 상승 → 선천 획득 기회로 이동에 전략적 의미 부여
- "이 용병에게 빈 선천 슬롯이 있으니 먼 지역으로 이동할까"라는 의사결정 유도
- 구현 난이도: 중간

### Phase B — 기존 시스템 연결

| 항목 | 내용 |
|------|------|
| 포함 | #16 시설 ↔ 트레잇 연계 |
| 목표 | 시설을 수동적 보너스 제공에서 트레잇 획득 촉발점으로 격상한다 |
| 핵심 근거 | 구현 난이도가 낮고(기존 행동 지표 시스템에 지표 추가), 시설의 전략적 가치를 높인다 |
| 레퍼런스 | Melvor Idle — 마스터리 시스템 (특정 활동 반복 → 관련 역량 성장) |

- 훈련소 사용 → 훈련 관련 지표 카운트 → 특정 트레잇 획득 조건
- 의무실 회복 → 생존 계열 지표 카운트
- `MercenaryStatService`에 지표 추가 + 시설 사용 시 카운트 갱신
- 구현 난이도: 낮음

---

### Phase C — 사회적 차원

| 항목 | 내용 |
|------|------|
| 포함 | #17 용병 간 상호작용 이벤트 |
| 목표 | 용병 간 관계를 통해 파견 조합에 전략적 깊이를 추가한다 |
| 핵심 근거 | 반복 공동 파견 → 우정/라이벌 트레잇으로 파견 의사결정에 새 고려 요소 추가 |
| 레퍼런스 | Fire Emblem — 지원(Support) 시스템, Rimworld — 사회적 상호작용 |

- 용병 쌍별 공동 파견 횟수 추적 (Hive에 `coDispatchCount` Map 추가)
- 일정 횟수 이상 공동 파견 시 우정/라이벌 트레잇 발생
- `future_ideas.md`의 "상성 시스템"과 연계 가능
- 구현 난이도: 중간

### Phase D — 컨텐츠 확장

| 항목 | 내용 |
|------|------|
| 포함 | #14 특수 임무 시스템 |
| 목표 | 희귀 퀘스트를 통해 트레잇 직접 부여라는 새 보상 경로를 추가한다 |
| 핵심 근거 | 기획 분량이 많아 상세 설계가 필요하며, `future_ideas.md`의 "퀘스트 후속 효과"와 묶어서 설계하면 시너지가 크다 |

- 특정 조건에서 생성되는 희귀 퀘스트 (리전 조건? 명성 조건? 연속 성공?)
- 트레잇 직접 부여 보상이라는 새 보상 타입
- 기획 선행 필수
- 구현 난이도: 중~높음

### Phase E — 대규모 시스템 (장기)

| 항목 | 내용 |
|------|------|
| 포함 | #15 아이템/스킬북 시스템, #19 용병 티어 업그레이드 |
| 목표 | 게임의 대규모 시스템 확장 |
| 핵심 근거 | 아이템 시스템은 인프라(인벤토리, Hive 박스, Supabase 테이블, UI)가 전무하며, 티어 업그레이드는 경제 밸런스를 근본적으로 변경한다. 모두 별도의 대형 기획이 필요하다 |

**#15 아이템/스킬북 시스템**

- 인벤토리, 저장, 사용 UI 등 대규모 인프라 구축 선행 필요
- `Docs/Archive/misc/idea_note.md`의 "넘버스 아이템", `future_ideas.md`의 "보상 다양화"와 함께 설계

**#19 용병 티어 업그레이드**

- 경제 밸런스 재설계 필수
- `/balance-designer`를 통한 사전 밸런스 검토 권장
