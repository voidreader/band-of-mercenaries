# 📄 게임 기획 러프 문서 (v0.1)

## 1. 프로젝트 개요

- 장르: 텍스트 기반 전략 시뮬레이션
- 플랫폼: 모바일 (Flutter)
- 백엔드: 초기 로컬(JSON), 이후 Supabase 확장
- 레퍼런스: OGAME, 아크메이지, Melvor Idle, Kingdom of Loathing
- 컨셉: 용병단을 운영하며 대륙을 탐험하는 게임

---

## 2. 세계 구조

### 좌표 시스템

Continent : Region : Sector

- Continent: 1개 (프로토타입)
- Region: 1 ~ 199
- Sector: 1 ~ 10
- 총 좌표 수: 1990

---

## 3. Region 설계

### 특징

- Region은 난이도가 아니라 "분포 기반 구조"
- RegionTier 따라 난이도 결정
- Region.json 파일 참조

### 분포 원칙

- 초원: 많음
- 숲/폐허: 중간
- 고대유적/마계경계: 적음
- 심연: 매우 적음

---

## 4. 핵심 게임 루프

용병 모집
→ 위치 이동
→ 퀘스트 랜덤 생성
→ 파견
→ 시간 대기
→ 결과 획득
→ 성장
→ 반복

---

## 5. 용병 시스템

### 용병 모집

- 티어가 더 낮을수록 자주 나오는 구조
- 골드를 사용해서 뽑기
- 2시간마다 한명씩 뽑기

### 구조

용병 {
이름
직업
특성
능력치
상태
}

- 이름/직업/특성이 뽑기시 랜덤하게 주어진다.

### 직업 (Job.json)

티어별로 다양한 직업

### 특성 (Trait.json)

- 강인함 (체력 증가)
- 노련함 (성공률 증가)
- 겁쟁이 (생존률 증가)

다양한 특성 추가 예정

### 상태

- 정상
- 피곤함 (능력치 일정 비율 하락)
- 부상 (시간 후 회복)
- 사망 (영구 제거)

---

## 6. 퀘스트 시스템

### 종류

- 약탈 (보상 높음, 위험 중간)
- 탐험 (보상 랜덤, 위험 낮음)
- 토벌 (보상 중간, 위험 높음)
- 호위 (안정형)
- 습격 (PVP, 프로토타입에서 구현하지 않음)

### 생성 방식

현재 Region 기준 ± 거리

### 난이도

quest_difficulty = region_tier ± 1

```
success_rate =
  50%
+ (power_ratio - 1) × 50%
+ trait_bonus
+ quest_modifier
- distance_penalty
+ random_variance
```

- 대성공 : 보상 2배
- 성공 : 기본 보상
- 실패 : 부상
- 대실패 : 사망률 증가

### 파견 방식

용병대 멤버 일부를 차출하는 방식
파견간 멤버는 돌아오기 까지 다시 파견을 갈수 없다.

파견 멤버가 많을수록 성공률이 올라가는 시스템

## 7. 이동 시스템

distance = region_diff + sector_diff

### 이동 시간

## 13. 게임 플레이 흐름 및 UI 설명

### 하단 버튼

이동 / 파견 / 홈 / 모집 / 설정

이동 : 현재 위치한 대륙/지역 좌표를 보여주고 좌우 이동 버튼을 통해서 지역을 이동할 수 있다. 원하는 지역/섹션을 선택하면 이동소요시간이 표시되고 확인을 누르면 이동하는 방식
파견 : 현재 위치에서 랜덤 퀘스트 보여주고 퀘스트 선택시 파견 인원 지정 및 파견 확인 처리. 파견에 필요한 소요시간과 성공 확률을 보여주면 좋겠다. - 이동중에는 사용할 수 없다
홈 : 중앙에 용병단 야영지 이미지를 굉장히 심플하게 보여주면 좋겠다. 스케치 형식만 되어도 좋겠음. 모닥불 주변에 용병들이 쉬고 있는 모습 단순하게 도트 이미지도 괜찮음

- 파견중인 퀘스트의 남은 시간, 이동중이라면 이동까지 남은 시간이 보여지게 해줄것
  모집 : 새로운 용병을 모집한다
- 2시간마다 한명씩 무료 용병 모집 가능
- 골드를 사용해서 용병 모집 가능

설정 : 이 개발 단계에서는 해야할일이 없다.

첫 시작시 Region Tier 1 구역 중에서 랜덤한 위치에서 시작

이렇게 러프하게만 설명해도 진행이 될지는 잘 모르겠다.
