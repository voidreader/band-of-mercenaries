# 용병단 이야기 — AI 조언용 프로젝트 스냅샷

> 이 문서는 다른 AI에게 프로젝트를 설명하고 조언을 구하기 위한 스냅샷입니다.  
> 작성일: 2026-04-10

---

## 1. 게임 컨셉 요약

**용병단 이야기(Band of Mercenaries)**는 플레이어가 용병단 단장이 되어 용병들을 고용하고 의뢰를 수행하며 명성을 쌓아가는 **방치형(Idle) 텍스트 전략 시뮬레이션 모바일 게임**입니다.

- **플랫폼**: Android/iOS (Flutter)
- **장르**: 방치형 + 텍스트 전략 + 경영 시뮬레이션
- **핵심 감성**: "내 용병단이 성장하고 있다"는 느낌. 레벨업, 명성 상승, 더 어려운 지역 개방.
- **직접 전투 없음**: 플레이어는 **어떤 용병을 보낼지, 어디로 이동할지, 어떤 의뢰를 받을지**를 결정. 파견 후 실제 시간이 흐르고 결과를 확인하는 방식.

## 2. 핵심 게임 루프

```
① 용병 모집 → ② 지역 이동 → ③ 의뢰 파견 → ④ 시간 대기 → ⑤ 결과 확인
      ↑                                                             |
      └──────────────── 골드로 재모집, 반복 ────────────────────────┘
```

## 3. 세계관

- 대륙에는 **199개의 리전(지역)**이 존재하며, 각 리전은 10개의 섹터로 세분화
- 리전은 위험도에 따라 **5단계 티어**로 나뉨
- 상위 티어 지역은 **명성 랭크** 달성 후 잠금 해제

| 티어 | 분위기 | 해금 조건 |
|------|--------|----------|
| 1 | 평화로운 초보 지역 | 처음부터 개방 |
| 2 | 약간의 위험 | 명성 500 (신출내기) |
| 3 | 숙련 용병 필요 | 명성 2,000 (일반) |
| 4 | 위험한 고난도 | 명성 8,000 (숙련) |
| 5 | 극한의 도전 | 명성 25,000 (정예) |

## 4. 시스템별 상세

### 4-1. 용병 시스템

- **티어**: 5단계, 총 **85종 직업**
- **특성**: 강인함(HP+20%), 노련함(성공률+10%), 겁쟁이(생존+30%), 광전사(ATK+25%)
- **상태**: 정상 → 피곤함(5분, 능력치 80%) → 부상(난이도×10분, 파견불가) → 사망(영구 제거)
- **레벨**: 최대 5레벨 (XP 임계값: 0/100/350/850/1850)
- **모집**: 무료(2시간 쿨타임) + 골드 유료 모집. 티어 확률 T1:45% / T2:30% / T3:15% / T4:8% / T5:2%
- **방출**: 퇴직금(인건비×레벨) 지급 후 영구 방출. 재모집 불가

| 티어 | 인건비(의뢰당) | 모집 확률 |
|------|--------------|----------|
| 1 | 10G | 45% |
| 2 | 25G | 30% |
| 3 | 50G | 15% |
| 4 | 100G | 8% |
| 5 | 200G | 2% |

### 4-2. 의뢰(퀘스트) 시스템

- 현재 위치에서 **5~8개 의뢰 자동 생성**, 1시간마다 자동 갱신
- 5개 미만이면 수동으로 채우기 가능

| 유형 | 기본 보상 |
|------|----------|
| 약탈 | 100G |
| 토벌 | 120G |
| 호위 | 90G |
| 탐험 | 80G |

**난이도 5단계** (적 전투력 10~80, 보상 배수 1.0~4.5배)

**성공률 공식**:
```
기본 50% + (아군 전투력 / 적 전투력 - 1) × 50%
          + 특성 보너스 + 퀘스트 보정 - 거리 패널티 + 랜덤 편차
범위: 5% ~ 95%
```

**결과**: 대성공(보상 2배) / 성공 / 실패(부상) / 대실패(사망 위험 증가)

### 4-3. 이동 시스템

- 이동 시간: 거리 1칸 = 30초
- 이동 중 **12종 여행 이벤트** 랜덤 발생 (이동 거리 비례, 최대 80%)
  - 골드 발견, 도적 습격, 날씨 이변, 행운, 특수 조우 등
- **이동 제한**: 파견 중인 용병이 있으면 이동 불가

### 4-4. 경제 시스템

```
순수익 = 기본 보상 × 난이도 배수 - 인건비 - 파견비용
```

| 항목 | 내용 |
|------|------|
| 초기 골드 | 500G |
| 파견비용 | 난이도 및 소요시간 비례 (최대 200G) |
| 방치형 보상 | 앱 미접속 시 분당 1G, 최대 480G(8시간) |
| 여행 이벤트 | +20~120G 또는 -30~60G |

### 4-5. 야영지 시설

시설 업그레이드로 용병단 전체 능력치 향상:

| 시설 | 효과 | 최대 레벨 |
|------|------|----------|
| 훈련소 | 경험치 +10~+50% | 5 |
| 의무실 | 회복 시간 -10~-50% | 5 |
| 주둔지 | 용병 상한 +2~+10 | 5 |
| 정보망 | 의뢰 추가 생성 +1~+3 | 3 |

### 4-6. 명성/랭크 시스템

| 랭크 | 호칭 | 필요 명성 |
|------|------|----------|
| F | 무명 | 0 |
| E | 신출내기 | 500 |
| D | 일반 | 2,000 |
| C | 숙련 | 8,000 |
| B | 정예 | 25,000 |
| A | 전설 | 80,000 |

## 6. 기술 스택

- **언어/프레임워크**: Flutter (Dart)

## 8. 현재 고민 중인 질문들

이 게임을 발전시킬 방향에 대해 조언을 구하고 싶습니다:

1. **게임 루프의 장기 재미**: 방치형 게임에서 플레이어가 장기간 흥미를 유지하려면 어떤 요소가 중요한가? 현재 루프(모집→이동→파견→대기→결과)에서 어떤 부분이 약한가?

2. **콘텐츠 깊이**: 현재 퀘스트 유형이 4종(약탈/토벌/호위/탐험)인데, 이를 확장하거나 다양화하는 좋은 방법은?

3. **이동 시스템 활용도**: 이동이 단순한 이동이 아니라 의미 있는 전략적 선택이 되려면 어떤 요소가 추가되어야 하나?

4. **경제 균형**: 초기 골드 500G에서 시작해서 의뢰/방치 보상으로 성장하는 경제 곡선이 자연스러운가? 어떤 수익/비용 구조가 방치형 게임에 적합한가?

5. **용병의 개성**: 85종 직업은 있지만 실질적 개성이 스탯 수치에 그친다. 용병이 더 "살아있는" 존재처럼 느껴지게 하려면?

6. **세계관 활용**: 199개 리전은 그냥 숫자에 불과하다. 지역을 더 의미 있고 기억에 남는 공간으로 만들려면?
